# Vault 图形历史、设置导览与批处理文件夹安全设计

语言：[English](./2026-07-11-vault-history-settings-navigation-batch-folder-design.md) | **简体中文**

## 目标

Notemd 将提供 Vault 级图形资产历史、支持搜索和收藏的设置导览，以及安全的批处理输出文件夹准备流程。整体保持 Obsidian 原生控件风格，并把持久化、索引、策略和界面渲染职责分离。

## 当前架构与差距

- `src/ui/diagramPreviewHistory.ts` 使用模块级数组保存预览历史，只能选择近期会话；插件重载后数据丢失，也没有搜索、筛选、分页、删除策略和产物管理。
- `src/ui/DiagramPreviewModal.ts` 在预览弹窗中渲染完整历史列表，适合近期记录，不适合 Vault 级管理。
- `src/ui/NotemdSettingTab.ts` 以单个很长的顺序页面渲染设置。设置名称是翻译文本而不是稳定身份，因此基于 DOM 文本的搜索或收藏会随语言变化而漂移。
- 从标题批量生成会把配置的文件夹路径直接传入 `src/fileUtils.ts`，文件夹不存在时立即抛错。创建与非空文件夹授权属于任务开始前的 UI 策略，不应留到批处理执行过程中处理。
- README 介绍了主要能力，但尚未为不断扩大的设置面提供完整、易查找的用户目录。

## 选定架构

### 1. Vault 图形历史索引

建立由插件管理、通过 Obsidian 插件数据持久化的历史仓库。每条记录包含稳定 ID、完成时间、来源笔记路径、图表类型、源格式、标题、源产物路径、已有导出路径、状态和轻量错误元数据。SVG、PNG、PDF、TeX 等大型正文不会重复写入插件设置。

仓库提供完整操作：记录完成、分页查询、读取单条、只删除索引，以及在明确确认后删除索引和指定产物。查询支持关键词、图表类型、源格式、导出可用性、来源笔记、完成时间范围、页码和每页数量。默认按完成时间倒序，每页 20 条。

预览弹窗只保留紧凑的近期历史，并提供“查看全部历史”入口。独立 Vault 历史弹窗负责搜索、筛选、分页、重新打开预览、定位来源笔记、导出和删除。

### 2. 设置目录、搜索、导览与收藏

建立设置目录，为每个设置提供稳定 ID、分类 ID、本地化名称与说明、搜索别名，以及可选的高级或开发者标记。大类包括：常规、模型与提供商、笔记处理、图表、批处理、幻灯片和开发者。

设置页使用 Obsidian 原生的双栏结构：左侧为紧凑的大类与收藏导览，右侧保留现有控件。在窄容器中，导览转为横向选择器。模糊搜索基于规范化后的本地化名称、说明、别名和分类文本；结果继续按分类分组，并说明匹配原因。收藏以稳定 ID 按 Vault 保存，切换语言后仍然有效。

现有设置控件和保存逻辑保持不变。目录只增加身份与发现能力，不复制配置状态，也不引入第二套设置后端。

### 3. 批处理文件夹准备边界

在批处理启动前调用统一的文件夹准备操作：

- 文件夹不存在：询问是否创建，并提供“以后自动创建缺失的批处理目标文件夹”选项。
- 文件夹存在且为空：直接继续，不询问。
- 文件夹存在且非空：显示项目数量和少量示例，对整次批处理确认一次。
- 路径对应文件而不是文件夹：返回明确、可操作的错误。
- 用户取消：正常终止，不写错误日志、不写文件。
- CLI 或无交互 host：返回结构化可恢复结果；没有明确策略时绝不自动创建或使用非空文件夹。

持久化偏好只适用于“缺失文件夹自动创建”。非空文件夹每次批处理调用仍确认一次，绝不逐个生成文件确认。

### 4. README 同步

增加面向用户的设置目录，覆盖新的设置导览、收藏、图形历史、批处理文件夹行为，以及当前说明不足的设置项。英文为 canonical。现有多语言 README 使用受控翻译片段同步，同时保持 Markdown 结构和链接。仓库测试校验语言覆盖和关键设置标识，不提交模型响应或网站构建产物。

## 数据与兼容性

- 已完成的内存预览会话会写入新仓库；此前不存在需要迁移的持久化历史。
- 历史数量由可配置上限约束并提供安全默认值；清理只删除索引，不删除用户文件。
- 新设置采用明确名称，例如 `favoriteSettingIds`、`autoCreateMissingBatchTargetFolders` 和 `diagramHistoryRetentionLimit`。
- 已删除或未知的收藏 ID 在渲染时忽略，只有仍注册的 ID 才显示。
- 所有文件系统路径在边界统一规范化为 Vault 相对路径。

## 错误处理与安全

持久化写入串行执行，避免更新丢失。历史索引写入失败不会使已生成产物失效，而是显示警告并记录诊断。搜索不会修改设置。删除产物与删除索引分开确认，并校验所有解析路径都位于 Vault 内。

## 验证

- 仓库单元测试覆盖排序、模糊查询、筛选、分页、保留上限和删除边界。
- 设置目录测试覆盖稳定 ID、分类完整性、与语言无关的收藏、模糊匹配和响应式导览状态。
- 批处理文件夹测试覆盖缺失、空、非空、文件冲突、记忆自动创建、取消和无交互结果。
- Obsidian 集成检查覆盖插件重载后的历史持久化、历史重新打开和导出、设置搜索与收藏导览，以及非空文件夹的一次性确认。
- README 和本地化契约测试确认所有受支持 README 语言都包含新的用户说明。

## 交付阶段

1. 持久化历史仓库与查询契约。
2. 历史管理界面与预览集成。
3. 设置目录、模糊搜索、大类导览与收藏。
4. 批处理文件夹准备流程和 command host 集成。
5. README 多语言同步、全量验证、Obsidian 实机检查和 main 交付。
