# 图形预览与自适应 Vault 历史设计

语言：[English](./2026-07-19-diagram-popup-adaptive-history-design.en.md) | **简体中文**

## 目标

把图形检查与导出变成最短且可靠的路径，同时让用户无需先打开预览即可访问 Vault 历史。预览与历史必须共享同一套历史视图和操作，但根据宿主表面自适应：预览内使用右侧抽屉，命令面板或侧边栏入口使用独立管理窗口。

## 当前摩擦

- 预览弹窗使用高大的左侧 rail 和 6 个全宽文字按钮，次要操作与图形本身争夺注意力。
- SVG、PNG、PDF 导出以 3 个同级控件出现，没有形成单一明确的导出任务。
- 历史管理器只能从预览中的“管理 Vault 图形历史”进入，没有始终可用的命令或侧边栏入口。
- 打开历史会在预览上再叠一层 Modal。真实 1500x855 测量显示关键控件只有 30px，预览有 10 个可见操作，历史有 13 个控件。
- 历史工具栏默认一次展示所有筛选器，记录把标题、元数据、文件路径、导出状态和操作混在一起，没有稳定的信息分组。

## 选定架构

将 `DiagramHistoryView` 抽为不依赖宿主的共享视图组件。它负责查询状态、加载、重试、空状态/无结果状态、渐进式筛选、记录渲染、分页和操作回调，但不能自行创建或关闭 Modal。

`DiagramHistoryModal` 变为命令面板和侧边栏入口使用的轻量独立宿主。`DiagramPreviewModal` 把同一视图渲染在现有预览弹窗内部的右侧抽屉中。抽屉不是嵌套 Modal：它拥有内部遮罩、焦点进入/返回、Escape 处理，并在窄窗口下切换为全宽内部面板。

由 `main.ts` 负责创建 Vault 历史仓库及完整回调集合。命令面板、侧边栏和预览抽屉宿主统一调用公开的 `openDiagramHistory()` 操作，保证持久化、产物删除、重新打开和错误策略只有一个归属。

## 预览布局

预览弹窗从“两列 rail/舞台”改为画布优先的表面：

1. 顶部栏：图形目标、源路径、紧凑图标操作、唯一主操作“导出”菜单和原生关闭入口。
2. 主舞台：SVG、iframe 或源码预览占据可用宽度，继续作为唯一主视觉焦点。
3. 可选诊断区：警告和错误靠近舞台显示，但不挤压顶部操作。
4. 历史抽屉：固定宽度的右侧抽屉覆盖舞台，不缩小渲染产物；窄窗口下切换为全宽内部面板。

复制源码、保存源码和打开历史使用不低于 44px 的命中区，并提供本地化无障碍名称和 tooltip。导出打开 Obsidian 原生菜单，包含 SVG、PNG、PDF。现有导出函数和 PPI 设置保持不变。

## 历史视图

默认历史表面只显示搜索、总数、最新优先提示和记录列表。单个“筛选”控件渐进展开图形类型、源格式、导出格式和完成日期条件。每条记录固定分成标题/来源、完成元数据、产物/导出状态、主要重新打开操作、导航操作、非破坏性索引移除和单独确认的产物删除。

视图提供稳定的加载骨架区域、Vault 专属空状态、无结果时的一键清除、行内重试状态。重新打开成功后关闭宿主并打开预览；失败时留在当前记录内，不抢走焦点。移除索引永远不删除文件；产物删除永远不包含源笔记，并且只有所有目标产物删除成功后才移除索引。

## 直达入口

- 注册始终可用的“打开图形历史”命令。
- 在侧边栏图形区域增加同名的“文字 + 历史图标”入口；没有活动源文件时仍可使用，能够读取时显示当前 Vault 记录数量。
- 两个入口都使用同一个 `DiagramHistoryView`、独立宿主和仓库回调。
- 预览中的历史入口打开内部抽屉，关闭时把焦点还给触发按钮。

## 无障碍与响应式规则

- 所有关键按钮、select、搜索输入、抽屉关闭控件和分页控件命中区不低于 44px。
- 抽屉打开时管理焦点范围、提供名称，并把焦点恢复到触发器。
- 主操作、次级操作和危险操作使用一致的角色 token；危险操作单独强调，不能使用主操作样式。
- 默认可见选择保持不超过 7 个；高级筛选只有用户请求后才展开。
- 长标题和路径限制为两行截断，同时通过 title/tooltip 提供完整值。
- 窄宽度下抽屉切换为全宽，筛选器堆叠，操作保持可达且不产生横向滚动。
- 遵循 reduced-motion 偏好；抽屉动画只使用 transform/opacity。

## 验证计划

- 增加共享视图渲染、筛选展开、抽屉开关与焦点返回、命令/侧边栏入口、响应式 class、导出菜单、加载/空/错误状态和危险操作边界的单元/UI 合同测试。
- 运行现有图形预览/历史、i18n、设置样式和命令注册套件，以及完整 Jest 和生产构建。
- 使用真实桌面和窄窗口 DOM 指标运行 Frontend Law Auditor。验收目标：快速门禁为 0，得分至少 85，关键目标不低于 44px，默认可见选择不超过 7 个，只有一个视觉焦点，P0/P1 不存在未知证据。
- 将构建插件部署到配置好的 Study Vault 后再进行实机测试，通过 `Obsidian.com` 重载，检查两个直达入口和预览抽屉，并确认可见错误面、`dev:errors` 与 error 级 `dev:console` 为空。
- 验收后删除所有临时 Vault 记录/文件，核对源/部署 bundle 哈希并确认 Git 工作区 clean。

## 不在范围内

本次不修改历史数据结构、渲染产物、导出编码、provider 行为、README 多语言同步或长期图形引擎路线图，也不把历史改造成持久化 Obsidian workspace leaf。

## 实施状态 — 2026-07-19

基于当前已发布的 Modal 架构，计划中的所有阶段均已完成。始终可用的命令入口与侧边栏历史入口已经统一连接到由 repository 支撑的共享 `DiagramHistoryView`。本次实现进一步修正了历史工具栏 class 与无障碍/响应式样式不一致的问题，加入语义化 search/list/status 区域，通过原生 tooltip 保留长标题和路径全文，为预览与历史控件提供 44px 命中区和清晰的键盘焦点，并在不改变 renderer 或导出行为的前提下强化预览标题层级。

预览仍保留直接打开 Vault 历史的按钮和会话内预览历史。当前宿主继续使用独立历史 Modal；把它替换为带焦点陷阱的内部抽屉属于另一项交互系统改造。已交付的结构优先解决控件拥挤，完整保留现有回调，并确保渲染图形继续作为主视觉表面。
