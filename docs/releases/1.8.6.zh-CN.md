# Notemd v1.8.6

## 亮点

- **设置重置能力补齐**：设置页新增两种重置动作，避免用户只能手动逐项回退：
  - **完整重置**：全部插件设置恢复默认值。
  - **部分重置**：除 LLM 提供商相关设置外，其余设置恢复默认值。
- **概念提取同义词抑制开关**：新增概念提取相关设置，可对会生成概念的任务前置同义词规避约束。
- **发布真值同步**：随包版本元数据、欢迎弹窗最近两次更新摘要、README 版本标记与双语 release notes 已统一对齐到 `1.8.6`。

## 新功能

- 设置页新增 **重置设置** 分区，包含：
  - `完整重置`
  - `部分重置（保留提供商）`
- 在 **任务：提取概念** 下新增设置项 **概念提取时替换同义词**。
- 当该开关开启时，会在以下两个任务提示词前置同义词抑制指令：
  - `处理文件/文件夹（添加链接）`
  - `提取概念`
- 前置指令内容为：
  - `Avoid extracting synonyms, semantically similar core concepts, or keywords whenever possible.`

## 修复与鲁棒性

- 新增设置重置专项测试，覆盖完整重置/部分重置行为、提供商保留边界和对象克隆安全性。
- 新增提示词专项测试，覆盖同义词抑制：
  - 仅在开关开启时生效；
  - 仅作用于目标两个任务；
  - 不影响其他任务。
- 补齐简体中文与繁体中文下新增设置面的本地化文案覆盖。

## 对比验证摘要

- **开关关闭前**：`addLinks` / `extractConcepts` 提示词保持原样。
- **开关开启后**：两个任务提示词都会前置同义词抑制指令，原有提示词主体保持不变。
- **部分重置模式** 会保留 provider 预设与任务级 provider/model 选择，仅重置非 provider 设置。
- **完整重置模式** 会恢复完整默认设置对象。
