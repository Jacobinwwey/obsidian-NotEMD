# Notemd v1.8.4

## Highlights

- **Diagram Intent Control**: Added explicit diagram intent selectors in settings and the Notemd Workbench, then hardened generation so explicit user choice is treated as a real output contract instead of a loose preference.
- **Batch Progress + Sync Locality**: Added interrupt-resume progress persistence for batch flows and per-setting sync locality control for API-key-style secrets.
- **Mermaid/Preview Reliability**: Strengthened Mermaid sanitation, retry, artifact saving, and preview-path behavior to reduce malformed-output and wrong-format regressions.
- **Release And Docs Alignment**: Refreshed multilingual READMEs, architecture/progress docs, and the GitHub release workflow so shipped metadata matches the actual codebase.

## New Features

- Preferred diagram intent selector in settings plus a matching selector in the Notemd Workbench/sidebar flow.
- Batch progress store for long-running folder flows, enabling better interrupt/resume behavior.
- Per-setting sync toggle so provider/API-key-adjacent values can stay local instead of syncing blindly across devices.
- In-memory response cache for repeated identical LLM calls during a session.
- Added bilingual architecture/progress direction docs to make the diagram platform trajectory easier to maintain.

## Fixes

- Explicit diagram intent now blocks HTML fallback when the user asked for a diagram, retries with the requested type, and auto-generates missing node labels from ids when needed.
- Mermaid handling now sanitizes trailing whitespace, retries after parse errors, preserves better fence boundaries, and stays pinned to Obsidian's Mermaid 11.4.1 behavior.
- Mermaid/preview export paths now save best-effort content after retry failure, and Vega-Lite artifacts are written as readable `.md` output instead of raw `.json`.
- Release workflow JavaScript action majors were updated to supported versions to avoid future Node 20 deprecation fallout.
- README version/sponsor/badge alignment is now locked by dedicated contract coverage across all localized README files.

## Chores

- Refreshed roadmap, audit, Drawnix feasibility, and maintainer release-workflow docs so they match the current branch state and release boundary.
- Added bilingual `1.8.4` release notes and synchronized `package.json`, `manifest.json`, `versions.json`, and README version banners to `1.8.4`.
