# Notemd v1.8.4

## Highlights

- **Provider Robustness And China-Focused Coverage**: Expanded the OpenAI-compatible preset set with `Xiaomi MiMo` and hardened endpoint handling so provider documentation URLs can be pasted directly without malformed-path regressions.
- **Concept-Note Workflow Guardrails**: Added prerequisite guidance for concept-note-producing flows so add-links / concept-extraction commands and workflows can route users to the right settings before generating broken outputs.
- **Batch Original-Text Extraction + Settings UX Hardening**: Added batch support for "Extract Specific Original Text" and fixed the settings-tab input blur regression that forced users to re-focus after every digit.
- **Release, Docs, And Workspace Hygiene Alignment**: Refreshed release/docs truth, protected local `.trellis` workflow state from future sync cleanup mistakes, and aligned the shipped assets with the current mainline behavior.

## New Features

- Added a first-class `Xiaomi MiMo` preset with MiMo defaults (`https://api.xiaomimimo.com/v1`, `mimo-v2.5-pro`) on the shared OpenAI-compatible runtime.
- Added **Batch Extract Specific Original Text** to the sidebar/workflow action catalog so the configured extraction questions can run across a selected folder.
- Added a concept-note-path warning modal with **Configure**, **Skip once**, and **Do not show again** for add-links / concept-extraction flows that require concept-note generation.
- Expanded the first-install welcome modal so it now includes the latest two release summaries in a dedicated scrollable panel.
- Reordered key settings so concept-note output and processing parameters appear before Translate, and moved merged-query mode into the "Extract Specific Original Text" task settings where it belongs.

## Fixes

- OpenAI-compatible endpoints now normalize pasted `/chat/completions` and `/models` URLs before request assembly, preventing duplicated-path failures.
- Settings numeric/text inputs that previously called `display()` on every keystroke now commit on blur/Enter where appropriate, fixing the "type one digit and the input blurs" regression across the remaining affected fields.
- `Customize Concept Note Path` and `Enable Mermaid Error Detection` now default to enabled, aligning new-user defaults with the workflows that depend on them.
- `Max Tokens` now appears before `Chunk Word Count`, and the chunk-size field documents the one-third recommendation while auto-filling that recommendation when users have not customized it yet.

## Chores

- Refreshed README, architecture, progress-audit, and release documentation so the current codebase, release boundary, and next-direction analysis line up again.
- Added `.trellis/` to `.gitignore` and recorded concise preservation rules in `AGENTS.md` so local Trellis state is not lost during future sync or cleanup work.
