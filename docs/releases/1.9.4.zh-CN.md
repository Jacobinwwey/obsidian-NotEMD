# Notemd v1.9.4

## 亮点

- **图表现在是识别源类型的多格式平台**：CircuitikZ、Draw.io、Drawnix、SVG、PNG、PDF、HTML 与可编辑产物都具有明确的源、渲染、预览与导出契约。
- **CircuitikZ 现在提供可选的桌面端托管运行时**：Notemd 可以发现可信编译器，或在 Vault 外安装固定版本且经过完整性校验的 Tectonic 环境，同时继续提供零依赖预览。
- **图形工作更易查找和管理**：按 Vault 持久化的历史支持搜索、筛选、完成时间排序、分页、产物操作、直达入口与自适应右侧抽屉。
- **复杂设置更易导航**：模糊搜索、稳定收藏与大项导览降低配置成本，同时保留高级控制项。

## 主要升级

- 新增受约束的 CircuitikZ 生成契约、电路族 fixture、拓扑校验、原生编译诊断、修复 brief，以及 SVG/PNG 几何质量门禁。
- 新增清晰可读的 SVG、PNG、PDF companion、可编辑图表导出目标，以及支持 BOM 输入的离线 artifact CLI。
- 新增 CircuitikZ 托管环境，包含 ownership-scoped 激活、安全下载校验、安装/修复/移除操作、编译器发现与原生 smoke test。
- 新增按 Vault 保存的图形历史、搜索、筛选、分页、缺失产物反馈、重新打开/导出操作，以及从命令与侧边栏进入共享自适应历史视图的入口。
- 新增设置模糊匹配、已本地化的稳定收藏标识、大项导览，以及搜索无结果时的明确反馈。
- 新增批处理目标文件夹预检：缺失文件夹或非空文件夹会在任务级确认，并可选择以后自动创建缺失文件夹。
- 补齐已发布的多语言文档路由与内容一致性，同时将生成式 MDX 同步限制在用户可见路由或内容确有变化的场景。
- 加固 Slidev standalone 导出，包括跨页内容、Mermaid 自适应、MP4 帧顺序与清晰度，以及 Windows 环境检测。

## 修复与鲁棒性

- 修复图形历史抽屉的滚动模型，使长历史列表可在预览 shell 内正常上下滚动。
- 改善 CircuitikZ 预览的可读性、标签布局、线宽、裁切检测、箭头对齐与多格式 companion 清晰度。
- 修复历史产物缺失反馈与仅源文件产物的重新打开行为，避免静默或具有误导性的预览状态。
- 修复 diagram artifact CLI 对 UTF-8 BOM 输入及 Windows npm 参数重写的处理。
- 清理生成文档页面中的 NUL 字节，并增强 GitHub Pages 部署重试。
- 恢复 README 中 Star History 与 Friendly Links 的正确层级。
- 建立明确的源与 render target 路由，避免 CircuitikZ 错误落入无关的 mind-map 生成契约。
- 修复 npm 10 冷安装的可复现性，并使 CircuitikZ、CLI fallback 与网站文档契约测试可在 Windows 和 Linux 发布 runner 上稳定执行。

## 对比摘要

- **改动前**：CircuitikZ 依赖用户自行准备 LaTeX 工具链，失败原因也难以诊断。**改动后**：用户可继续使用零依赖预览、选择可信系统编译器，或安装经过校验的托管 Tectonic，并获得原生诊断。
- **改动前**：图表源类型与 render 类型容易混淆，仅源文件产物的导出证据有限。**改动后**：源、renderer、预览 companion 与导出目标均明确分离，并在支持时提供清晰的 SVG/PNG/PDF 与可编辑产物。
- **改动前**：生成图形主要依附当前预览会话。**改动后**：历史按 Vault 持久化，可搜索、筛选、分页、重新打开，并可从命令与侧边栏直达。
- **改动前**：设置页内容密集，只能手动逐项查找。**改动后**：用户可以模糊搜索、收藏常用设置，并在主要设置分区间快速跳转。
- **改动前**：已发布文档路由以及 Slidev/Pages 交付存在一致性与收敛缺口。**改动后**：多语言路由完整，导出与部署流程也具有明确的验证门禁。
