# Notemd v1.8.7

## Highlights

- **Folder-Task Selector Convergence**: Folder-level process/extract/translate/fix flows now share one file-selection contract instead of scattered task-specific logic.
- **Operation-Override Parity**: Folder operation inputs now consistently support per-run include-subfolders and file-filter overrides, including `batch extract original text`.
- **Release Truth Synchronization**: Packaged metadata, welcome-modal latest-two release digest, README version markers, and bilingual release notes are aligned to `1.8.7`.

## New Features

- Added shared folder-task selector helper coverage for:
  - filter modes: `No filter` / `Contains` / `Regex` / `Glob`
  - filter targets: `Relative path` / `Basename`
  - case sensitivity and invert matching
  - include-subfolders compatibility mode (`legacy` default keeps translation non-recursive unless explicitly changed)
- Added operation-level folder-selection override propagation through canonical host-adapter paths for folder workflows.
- Completed operation contract registration and schema parity for `content.batch-extract-original-text`.

## Fixes And Robustness

- Converged regex validation semantics by introducing shared selector-layer precheck helper `getFolderTaskRegexValidationError`, used by both settings UI and runtime matcher logic.
- Added host-adapter regression tests to lock:
  - `folderPathOverride + fileSelectionOverride` execution behavior
  - base settings non-mutation guarantees during override merge
- Kept invalid-regex feedback non-blocking in settings while surfacing earlier localized warnings to reduce delayed runtime failure discovery.

## Comparative Verification Summary

- **Before**: folder-task selection/filter logic was fragmented with higher drift risk between settings guidance and runtime behavior.
- **After**: selector contract, settings validation, and runtime matching use aligned semantics with shared helper-backed regex validation.
- **Before**: operation-level folder overrides were inconsistent across folder workflows.
- **After**: canonical host-adapter paths now consistently apply scoped overrides, with batch original-text extraction contract explicitly covered.
- **Stability**: translation legacy default scope remains preserved under `includeSubfolders = legacy`.
