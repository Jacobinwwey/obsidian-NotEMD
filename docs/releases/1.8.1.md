### Highlights
This patch closes the language-support gap that remained after `1.8.0`. Notemd now completes the visible UI localization pass across every advertised in-app UI locale, so settings, sidebar actions, workflow builder controls, extracted-text settings, advanced options, and output-path configuration surfaces follow Obsidian's language selection instead of silently falling back to English.

It also corrects the documentation contract. The root READMEs and Language Hub now distinguish maintainer source docs, published README translations, and the exact in-app UI locale catalog. English fallback remains in the implementation as a safety net, not the documented normal path for supported locales.

### Breaking Changes
* **No user-facing migration is required**: Existing settings continue to work, and `uiLocale: auto` still follows Obsidian.
