# Notemd v1.8.9

## Highlights

- **Preview consistency is now product-real**: `Preview diagram` reopens the saved Mermaid source path, so the first preview shown right after generation and later manual preview runs stay on the same artifact truth.
- **Preview modal UX is finally frame-safe**: the Mermaid preview modal now uses a vertical action rail, preview history panel, and wrapper-level width control so small diagrams no longer create unnecessary horizontal scrolling.
- **Release truth is synchronized again**: packaged metadata, welcome-modal release digest, README markers, and bilingual release notes are aligned to `1.8.9`.

## New Features

- Added preview-session history inside the Mermaid preview modal so users can switch between previously opened diagram previews.
- Moved preview actions into a vertical rail, keeping all controls visible inside the initial modal frame.
- Extended the welcome-modal release digest so the latest entries now advance to `1.8.9` and `1.8.8`.

## Fixes And Robustness

- Follow-up `Preview diagram` runs now reopen the persisted saved source content instead of reusing potentially stale in-memory artifact state.
- The preview modal now sizes the outer Obsidian `.modal` shell together with the inner content, eliminating the bottom horizontal scrollbar caused by the old 560px wrapper default.
- Preview history entries, source-path labels, and action buttons now wrap safely without inflating layout width.
- Saved Mermaid preview reopening, history switching, and modal sizing were verified in a live local Obsidian vault against both `index.zh-CN_summ.md` and `architecture.md`.

## Comparative Verification Summary

- **Before**: post-generation preview and later manual preview could drift because they were not guaranteed to reopen the same saved artifact truth.
- **After**: both paths reopen the saved source boundary, so preview behavior is stable across repeated user entrypoints.
- **Before**: the preview content area could be visually small while the outer Obsidian modal still kept a hidden 560px shell and exposed an unnecessary horizontal scrollbar.
- **After**: the wrapper and content widths now converge, and the modal keeps all controls visible without horizontal swiping for the validated Mermaid cases.
