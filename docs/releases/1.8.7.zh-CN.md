# Notemd v1.8.7

## 亮点

- **文件夹任务 selector 收敛**：文件夹级处理/提取/翻译/修复路径已统一复用一套文件选择契约，不再分散实现。
- **operation 覆盖参数对齐**：文件夹任务的按次覆盖参数已在 canonical host-adapter 路径上统一支持，包含 `批量提取指定原文`。
- **发布真值同步**：随包版本元数据、欢迎弹窗最近两次更新摘要、README 版本标记与双语 release notes 已统一对齐到 `1.8.7`。

## 新功能

- 文件夹任务共享 selector 契约支持：
  - 筛选模式：`No filter` / `Contains` / `Regex` / `Glob`
  - 筛选目标：`Relative path` / `Basename`
  - 大小写敏感、反向匹配
  - 子目录兼容模式（默认 `legacy` 保持翻译任务非递归，除非用户显式改为 include）
- 文件夹任务 operation 输入的覆盖参数已贯通到 host-adapter 统一执行路径。
- `content.batch-extract-original-text` 的 operation 注册、输入/输出 schema 与能力清单契约已补齐对齐。

## 修复与鲁棒性

- 引入共享 regex 校验 helper `getFolderTaskRegexValidationError`，并同时用于设置页预检与运行时 matcher 逻辑，消除双实现漂移风险。
- 增加 host-adapter 回归测试，锁定：
  - `folderPathOverride + fileSelectionOverride` 的执行语义
  - 覆盖参数合成过程中 base settings 不被污染
- 设置页在 regex 模式下可更早给出本地化非法 pattern 提示，同时保持“非阻塞编辑”行为，降低“运行时才发现错误”的延迟成本。

## 对比验证摘要

- **改动前**：文件夹任务筛选逻辑分散，设置提示与运行时语义存在潜在漂移风险。
- **改动后**：selector 契约、设置校验与运行时匹配语义统一，regex 校验由共享 helper 锁定一致性。
- **改动前**：operation 级文件夹覆盖在不同工作流中对齐度不足。
- **改动后**：canonical host-adapter 路径已统一应用覆盖参数，并明确覆盖了批量原文提取契约。
- **稳定性**：`includeSubfolders = legacy` 下翻译任务默认范围保持不变，避免历史行为回归。
