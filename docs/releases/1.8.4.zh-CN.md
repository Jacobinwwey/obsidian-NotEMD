# Notemd v1.8.4

## 亮点

- **图表意图控制**：设置页与 Notemd Workbench 均新增显式图表意图选择器，并把生成链路加固为“用户明确选择就是输出约束”，不再只是软偏好。
- **批处理进度与同步本地性**：为批量流程加入可中断恢复的进度持久化，并为 API Key 一类配置加入逐项同步本地性控制。
- **Mermaid / 预览链路稳定性**：加强 Mermaid 清洗、重试、产物保存和预览路径行为，降低格式错误和输出退化问题。
- **发布与文档对齐**：刷新多语种 README、架构/进展文档和 GitHub release workflow，使对外发布元数据与真实代码状态一致。

## 新功能

- 设置页新增首选图表意图选择器，Notemd Workbench / 侧边栏链路新增对应控制入口。
- 为长耗时批量文件夹流程新增 batch progress store，改进中断后恢复能力。
- 为 provider / API Key 相关配置新增逐项 sync toggle，可按项决定保留本地还是参与同步。
- 为同一会话中的重复相同 LLM 请求新增内存级响应缓存。
- 新增双语架构与进展方向文档，便于持续维护图表平台主线。

## 修复

- 显式图表意图现在会阻止 diagram 请求退化到 HTML fallback，在失败时按请求类型重试，并在需要时从 id 自动补出缺失节点标签。
- Mermaid 处理链路现在会清洗尾随空白、在 parse error 后重试、保持更稳的代码围栏边界，并对齐到 Obsidian 使用的 Mermaid 11.4.1 行为。
- Mermaid / 预览导出链路在重试失败后会落盘 best-effort 内容；Vega-Lite 产物改为保存为可读的 `.md`，不再直接落原始 `.json`。
- release workflow 中 GitHub 维护的 JavaScript actions 已升级到受支持 major，避免未来 Node 20 弃用路径演变为发布失败。
- 所有本地化 README 的版本号、赞助行与 badge 对齐现已由专门契约测试锁定。

## 杂项

- 已刷新路线图、审计、Drawnix 可行性和维护者发布流程文档，使其与当前分支状态和发布边界一致。
- 已新增 `1.8.4` 双语发布说明，并同步 `package.json`、`manifest.json`、`versions.json` 与 README 版本标识到 `1.8.4`。
