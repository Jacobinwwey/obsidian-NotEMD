# Notemd v1.8.4

## 亮点

- **Provider 鲁棒性与中国区覆盖增强**：在共享 OpenAI-compatible 运行时上新增 `Xiaomi MiMo` 预设，并加固 endpoint 处理，使用户可直接粘贴 Provider 文档中的完整接口地址而不再触发路径拼接错误。
- **概念笔记工作流护栏**：为依赖概念笔记生成的命令与工作流补齐前置配置提示，避免在未正确设置概念笔记路径时继续生成错误产物。
- **批量提取指定原文与设置页交互加固**：新增“提取指定原文”的批量能力，并修复设置输入框输入一个数字就立刻失焦的回归问题。
- **发布、文档与工作区卫生对齐**：刷新 release/docs 真值，保护本地 `.trellis` 工作流状态，确保当前主线行为与发布资产重新一致。

## 新功能

- 新增一等 `Xiaomi MiMo` 预设，内置 MiMo 默认配置（`https://api.xiaomimimo.com/v1`、`mimo-v2.5-pro`），并继续复用共享 OpenAI-compatible 调用链路。
- 侧边栏 / 工作流动作目录新增 **批量提取指定原文**，可对选中文件夹内的全部 `.md` / `.txt` 文件执行同一组提问配置。
- 新增概念笔记路径警告弹窗，为添加链接 / 提取概念相关流程提供 **进行配置**、**本次不提示**、**不再提示** 三个选项。
- 首次安装欢迎弹窗现已内置最近两个版本的更新摘要，并提供独立可滚动区域以便查看较长内容。
- 关键设置项顺序已调整：概念笔记输出与处理参数前移到 Translate 之前，合并查询模式已收纳到“提取指定原文”任务设置中。

## 修复

- OpenAI-compatible endpoint 现在会归一化已经包含 `/chat/completions` 或 `/models` 的 Base URL，避免重复拼接导致的请求失败。
- 先前会在输入过程中触发 `display()` 重绘的设置项现已改为在 blur / Enter 时提交，修复“输入一个数字就立刻弹开”的失焦问题，并覆盖剩余受影响字段。
- `自定义概念笔记路径` 与 `启用 Mermaid 错误检测` 现默认开启，更符合依赖概念笔记与 Mermaid 后处理的实际使用路径。
- `最大 Token 数` 已前移到 `分块词数` 之前，`分块词数` 现明确提示推荐值约为前者的三分之一，并在用户尚未自定义时自动填充该推荐值。

## 杂项

- 已刷新 README、架构、进度审计与发布文档，使当前代码、发布边界和后续推进方向重新保持一致。
- 已将 `.trellis/` 纳入 `.gitignore`，并在 `AGENTS.md` 中补充简练的本地 Trellis 状态保护要求，避免后续同步或清理误删。
