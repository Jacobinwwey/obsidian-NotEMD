# Notemd v1.8.6

## Highlights

- **Settings Reset Controls**: Added two explicit reset actions in Settings so users can recover quickly without manual cleanup:
  - **Complete reset**: reset all plugin settings to defaults.
  - **Partial reset**: reset all non-provider settings while preserving current LLM provider configuration.
- **Concept Extraction Synonym-Suppression Toggle**: Added a new Extract Concepts option that prepends a strict synonym-avoidance instruction for concept-producing workflows.
- **Release Truth Synchronization**: Packaged metadata, welcome-modal latest-two release digest, README version markers, and bilingual release notes are aligned to `1.8.6`.

## New Features

- Added a new **Reset settings** section in the settings page with:
  - `Complete reset`
  - `Partial reset (keep providers)`
- Added the new setting **Replace synonyms during concept extraction** under **Task: Extract Concepts**.
- When this setting is enabled, Notemd prepends the following instruction to both `Process File/Folder (Add Links)` and `Extract Concepts` task prompts:
  - `Avoid extracting synonyms, semantically similar core concepts, or keywords whenever possible.`

## Fixes And Robustness

- Added dedicated reset-helper tests to verify complete vs partial reset behavior, provider-preservation boundaries, and object-cloning safety.
- Added dedicated prompt tests to verify that synonym suppression:
  - only activates when enabled,
  - applies only to the two intended tasks,
  - does not affect unrelated tasks.
- Added localized settings coverage for Simplified/Traditional Chinese in the new settings surfaces.

## Comparative Verification Summary

- **Before (toggle off)**: `addLinks` / `extractConcepts` prompts remain unchanged.
- **After (toggle on)**: both prompts are prepended with the synonym-suppression instruction, with all existing prompt content preserved after it.
- **Partial reset mode** preserves provider presets and task-level provider/model selections while resetting non-provider settings.
- **Complete reset mode** restores the full default settings object.
