# Notemd v1.8.8

## Highlights

- **Sidebar API observability is now operator-ready**: quick deep-debug toggle, retry-aware liveness semantics, and compact API activity history now ship together without burying `Log output`.
- **`Preview diagram` is now a real saved-artifact fallback**: supported Mermaid, JSON Canvas, Vega-Lite, and HTML diagram files can be previewed directly instead of re-entering generation for already-saved outputs.
- **Release truth is synchronized**: packaged metadata, welcome-modal latest-two release digest, README markers, and bilingual release notes are aligned to `1.8.8`.

## New Features

- Added direct preview support for:
  - Mermaid Markdown fences
  - Vega-Lite Markdown fences
  - Vega-Lite JSON source files (`.json`)
  - JSON Canvas files (`.canvas`)
  - HTML artifact files (`.html`)
- Expanded canonical preview command/sidebar entrypoints so supported saved artifact files are accepted instead of remaining Markdown-only.
- Added quick `Deep debug` control in the sidebar `Log output` footer header.
- Added request-scoped API liveness states covering waiting, accepted, receiving, long-running healthy, received, and interrupted phases.
- Added compact active/recent API activity sections with inline per-request histories and copied report export.

## Fixes And Robustness

- Saved Mermaid preview no longer re-enters the generation path, removing the `requested intent` mismatch failure from the direct preview fallback flow.
- Legacy Mermaid execution no longer coerces Mermaid-compatible requested intents such as `erDiagram` into `mindmap`.
- API activity history is now capped and collapsible so it does not crowd out the log output panel during multi-request runs.
- Preview tooltip wording now matches shipped behavior: preview supported diagram sources or saved artifact files instead of describing a save-less generation flow.
- Request-scoped liveness semantics now stay retry-aware and concurrency-safe across overlapping provider calls.

## Comparative Verification Summary

- **Before**: sidebar observability existed mostly as logs and settings-only debug controls, while API activity could crowd the footer and saved diagram preview still had gaps outside Vega-Lite.
- **After**: sidebar observability, quick debug control, compact request activity, and saved-artifact preview behavior are all wired through the shipped runtime and command surfaces together.
- **Before**: `Preview diagram` was not a reliable fallback for saved Mermaid/Canvas/HTML artifacts.
- **After**: direct preview works from supported saved source files, so native Obsidian rendering issues no longer force a regeneration detour just to inspect the diagram.
