# Notemd v1.8.9

## 亮点

- **预览一致性已收口到真实产物边界**：`预览图形` 现在会重新打开已保存的 Mermaid 源路径，因此生成后首次弹出的预览与后续手动预览都会落到同一份 artifact 真值上。
- **预览弹窗 UX 终于做到 frame-safe**：Mermaid 预览弹窗现已采用纵向操作栏、预览历史面板，以及 wrapper 级宽度控制，小图场景不再出现无意义的横向滚动。
- **发布真值再次同步完成**：随包版本元数据、欢迎弹窗更新摘要、README 版本标记与双语 release notes 已统一对齐到 `1.8.9`。

## 新功能

- 在 Mermaid 预览弹窗中新增 preview session 历史，用户可直接切换此前打开过的图形预览。
- 将预览操作按钮改为纵向栏布局，保证所有控制项在初始弹窗框架内即可见。
- 欢迎弹窗中的版本摘要已推进到 `1.8.9` / `1.8.8`。

## 修复与鲁棒性

- 后续 `预览图形` 调用现在会重新打开已保存的源内容，而不是继续复用可能过期的内存态 artifact。
- 预览弹窗现在会同时控制 Obsidian 外层 `.modal` shell 与内部内容宽度，消除了旧默认约 560px wrapper 带来的底部横向滚动条。
- 预览历史条目、源文件路径标签与操作按钮现在都支持安全换行，不再通过撑宽布局来“争空间”。
- 已在本机真实 Obsidian vault 中使用 `index.zh-CN_summ.md` 与 `architecture.md` 对已保存 Mermaid 预览重开、history 切换与 modal 尺寸行为进行了验证。

## 对比验证摘要

- **改动前**：生成后的首次预览与后续手动预览可能发生漂移，因为它们不一定重新打开同一份已保存 artifact 真值。
- **改动后**：两条路径都会重新进入已保存源边界，因此重复预览时行为稳定一致。
- **改动前**：即使图本身很小，外层 Obsidian modal 仍可能停留在隐藏的约 560px 壳层宽度，并暴露不必要的横向滚动条。
- **改动后**：wrapper 与内容宽度已收敛，经过实测的 Mermaid 场景中，弹窗无需横向滑动即可完整使用。
