# Notemd v1.9.0

## Highlights

- **Advanced batch selection is now run-safe**: the developer-gated folder-task popup now supports saved file-selection profiles, one-run folder/rule overrides, preset chips, and an inline selected-file preview before execution.
- **Relaxed input types now land as a real contract, not a fake extension toggle**: selected source-preserving tasks can read broader text-like files plus PDF text extracted through Obsidian's PDF runtime, while source-mutating and verbatim-sensitive workflows remain intentionally restricted.
- **Release truth is synchronized to `1.9.0`**: packaged metadata, welcome-modal release digest, README markers, chronicle refresh, and bilingual release notes now agree on the shipped boundary.

## New Features

- Added the developer-gated `Relax input file restrictions` setting.
- Added a shared supported-input reader layer that can read additional text-like files and PDF text for allowlisted tasks.
- Expanded the advanced batch folder popup with:
  - saved file-selection profiles
  - current-default virtual profile handling
  - temporary folder overrides that do not mutate the saved profile
  - temporary rule overrides and clickable preset chips
  - selected-file preview with scrollable inspection

## Fixes And Robustness

- Sidebar actions, command-palette entrypoints, batch host adapters, and path-based diagram generation now reuse the same task-aware input validation contract instead of drifting per surface.
- Batch translate and batch concept extraction now report `no supported input files` when relaxed mode is enabled instead of incorrectly claiming the folder only lacked `.md` / `.txt` files.
- PDF handling now uses Obsidian's PDF.js runtime via binary reads, avoiding the unstable pattern of loosening extension checks without changing the reader path.
- Developer-only batch selection complexity remains hidden unless both `Developer mode` and `Advanced batch file selection` are enabled, preserving the simpler default picker for regular users.

## Comparative Verification Summary

- **Before**: broader file-type support would have required ad-hoc extension loosening, while downstream readers still assumed plain text and could fail at runtime.
- **After**: task allowlists and a shared reader layer make the relaxed-input path explicit, while unsafe tasks remain locked to stable file classes.
- **Before**: saved regex/glob selection flows were increasingly powerful but still lacked enough run-time guardrails for quick one-off adjustment and inspection.
- **After**: batch runs can reuse a saved profile, temporarily adjust folder or filter scope, preview the exact matched files, and keep the persisted profile untouched.
- **Verification**: validated locally with repo-saga serial refresh, `npm run build`, `npm test -- --runInBand`, `npm run audit:i18n-ui`, `npm run audit:render-host`, `obsidian help`, `obsidian-cli help`, repo-local `obsidian build`, repo-local `obsidian test`, and `git diff --check` before release.
