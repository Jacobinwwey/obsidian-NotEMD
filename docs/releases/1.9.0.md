# Notemd v1.9.0

## Highlights

- **This is the full accumulated `1.8.9 -> 1.9.0` roll-up**: the live `1.9.0` notes now describe the whole shipped delta instead of only the final advanced-batch slice.
- **Notemd now carries a deliberate CLI/public-surface architecture**: registry-backed operations, typed capability/contract export, redacted provider-profile export, and a repo-local maintainer helper make the automation boundary explicit without over-claiming a broad public CLI.
- **Diagram generation and preview have matured into a verified platform slice**: spec-first multi-target generation, saved-artifact preview truth, preview history, export flow, and semantic verification helpers now land together.
- **Stage C knowledge workflows are materially stronger**: local knowledge retrieval, chapter split contracts, advanced batch selection profiles, and relaxed input restrictions all gained safer runtime contracts and better inspection.

## Major Upgrades

- Expanded provider/runtime support and stability, including broader China-friendly presets, long-request diagnostics, streaming-aware fallback handling, and more defensive connection/retry logic.
- Completed a large i18n pass across settings, notices, workflow surfaces, sidebar actions, preview chrome, welcome/onboarding copy, and the translated README family.
- Landed the registry-backed CLI extraction layer: operation registry, capability manifest export, invocation contract export, bounded public-safe surface export, provider/config profile commands, and a repo-local maintainer invoke/help bridge.
- Built out the diagram platform: spec-first Mermaid generation, JSON Canvas and Vega-Lite targets, renderer/cache/preview plumbing, saved-artifact preview reopening, preview history, canonical action wording, and typed follow-through results beneath `diagram.generate`.
- Added maintainer-grade semantic and packaging verification: render-host bundle audit, `npm run verify:diagram-semantics`, packaging-contract checks derived from build/release truth, numeric tag enforcement, and chronicle/release workflow hardening.
- Expanded note-processing and knowledge workflows: local knowledge retrieval for batch title generation, research & summarize, and diagram generation; chapter split with deterministic TOC artifacts and guarded reruns; batch progress persistence and response caching.
- Shipped advanced batch folder selection and relaxed input control: named file-selection profiles, regex/glob rules, `relativePath` vs `basename` targeting, include-subfolder control, preset chips, one-run overrides, selected-file preview, and the developer-gated relaxed-input reader path including PDF extraction for allowlisted tasks.

## Fixes And Robustness

- Synced preview flows so manual `Preview diagram` reopens the saved artifact source rather than drifting from the post-generation preview.
- Hardened Mermaid compatibility with parse-error retry, pipe-label escaping, best-effort save behavior, explicit intent routing, and safer legacy/fixer decomposition boundaries.
- Reduced drift across command palette, sidebar, folder-task hosts, maintainer helpers, and path-based operations by reusing shared host-adapter and operation contracts.
- Recovered bounded backup-branch work onto current `main`, re-audited packaging truth, protected local `.trellis/` state, and kept release/docs truth aligned after recovery.
- Tightened automation hygiene with bilingual checked-in release notes, required release assets, numeric-only release tags, repo-saga serialization locks, contributor normalization, and docs scans restricted to tracked files.
