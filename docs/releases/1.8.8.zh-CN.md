# Notemd v1.8.8

## 亮点

- **Sidebar API 可观测性已具备实战可用性**：快捷 deep-debug 开关、retry-aware 测活语义与紧凑 API activity 历史现在已经一起落地，同时继续保证 `Log output` 可见。
- **`预览图形` 现在是真正的已保存产物兜底入口**：受支持的 Mermaid、JSON Canvas、Vega-Lite 与 HTML 图形文件都可以直接预览，不再对已保存输出重新走生成链路。
- **发布真值同步完成**：随包版本元数据、欢迎弹窗最近两次更新摘要、README 版本标记与双语 release notes 已统一对齐到 `1.8.8`。

## 新功能

- 新增对以下已保存图形源的直接预览支持：
  - Mermaid Markdown fence
  - Vega-Lite Markdown fence
  - Vega-Lite JSON 源文件（`.json`）
  - JSON Canvas 文件（`.canvas`）
  - HTML 产物文件（`.html`）
- 扩展 canonical preview 命令与 sidebar 入口：受支持的已保存产物文件现在可以直接触发预览，不再局限于 Markdown 当前文件。
- 在 sidebar `Log output` footer header 中新增快捷 `Deep debug` 控制。
- 新增 request-scoped API 测活状态，覆盖 waiting、accepted、receiving、长任务健康、received 与 interrupted 阶段。
- 新增紧凑的 active/recent API activity 区块，支持 inline per-request history 与报告导出。

## 修复与鲁棒性

- 预览已保存 Mermaid 图形时不再重新进入生成链路，从而移除了 direct preview fallback 流程中的 `requested intent` mismatch 失败。
- legacy Mermaid 执行不再把 `erDiagram` 这类兼容 Mermaid 的显式 requested intent 强制回退为 `mindmap`。
- API activity 历史现在具备上限与折叠能力，避免多请求场景下挤占日志输出区域。
- Preview tooltip 文案现已与实际行为对齐：明确说明它会预览受支持的图形源或已保存产物文件，而不是“生成但不保存”。
- request-scoped 测活语义现在在重试与并发场景下保持更稳定的一致性。

## 对比验证摘要

- **改动前**：sidebar 可观测性更多停留在日志与设置页 debug 控制上，API activity 容易挤占 footer，而已保存图形的预览兜底除 Vega-Lite 外仍有缺口。
- **改动后**：sidebar 可观测性、快捷 debug 控制、紧凑 request activity 与已保存产物预览行为已经在已发货 runtime 与命令表面上一起收口。
- **改动前**：`预览图形` 不是 Mermaid / Canvas / HTML 已保存产物的可靠兜底入口。
- **改动后**：受支持的已保存图形源文件可直接预览，即使 Obsidian 原生渲染报错，也不必为了查看图形再绕回生成链路。
