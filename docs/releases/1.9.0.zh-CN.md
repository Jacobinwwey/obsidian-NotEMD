# Notemd v1.9.0

## 亮点

- **这是一份完整的 `1.8.9 -> 1.9.0` 累积发布汇总**：线上 `1.9.0` release notes 现已覆盖整个已交付区间，而不再只描述最后一段 advanced batch 增量。
- **Notemd 现在具备明确收敛的 CLI / public-surface 架构**：基于 registry 的 operation 元数据、类型化 capability/contract 导出、脱敏 provider profile 导出，以及 repo-local maintainer helper，共同把自动化边界定义清楚，而不是过度宣称“全面 public CLI”。
- **图形生成与预览已经从零散实验态收口成可验证的平台切片**：spec-first 多目标生成、已保存 artifact 真值预览、history、导出链路与语义核验 helper 已组合落地。
- **Stage C 的知识工作流明显变强**：本地知识库检索、章节拆分契约、高级 batch 文件选择档案，以及放开输入文件限制，都拿到了更安全的运行时契约与更好的可见性。

## 主要升级

- 扩展了 provider/runtime 支持与稳定性，包括更广的中国可用 provider 预设、长请求诊断、streaming 感知的 fallback 链路，以及更防御式的连接/重试逻辑。
- 完成了大规模 i18n 收口：覆盖 settings、notice、workflow surface、sidebar action、preview chrome、welcome/onboarding 文案，以及整套多语言 README。
- 落地了 registry-backed 的 CLI 抽取层：operation registry、capability manifest 导出、invocation contract 导出、收敛的 public-safe surface 导出、provider/config profile 命令，以及 repo-local maintainer invoke/help bridge。
- 补齐了图形平台能力：spec-first Mermaid 生成、JSON Canvas 与 Vega-Lite 目标、renderer/cache/preview 管线、已保存 artifact 重开预览、preview history、统一后的可见操作文案，以及挂在 `diagram.generate` 之下的 typed follow-through 结果层。
- 增加了维护者级语义与打包核验：render-host bundle audit、`npm run verify:diagram-semantics`、从 build/release 真值推导出的 packaging-contract 检查、numeric tag 约束，以及 chronicle/release workflow 硬化。
- 扩展了笔记处理与知识工作流：本地知识库检索已接入批量标题生成、研究与总结、图形生成；章节拆分拿到了确定性的 TOC artifact 与 guarded rerun 语义；batch 进度持久化与响应缓存也已落地。
- 交付了高级 batch 文件夹选择与放开输入控制：可命名 file-selection profiles、regex/glob 规则、`relativePath` / `basename` 目标、子文件夹范围控制、preset chips、单次覆盖、已选文件预览，以及仅对 allowlist 任务开放的 PDF 提取式 relaxed-input reader。

## 修复与鲁棒性

- 统一了预览真值边界：手动 `预览图形` 现在会重新打开已保存 artifact 源，而不会与生成后首次预览发生漂移。
- 加强了 Mermaid 兼容性：包括 parse-error retry、pipe label 转义、best-effort 保存行为、显式 intent routing，以及更安全的 legacy/fixer 拆分边界。
- 通过共享 host-adapter 与 operation contract，减少了命令面板、sidebar、folder-task host、maintainer helper 与 path-based operation 之间的漂移。
- 将 backup 分支中的有界切片恢复回当前 `main`，重新审计 packaging 真值，保护本地 `.trellis/` 状态，并在恢复后保持 release/docs 真值同步。
- 强化了自动化卫生：双语检入式 release notes、必需 release assets、纯数字 release tag、repo-saga 串行锁、贡献者归一化，以及仅扫描 tracked files 的文档检查约束。

## 对比验证摘要

- **改动前**：已发布的 `1.9.0` notes 只覆盖最后一段 advanced batch 输入切片，明显低估了这个版本实际交付的 CLI、图形、检索、打包与运行时硬化范围。
- **改动后**：仓库内与 GitHub 上的 `1.9.0` notes 现已覆盖完整 `1.8.9..1.9.0` 累积区间，同时 release helper 的 repair path 也会基于仓库中的双语 notes 回写已存在 release 的 body。
- **已交付 `1.9.0` 产物集的功能核验**：此前已经完成 repo-saga 串行刷新、`npm run build`、`npm test -- --runInBand`、`npm run audit:i18n-ui`、`npm run audit:render-host`、`obsidian help`、`obsidian-cli help`、repo-local `obsidian build`、repo-local `obsidian test`、`obsidian-cli doctor` 与 `git diff --check`。
- **本次补修 pass** 额外复核了 release-note repair 流程、GitHub release body，以及同步 release 文档后的 clean git state。
