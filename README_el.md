## Γρήγορη εκκίνηση

1.  **Εγκατάσταση και ενεργοποίηση**: αποκτήστε το πρόσθετο από το Obsidian Marketplace.
2.  **Διαμόρφωση LLM**: μεταβείτε στο `Settings -> Notemd`, επιλέξτε τον πάροχο LLM που θέλετε να χρησιμοποιήσετε (όπως OpenAI ή κάποιον τοπικό, όπως Ollama) και εισαγάγετε το API κλειδί ή το URL σας.
3.  **Άνοιγμα πλευρικής στήλης**: κάντε κλικ στο εικονίδιο του ραβδιού Notemd στην αριστερή κορδέλα για να ανοίξετε την πλευρική στήλη.
4.  **Επεξεργασία σημείωσης**: ανοίξτε οποιαδήποτε σημείωση και κάντε κλικ στο **"Process File (Add Links)"** στην πλευρική στήλη για να προστεθούν αυτόματα `[[wiki-links]]` σε βασικές έννοιες.
5.  **Εκτέλεση γρήγορης ροής εργασίας**: χρησιμοποιήστε το προεπιλεγμένο κουμπί **"One-Click Extract"** για να αλυσιδώσετε επεξεργασία, μαζική παραγωγή και καθαρισμό Mermaid από ένα μόνο σημείο.

Αυτό ήταν. Εξερευνήστε τις ρυθμίσεις για να ξεκλειδώσετε περισσότερες δυνατότητες, όπως έρευνα ιστού, μετάφραση και δημιουργία περιεχομένου.

## Υποστήριξη γλωσσών

### Συμβόλαιο γλωσσικής συμπεριφοράς

| Πτυχή | Εμβέλεια | Προεπιλογή | Σημειώσεις |
|---|---|---|---|
| `UI Locale` | Μόνο το κείμενο του UI του πρόσθετου (ρυθμίσεις, πλευρική στήλη, ειδοποιήσεις, διάλογοι) | `auto` | Ακολουθεί τη γλώσσα του Obsidian. Οι τρέχοντες κατάλογοι UI είναι `en`, `ar`, `de`, `es`, `fa`, `fr`, `id`, `it`, `ja`, `ko`, `nl`, `pl`, `pt`, `pt-BR`, `ru`, `th`, `tr`, `uk`, `vi`, `zh-CN`, `zh-TW`. |
| `Task Output Language` | Έξοδος εργασιών που παράγεται από LLM (σύνδεσμοι, περιλήψεις, παραγωγή, εξαγωγή, γλώσσα στόχος μετάφρασης) | `en` | Μπορεί να είναι καθολική ή ανά εργασία όταν το `Use different languages for tasks` είναι ενεργοποιημένο. |
| `Disable auto translation` | Οι εργασίες εκτός `Translate` διατηρούν το πλαίσιο της γλώσσας προέλευσης | `false` | Οι ρητές εργασίες `Translate` εξακολουθούν να επιβάλλουν τη ρυθμισμένη γλώσσα στόχο. |
| Εφεδρική τοπικοποίηση | Επίλυση ελλιπών κλειδιών UI | locale -> `en` | Διατηρεί σταθερό το UI όταν ορισμένα κλειδιά δεν έχουν μεταφραστεί ακόμη. |

- Η επίσημη τεκμηρίωση διατηρείται στα Αγγλικά και τα Απλοποιημένα Κινεζικά, με πλήρη υποστήριξη για περισσότερες από 30 γλώσσες.
- Όλες οι υποστηριζόμενες γλώσσες συνδέονται στην κεφαλίδα παραπάνω.
- Περισσότερες λεπτομέρειες και οδηγίες συνεισφοράς παρακολουθούνται στο [Κέντρο Γλωσσών](./docs/i18n/README.md).

## Δυνατότητες

### Επεξεργασία εγγράφων με τεχνητή νοημοσύνη
- **Υποστήριξη Multi-LLM**: συνδεθείτε με διάφορους παρόχους LLM cloud και τοπικούς (δείτε [Υποστηριζόμενοι πάροχοι LLM](#υποστηριζόμενοι-πάροχοι-llm)).
- **Έξυπνος τεμαχισμός**: χωρίζει αυτόματα τα μεγάλα έγγραφα σε διαχειρίσιμα τμήματα με βάση τον αριθμό λέξεων για επεξεργασία.
- **Διατήρηση περιεχομένου**: στοχεύει στη διατήρηση της αρχικής μορφοποίησης ενώ προσθέτει δομή και συνδέσμους.
- **Παρακολούθηση προόδου**: ενημερώσεις σε πραγματικό χρόνο μέσω της πλευρικής στήλης Notemd ή ενός παραθύρου προόδου.
- **Ακυρώσιμες λειτουργίες**: ακυρώστε οποιαδήποτε εργασία επεξεργασίας (μεμονωμένη ή μαζική) που ξεκίνησε από την πλευρική στήλη μέσω του ειδικού κουμπιού ακύρωσης. Οι λειτουργίες από την παλέτα εντολών χρησιμοποιούν ένα modal που μπορεί επίσης να ακυρωθεί.
- **Διαμόρφωση πολλαπλών μοντέλων**: χρησιμοποιήστε διαφορετικούς παρόχους LLM *και* συγκεκριμένα μοντέλα για διαφορετικές εργασίες (Add Links, Research, Generate Title, Translate) ή χρησιμοποιήστε έναν πάροχο για όλα.
- **Stable API Calls (Retry Logic)**: ενεργοποιήστε προαιρετικά αυτόματες επαναλήψεις για αποτυχημένες κλήσεις API LLM με ρυθμιζόμενο διάστημα και όρια προσπαθειών.
- **Ανθεκτικά τεστ σύνδεσης παρόχων**: αν το πρώτο τεστ παρόχου συναντήσει μια παροδική αποσύνδεση δικτύου, το Notemd περνά τώρα στην ακολουθία σταθερής επανάληψης πριν αποτύχει, καλύπτοντας μεταφορές OpenAI-compatible, Anthropic, Google, Azure OpenAI και Ollama.
- **Εναλλακτική μεταφορά ανάλογα με το περιβάλλον εκτέλεσης**: όταν ένα αίτημα μεγάλης διάρκειας προς πάροχο διακόπτεται από το `requestUrl` με παροδικά σφάλματα δικτύου όπως `ERR_CONNECTION_CLOSED`, το Notemd επαναλαμβάνει τώρα την ίδια προσπάθεια μέσω μεταφοράς εφεδρείας ειδικής για το περιβάλλον πριν εισέλθει στον διαμορφωμένο βρόχο επαναλήψεων: οι desktop build εκδόσεις χρησιμοποιούν Node `http/https`, ενώ τα μη desktop περιβάλλοντα χρησιμοποιούν browser `fetch`. Αυτό μειώνει τις ψευδείς αποτυχίες σε αργά gateway και reverse proxy.
- **Σκλήρυνση σταθερής αλυσίδας μεγάλων αιτημάτων για OpenAI-compatible**: στη σταθερή λειτουργία, οι κλήσεις OpenAI-compatible χρησιμοποιούν τώρα μια ρητή σειρά 3 σταδίων για κάθε προσπάθεια: πρωτεύουσα άμεση streaming μεταφορά, στη συνέχεια άμεση μη streaming μεταφορά, και τέλος εφεδρεία `requestUrl` (που μπορεί να αναβαθμιστεί σε stream parsing όταν χρειάζεται). Αυτό μειώνει τα ψευδώς αρνητικά όταν οι πάροχοι ολοκληρώνουν buffered απαντήσεις αλλά οι σωλήνες streaming είναι ασταθείς.
- **Εναλλακτική ροή με επίγνωση πρωτοκόλλου σε όλα τα LLM API**: οι εναλλακτικές προσπάθειες μεγάλης διάρκειας αναβαθμίζονται τώρα σε stream parsing με επίγνωση πρωτοκόλλου σε κάθε ενσωματωμένη διαδρομή LLM, όχι μόνο στα OpenAI-compatible endpoint. Το Notemd χειρίζεται τώρα OpenAI/Azure-style SSE, Anthropic Messages streaming, Google Gemini SSE responses και Ollama NDJSON streams τόσο σε desktop `http/https` όσο και σε non-desktop `fetch`, και τα υπόλοιπα direct OpenAI-style provider entrypoints επαναχρησιμοποιούν την ίδια κοινή fallback διαδρομή.
- **Έτοιμα presets παρόχων για Κίνα**: τα ενσωματωμένα presets καλύπτουν πλέον τα `Qwen`, `Qwen Code`, `Doubao`, `Moonshot`, `GLM`, `Z AI`, `MiniMax`, `Huawei Cloud MaaS`, `Baidu Qianfan` και `SiliconFlow`, πέρα από τους υπάρχοντες παγκόσμιους και τοπικούς παρόχους.
- **Αξιόπιστη μαζική επεξεργασία**: η βελτιωμένη λογική ταυτόχρονης επεξεργασίας με **κλιμακωτές κλήσεις API** βοηθά στην αποφυγή σφαλμάτων rate-limit και διασφαλίζει σταθερή απόδοση σε μεγάλες μαζικές εργασίες. Η νέα υλοποίηση εξασφαλίζει ότι οι εργασίες ξεκινούν σε διαφορετικά διαστήματα και όχι όλες ταυτόχρονα.
- **Ακριβής αναφορά προόδου**: διορθώθηκε ένα σφάλμα όπου η μπάρα προόδου μπορούσε να κολλήσει, εξασφαλίζοντας ότι το UI αντικατοπτρίζει πάντα την πραγματική κατάσταση της λειτουργίας.
- **Ανθεκτική παράλληλη μαζική επεξεργασία**: επιλύθηκε ένα ζήτημα όπου οι παράλληλες μαζικές λειτουργίες σταματούσαν πρόωρα, εξασφαλίζοντας ότι όλα τα αρχεία επεξεργάζονται αξιόπιστα και αποδοτικά.
- **Ακρίβεια μπάρας προόδου**: διορθώθηκε ένα σφάλμα όπου η μπάρα προόδου της εντολής "Create Wiki-Link & Generate Note" κολλούσε στο 95%, εξασφαλίζοντας ότι πλέον δείχνει σωστά 100% κατά την ολοκλήρωση.
- **Βελτιωμένη αποσφαλμάτωση API**: το "API Error Debugging Mode" καταγράφει πλέον ολόκληρα σώματα απόκρισης από παρόχους LLM και υπηρεσίες αναζήτησης (Tavily/DuckDuckGo), και επίσης κρατά χρονογραμμή μεταφοράς ανά προσπάθεια με καθαρισμένα request URLs, διάρκεια, response headers, μερικά response bodies, μερικώς αναλυμένο περιεχόμενο stream και ίχνη στοίβας για καλύτερη αντιμετώπιση προβλημάτων σε fallback διαδρομές OpenAI-compatible, Anthropic, Google, Azure OpenAI και Ollama.
- **Πίνακας Developer Mode**: οι ρυθμίσεις περιλαμβάνουν πλέον έναν αφιερωμένο πίνακα διαγνωστικών μόνο για προγραμματιστές, που παραμένει κρυφός έως ότου ενεργοποιηθεί το "Developer mode". Υποστηρίζει επιλογή διαδρομών διαγνωστικής κλήσης και επαναλαμβανόμενες δοκιμές σταθερότητας για τον επιλεγμένο τρόπο.
- **Επανασχεδιασμένη πλευρική στήλη**: οι ενσωματωμένες ενέργειες ομαδοποιούνται σε πιο εστιασμένες ενότητες με σαφέστερες ετικέτες, ζωντανή κατάσταση, ακυρώσιμη πρόοδο και αντιγράψιμα logs, ώστε να μειωθεί η ακαταστασία της πλευρικής στήλης. Το footer προόδου/log παραμένει ορατό ακόμη και όταν όλες οι ενότητες είναι ανοικτές, ενώ η κατάσταση ετοιμότητας χρησιμοποιεί πιο σαφή standby progress track.
- **Βελτιώσεις αλληλεπίδρασης και αναγνωσιμότητας της πλευρικής στήλης**: τα κουμπιά της πλευρικής στήλης παρέχουν πλέον καθαρότερη ανάδραση σε hover/press/focus, και τα πολύχρωμα κουμπιά CTA, συμπεριλαμβανομένων των `One-Click Extract` και `Batch generate from titles`, χρησιμοποιούν ισχυρότερη αντίθεση κειμένου για καλύτερη αναγνωσιμότητα σε διαφορετικά θέματα.
- **Χαρτογράφηση CTA μόνο για ενέργειες ενός αρχείου**: η πολύχρωμη CTA εμφάνιση διατηρείται πλέον αποκλειστικά για ενέργειες πάνω σε ένα μόνο αρχείο. Οι μαζικές/σε επίπεδο φακέλου ενέργειες και οι μικτές ροές εργασίας χρησιμοποιούν μη CTA εμφάνιση για να μειώνονται τα λάθη από λανθασμένη εκτίμηση του εύρους δράσης.
- **Προσαρμοσμένες one-click ροές εργασίας**: μετατρέψτε τα ενσωματωμένα εργαλεία της πλευρικής στήλης σε επαναχρησιμοποιήσιμα προσαρμοσμένα κουμπιά με ονόματα ορισμένα από τον χρήστη και συναρμολογημένες αλυσίδες ενεργειών. Μια προεπιλεγμένη ροή `One-Click Extract` περιλαμβάνεται από την αρχή.

### Ενίσχυση γραφήματος γνώσης
- **Αυτόματη δημιουργία wiki-links**: εντοπίζει και προσθέτει `[[wiki-links]]` σε βασικές έννοιες μέσα στις επεξεργασμένες σημειώσεις σας με βάση την έξοδο του LLM.
- **Δημιουργία concept notes (προαιρετική και παραμετροποιήσιμη)**: δημιουργεί αυτόματα νέες σημειώσεις για έννοιες που ανακαλύπτονται σε συγκεκριμένο φάκελο του vault.
- **Προσαρμόσιμες διαδρομές εξόδου**: διαμορφώστε ξεχωριστές σχετικές διαδρομές μέσα στο vault για την αποθήκευση επεξεργασμένων αρχείων και νεοδημιουργημένων concept notes.
- **Προσαρμόσιμα ονόματα αρχείων εξόδου (Add Links)**: προαιρετικά **αντικαθιστά το αρχικό αρχείο** ή χρησιμοποιεί προσαρμοσμένο suffix/string αντικατάστασης αντί του προεπιλεγμένου `_processed.md` κατά την επεξεργασία αρχείων για συνδέσμους.
- **Συντήρηση ακεραιότητας συνδέσμων**: βασικός χειρισμός για ενημέρωση συνδέσμων όταν οι σημειώσεις μετονομάζονται ή διαγράφονται μέσα στο vault.
- **Καθαρή εξαγωγή εννοιών**: εξάγει έννοιες και δημιουργεί αντίστοιχα concept notes χωρίς να τροποποιεί το αρχικό έγγραφο. Είναι ιδανικό για γρήγορη τροφοδότηση μιας βάσης γνώσης από υπάρχοντα έγγραφα χωρίς αλλοίωσή τους. Η δυνατότητα αυτή περιλαμβάνει παραμετροποιήσιμες επιλογές για ελάχιστα concept notes και backlinks.

### Μετάφραση

- **Μετάφραση με τεχνητή νοημοσύνη**:
    - Μεταφράστε το περιεχόμενο των σημειώσεων χρησιμοποιώντας το διαμορφωμένο LLM.
    - **Υποστήριξη μεγάλων αρχείων**: χωρίζει αυτόματα μεγάλα αρχεία σε μικρότερα τμήματα βάσει της ρύθμισης `Chunk word count` πριν τα στείλει στο LLM. Τα μεταφρασμένα τμήματα επανενώνονται έπειτα σε ένα ενιαίο έγγραφο χωρίς κενά.
    - Υποστηρίζει μετάφραση μεταξύ πολλών γλωσσών.
    - Η γλώσσα στόχος μπορεί να προσαρμοστεί από τις ρυθμίσεις ή από το UI.
    - Ανοίγει αυτόματα το μεταφρασμένο κείμενο στη δεξιά πλευρά του αρχικού κειμένου για εύκολη ανάγνωση.
- **Μαζική μετάφραση**:
    - Μεταφράζει όλα τα αρχεία μέσα σε έναν επιλεγμένο φάκελο.
    - Υποστηρίζει παράλληλη επεξεργασία όταν είναι ενεργό το "Enable Batch Parallelism".
    - Χρησιμοποιεί προσαρμοσμένα prompts για μετάφραση αν έχουν ρυθμιστεί.
	- Προσθέτει επιλογή "Batch translate this folder" στο μενού περιβάλλοντος του file explorer.
- **Disable auto translation**: όταν αυτή η επιλογή είναι ενεργοποιημένη, οι εργασίες εκτός `Translate` δεν αναγκάζουν πλέον τα αποτελέσματα σε συγκεκριμένη γλώσσα, διατηρώντας το αρχικό γλωσσικό πλαίσιο. Η ρητή εργασία "Translate" εξακολουθεί να εκτελεί μετάφραση όπως έχει ρυθμιστεί.

### Έρευνα ιστού και δημιουργία περιεχομένου
- **Έρευνα ιστού και σύνοψη**:
    - Εκτελεί αναζητήσεις ιστού χρησιμοποιώντας Tavily (απαιτεί API key) ή DuckDuckGo (πειραματικό).
    - **Βελτιωμένη ανθεκτικότητα αναζήτησης**: η αναζήτηση DuckDuckGo διαθέτει πλέον βελτιωμένη λογική ανάλυσης (DOMParser με Regex fallback) για να διαχειρίζεται αλλαγές διάταξης και να εξασφαλίζει αξιόπιστα αποτελέσματα.
    - Συνοψίζει τα αποτελέσματα της αναζήτησης χρησιμοποιώντας το διαμορφωμένο LLM.
    - Η γλώσσα εξόδου της σύνοψης μπορεί να προσαρμοστεί από τις ρυθμίσεις.
    - Προσθέτει τις συνόψεις στην τρέχουσα σημείωση.
    - Ρυθμιζόμενο όριο token για το ερευνητικό περιεχόμενο που αποστέλλεται στο LLM.
- **Δημιουργία περιεχομένου από τίτλο**:

    - **Προαιρετική έρευνα**: ρυθμίστε αν θα εκτελείται έρευνα ιστού (με τον επιλεγμένο πάροχο) ώστε να παρέχεται συμφραζόμενο στη δημιουργία περιεχομένου.

Τα επιτυχώς επεξεργασμένα αρχεία μετακινούνται σε **παραμετροποιήσιμο "complete" υποφάκελο** (π.χ. `[foldername]_complete` ή προσαρμοσμένο όνομα) για να αποφεύγεται η επανεπεξεργασία.
- **Σύζευξη Mermaid Auto-Fix**: όταν η αυτόματη επιδιόρθωση Mermaid είναι ενεργοποιημένη, οι ροές εργασίας που σχετίζονται με Mermaid επιδιορθώνουν αυτόματα τα παραγόμενα αρχεία ή τους φακέλους εξόδου μετά την επεξεργασία.     - **Safe Processing**: διασφαλίζει ότι οι διορθώσεις σύνταξης εφαρμόζονται αποκλειστικά σε Mermaid code blocks, αποτρέποντας τυχαία τροποποίηση πινάκων Markdown ή άλλου περιεχομένου. Περιλαμβάνει ισχυρές ασφαλιστικές δικλίδες για προστασία της σύνταξης πινάκων (π.χ. `| :--- |`) από επιθετικές debug διορθώσεις.
    - **Deep Debug Mode**: αν τα σφάλματα παραμένουν μετά την αρχική διόρθωση, ενεργοποιείται ένα προηγμένο deep debug mode. Αυτή η λειτουργία χειρίζεται σύνθετες ακραίες περιπτώσεις, όπως:
        - **Comment Integration**: ενσωματώνει αυτόματα τα τελικά σχόλια (που ξεκινούν με `%`) στην ετικέτα της ακμής (π.χ. `A -- Label --> B; % Comment` γίνεται `A -- "Label(Comment)" --> B;`).
        - **Malformed Arrows**: διορθώνει βέλη που έχουν απορροφηθεί μέσα σε εισαγωγικά (π.χ. `A -- "Label -->" B` γίνεται `A -- "Label" --> B`).
        - **Inline Subgraphs**: μετατρέπει inline ετικέτες υπογραφημάτων σε ετικέτες ακμών.
        - **Reverse Arrow Fix**: διορθώνει τα μη τυπικά βέλη `X <-- Y` σε `Y --> X`.
        - **Direction Keyword Fix**: διασφαλίζει ότι η λέξη-κλειδί `direction` είναι πεζή μέσα στα subgraphs (π.χ. `Direction TB` -> `direction TB`).
        - **Comment Conversion**: μετατρέπει τα σχόλια `//` σε ετικέτες ακμών (π.χ. `A --> B; // Comment` -> `A -- "Comment" --> B;`).
        - **Duplicate Label Fix**: απλοποιεί επαναλαμβανόμενες ετικέτες σε αγκύλες (π.χ. `Node["Label"]["Label"]` -> `Node["Label"]`).
        - **Invalid Arrow Fix**: μετατρέπει την άκυρη σύνταξη βέλους `--|>` στο τυπικό `-->`.
        - **Robust Label & Note Handling**: βελτιωμένος χειρισμός για ετικέτες με ειδικούς χαρακτήρες (όπως `/`) και καλύτερη υποστήριξη για προσαρμοσμένη σύνταξη σημειώσεων (`note for ...`), διασφαλίζοντας ότι τα κατάλοιπα όπως περιττές αγκύλες αφαιρούνται καθαρά.
        - **Advanced Fix Mode**: περιλαμβάνει ισχυρές διορθώσεις για ετικέτες κόμβων χωρίς εισαγωγικά που περιέχουν κενά, ειδικούς χαρακτήρες ή εμφωλευμένες αγκύλες (π.χ. `Node[Label [Text]]` -> `Node["Label [Text]"]`), διασφαλίζοντας συμβατότητα με σύνθετα διαγράμματα όπως μονοπάτια Stellar Evolution. Επίσης διορθώνει κακοσχηματισμένες ετικέτες ακμών (π.χ. `--["Label["-->` σε `-- "Label" -->`). Επιπλέον μετατρέπει inline σχόλια (`Consensus --> Adaptive; # Some advanced consensus` σε `Consensus -- "Some advanced consensus" --> Adaptive`) και διορθώνει ατελή εισαγωγικά στο τέλος γραμμών (το `;"` στο τέλος αντικαθίσταται με `"]`).
                        - **Note Conversion**: μετατρέπει αυτόματα τα `note right/left of` και τα αυτόνομα `note :` σχόλια σε τυπικούς Mermaid ορισμούς κόμβων και συνδέσεις (π.χ. `note right of A: text` γίνεται `NoteA["Note: text"]` συνδεδεμένο με το `A`), αποτρέποντας συντακτικά σφάλματα και βελτιώνοντας τη διάταξη. Υποστηρίζει πλέον τόσο συνδέσεις με βέλος (`-->`) όσο και συμπαγείς συνδέσεις (`---`).
                        - **Extended Note Support**: μετατρέπει αυτόματα τα `note for Node "Content"` και `note of Node "Content"` σε τυπικούς συνδεδεμένους note κόμβους (π.χ. `NoteNode[" Content"]` συνδεδεμένο με `Node`), εξασφαλίζοντας συμβατότητα με συντακτικό που έχει επεκταθεί από τον χρήστη.
                        - **Enhanced Note Correction**: μετονομάζει αυτόματα note κόμβους με σειριακή αρίθμηση (π.χ. `Note1`, `Note2`) ώστε να αποτρέπεται η σύγχυση ψευδωνύμων όταν υπάρχουν πολλές σημειώσεις.                - **Parallelogram/Shape Fix**: διορθώνει κακοσχηματισμένα σχήματα κόμβων όπως `[/["Label["/]` στο τυπικό `["Label"]`, διασφαλίζοντας συμβατότητα με το παραγόμενο περιεχόμενο.
                        - **Standardize Pipe Labels**: διορθώνει και τυποποιεί αυτόματα ετικέτες ακμών που περιέχουν pipes, διασφαλίζοντας ότι περιβάλλονται σωστά από εισαγωγικά (π.χ. `-->|Text|` γίνεται `-->|"Text"|` και `-->|Math|^2|` γίνεται `-->|"Math|^2"|`).
        - **Misplaced Pipe Fix**: διορθώνει ετικέτες ακμών που εμφανίζονται λανθασμένα πριν από το βέλος (π.χ. `>|"Label"| A --> B` γίνεται `A -->|"Label"| B`).
                - **Merge Double Labels**: ανιχνεύει και συγχωνεύει σύνθετες διπλές ετικέτες στην ίδια ακμή (π.χ. `A -- Label1 -- Label2 --> B` ή `A -- Label1 -- Label2 --- B`) σε μία καθαρή ετικέτα με αλλαγές γραμμής (`A -- "Label1<br>Label2" --> B`).
                        - **Unquoted Label Fix**: βάζει αυτόματα σε εισαγωγικά ετικέτες κόμβων που περιέχουν δυνητικά προβληματικούς χαρακτήρες (π.χ. εισαγωγικά, σύμβολα ίσον, μαθηματικούς τελεστές) αλλά δεν έχουν εξωτερικά εισαγωγικά (π.χ. `Plot[Plot "A"]` γίνεται `Plot["Plot "A""]`), αποτρέποντας σφάλματα απόδοσης.
                        - **Intermediate Node Fix**: χωρίζει ακμές που περιέχουν ορισμό ενδιάμεσου κόμβου σε δύο ξεχωριστές ακμές (π.χ. `A -- B[...] --> C` γίνεται `A --> B[...]` και `B[...] --> C`), εξασφαλίζοντας έγκυρη σύνταξη Mermaid.
                        - **Concatenated Label Fix**: διορθώνει με ανθεκτικό τρόπο ορισμούς κόμβων όπου το ID είναι συγκολλημένο με την ετικέτα (π.χ. `SubdivideSubdivide...` γίνεται `Subdivide["Subdivide..."]`), ακόμη και όταν προηγούνται pipe labels ή όταν η επανάληψη δεν είναι ακριβής, μέσω επικύρωσης με γνωστά node IDs.
                        - **Extract Specific Original Text**:    - Ορίστε μια λίστα ερωτήσεων στις ρυθμίσεις.
                    - Εξάγει κατά λέξη τμήματα κειμένου από την ενεργή σημείωση που απαντούν σε αυτές τις ερωτήσεις.
                    - **Merged Query Mode**: επιλογή επεξεργασίας όλων των ερωτήσεων σε μία μόνο κλήση API για μεγαλύτερη αποδοτικότητα.
                    - **Translation**: επιλογή να συμπεριληφθούν μεταφράσεις του εξαγμένου κειμένου στην έξοδο.
                    - **Custom Output**: ρυθμιζόμενη διαδρομή αποθήκευσης και suffix ονόματος αρχείου για το εξαγμένο αρχείο κειμένου.- **LLM Connection Test**: επαληθεύει τις ρυθμίσεις API για τον ενεργό πάροχο.

### Από το Obsidian Marketplace (προτείνεται)
1. Ανοίξτε στο Obsidian το **Settings** -> **Community plugins**.
2. Βεβαιωθείτε ότι το "Restricted mode" είναι **απενεργοποιημένο**.
3. Κάντε κλικ στο **Browse** για τα community plugins και αναζητήστε το "Notemd".
4. Κάντε κλικ στο **Install**.
5. Μόλις εγκατασταθεί, κάντε κλικ στο **Enable**.

### Χειροκίνητη εγκατάσταση
1. Κατεβάστε τα πιο πρόσφατα release assets από τη [σελίδα GitHub Releases](https://github.com/Jacobinwwey/obsidian-NotEMD/releases). Κάθε release περιλαμβάνει επίσης το `README.md` για αναφορά μέσα στο πακέτο, αλλά για χειροκίνητη εγκατάσταση απαιτούνται μόνο τα `main.js`, `styles.css` και `manifest.json`.
2. Μεταβείτε στον φάκελο ρυθμίσεων του Obsidian vault σας: `<YourVault>/.obsidian/plugins/`.
3. Δημιουργήστε έναν νέο φάκελο με όνομα `notemd`.
4. Αντιγράψτε τα `main.js`, `styles.css` και `manifest.json` μέσα στον φάκελο `notemd`.
5. Επανεκκινήστε το Obsidian.
6. Μεταβείτε στο **Settings** -> **Community plugins** και ενεργοποιήστε το "Notemd".

## Διαμόρφωση

Η πρόσβαση στις ρυθμίσεις του πρόσθετου γίνεται μέσω:
**Settings** -> **Community Plugins** -> **Notemd** (κάντε κλικ στο εικονίδιο με το γρανάζι).

### Διαμόρφωση παρόχου LLM
1.  **Active Provider**: επιλέξτε τον πάροχο LLM που θέλετε να χρησιμοποιήσετε από το αναπτυσσόμενο μενού.
2.  **Provider Settings**: διαμορφώστε τις συγκεκριμένες ρυθμίσεις για τον επιλεγμένο πάροχο:
    *   **API Key**: απαιτείται για τους περισσότερους παρόχους cloud (π.χ. OpenAI, Anthropic, DeepSeek, Qwen, Qwen Code, Doubao, Moonshot, GLM, Z AI, MiniMax, Huawei Cloud MaaS, Baidu Qianfan, SiliconFlow, Google, Mistral, Azure OpenAI, OpenRouter, xAI, Groq, Together, Fireworks, Requesty). Δεν απαιτείται για Ollama. Είναι προαιρετικό για LM Studio και για το γενικό preset `OpenAI Compatible` όταν το endpoint σας δέχεται ανώνυμη ή placeholder πρόσβαση.
    *   **Base URL / Endpoint**: το API endpoint της υπηρεσίας. Παρέχονται προεπιλεγμένες τιμές, αλλά ίσως χρειαστεί να το αλλάξετε για τοπικά μοντέλα (LMStudio, Ollama), gateway (OpenRouter, Requesty, OpenAI Compatible) ή συγκεκριμένα Azure deployments. **Απαιτείται για Azure OpenAI.**
    *   **Model**: το συγκεκριμένο όνομα/ID μοντέλου που θα χρησιμοποιηθεί (π.χ. `gpt-4o`, `claude-3-5-sonnet-20240620`, `google/gemini-flash-1.5`, `grok-4`, `moonshotai/kimi-k2-instruct-0905`, `accounts/fireworks/models/kimi-k2p5`, `anthropic/claude-3-7-sonnet-latest`). Βεβαιωθείτε ότι το μοντέλο είναι διαθέσιμο στο endpoint ή στον πάροχό σας.
    *   **Temperature**: ελέγχει την τυχαιότητα της εξόδου του LLM (0 = ντετερμινιστικό, 1 = μέγιστη δημιουργικότητα). Οι χαμηλότερες τιμές (π.χ. 0.2-0.5) είναι συνήθως καλύτερες για δομημένες εργασίες.
    *   **API Version (Azure Only)**: απαιτείται για Azure OpenAI deployments (π.χ. `2024-02-15-preview`).
3.  **Test Connection**: χρησιμοποιήστε το κουμπί "Test Connection" για τον ενεργό πάροχο ώστε να επαληθεύσετε τις ρυθμίσεις σας. Οι OpenAI-compatible πάροχοι χρησιμοποιούν πλέον ελέγχους με επίγνωση παρόχου: endpoint όπως `Qwen`, `Qwen Code`, `Doubao`, `Moonshot`, `GLM`, `Z AI`, `MiniMax`, `Huawei Cloud MaaS`, `Baidu Qianfan`, `SiliconFlow`, `Groq`, `Together`, `Fireworks`, `LMStudio` και `OpenAI Compatible` δοκιμάζουν απευθείας το `chat/completions`, ενώ πάροχοι με αξιόπιστο `/models` endpoint μπορούν ακόμη να χρησιμοποιούν αρχικά λίστα μοντέλων. Αν η πρώτη δοκιμή αποτύχει με παροδική αποσύνδεση δικτύου όπως `ERR_CONNECTION_CLOSED`, το Notemd μεταπίπτει αυτόματα στην ακολουθία σταθερής επανάληψης αντί να αποτύχει αμέσως.
4.  **Manage Provider Configurations**: χρησιμοποιήστε τα κουμπιά "Export Providers" και "Import Providers" για να αποθηκεύσετε/φορτώσετε τις ρυθμίσεις παρόχων LLM σε/από αρχείο `notemd-providers.json` μέσα στον κατάλογο ρυθμίσεων του πρόσθετου. Αυτό επιτρέπει εύκολο backup και κοινή χρήση.
5.  **Preset Coverage**: εκτός από τους αρχικούς παρόχους, το Notemd περιλαμβάνει πλέον προκαθορισμένες εγγραφές για `Qwen`, `Qwen Code`, `Doubao`, `Moonshot`, `GLM`, `Z AI`, `MiniMax`, `Huawei Cloud MaaS`, `Baidu Qianfan`, `SiliconFlow`, `xAI`, `Groq`, `Together`, `Fireworks`, `Requesty`, καθώς και έναν γενικό στόχο `OpenAI Compatible` για LiteLLM, vLLM, Perplexity, Vercel AI Gateway ή προσαρμοσμένα proxy.

### Αρχιτεκτονική γλώσσας (UI Locale vs Task Output Language)

-   Το **UI Locale** ελέγχει μόνο το κείμενο του UI του πρόσθετου (ετικέτες Settings, κουμπιά πλευρικής στήλης, ειδοποιήσεις και διάλογοι). Η προεπιλεγμένη λειτουργία `auto` ακολουθεί την τρέχουσα γλώσσα UI του Obsidian.
-   Το **Task Output Language** ελέγχει την έξοδο που παράγεται από το μοντέλο για εργασίες (σύνδεσμοι, περιλήψεις, παραγωγή τίτλων, σύνοψη Mermaid, εξαγωγή εννοιών, γλώσσα στόχος μετάφρασης).
-   Το **Per-task language mode** επιτρέπει σε κάθε εργασία να επιλύει τη δική της γλώσσα εξόδου από ένα ενιαίο επίπεδο πολιτικής αντί για διάσπαρτες παρακάμψεις ανά module.
-   Το **Disable auto translation** διατηρεί τις εργασίες εκτός `Translate` στο πλαίσιο της γλώσσας προέλευσης, ενώ οι ρητές εργασίες `Translate` εξακολουθούν να επιβάλλουν τη διαμορφωμένη γλώσσα στόχο.
-   Οι διαδρομές παραγωγής που σχετίζονται με Mermaid ακολουθούν την ίδια γλωσσική πολιτική και μπορούν ακόμη να ενεργοποιούν το Mermaid auto-fix όταν είναι ενεργοποιημένο.

### Ρυθμίσεις σταθερών κλήσεων API
-   **Enable Stable API Calls (Retry Logic)**:
    *   **Disabled (Default)**: μία μόνο αποτυχία κλήσης API θα σταματήσει την τρέχουσα εργασία.
    *   **Enabled**: επαναλαμβάνει αυτόματα αποτυχημένες κλήσεις API LLM (χρήσιμο για διαλείπουσες δικτυακές αστοχίες ή rate limits).
    *   **Connection Test Fallback**: ακόμη κι όταν οι κανονικές κλήσεις δεν τρέχουν ήδη σε stable mode, τα τεστ σύνδεσης παρόχων περνούν τώρα στην ίδια ακολουθία επαναλήψεων μετά το πρώτο παροδικό σφάλμα δικτύου.
    *   **Runtime Transport Fallback (Environment-Aware)**: τα αιτήματα εργασιών μεγάλης διάρκειας που απορρίπτονται παροδικά από το `requestUrl` επαναλαμβάνουν τώρα πρώτα την ίδια προσπάθεια μέσω μιας εναλλακτικής μεταφοράς με επίγνωση περιβάλλοντος. Οι desktop build εκδόσεις χρησιμοποιούν Node `http/https`, ενώ τα μη desktop περιβάλλοντα χρησιμοποιούν browser `fetch`. Οι εναλλακτικές αυτές προσπάθειες χρησιμοποιούν πλέον stream parsing με επίγνωση πρωτοκόλλου σε όλες τις ενσωματωμένες LLM διαδρομές, καλύπτοντας OpenAI-compatible SSE, Azure OpenAI SSE, Anthropic Messages SSE, Google Gemini SSE και Ollama NDJSON output, ώστε τα αργά gateway να μπορούν να επιστρέφουν νωρίτερα τμήματα σώματος. Τα υπόλοιπα direct OpenAI-style provider entrypoints επαναχρησιμοποιούν την ίδια κοινή fallback διαδρομή.
    *   **OpenAI-Compatible Stable Order**: σε stable mode, κάθε OpenAI-compatible προσπάθεια ακολουθεί πλέον τη σειρά `direct streaming -> direct non-stream -> requestUrl (with streamed fallback when needed)` πριν μετρήσει ως αποτυχημένη προσπάθεια. Αυτό αποτρέπει υπερβολικά επιθετικές αποτυχίες όταν μόνο ένας τρόπος μεταφοράς είναι ασταθής.
-   **Retry Interval (seconds)**: (ορατό μόνο όταν είναι ενεργοποιημένο) χρόνος αναμονής μεταξύ προσπαθειών επανάληψης (1-300 δευτερόλεπτα). Προεπιλογή: 5.
-   **Maximum Retries**: (ορατό μόνο όταν είναι ενεργοποιημένο) μέγιστος αριθμός προσπαθειών επανάληψης (0-10). Προεπιλογή: 3.
-   **API Error Debugging Mode**:
    *   **Disabled (Default)**: χρησιμοποιεί τυπική, συνοπτική αναφορά σφαλμάτων.
    *   **Enabled**: ενεργοποιεί λεπτομερή καταγραφή σφαλμάτων (παρόμοια με το verbose output του DeepSeek) για όλους τους παρόχους και όλες τις εργασίες (συμπεριλαμβανομένων των Translate, Search και Connection Tests). Αυτό περιλαμβάνει HTTP status codes, raw response text, request transport timelines, καθαρισμένα request URLs και headers, διάρκειες προσπαθειών, response headers, partial response bodies, parsed partial stream output και ίχνη στοίβας, κάτι κρίσιμο για αντιμετώπιση προβλημάτων σύνδεσης API και reset σε upstream gateway.
-   **Developer Mode**:
    *   **Disabled (Default)**: αποκρύπτει όλα τα διαγνωστικά στοιχεία μόνο για προγραμματιστές από τους κανονικούς χρήστες.
    *   **Enabled**: εμφανίζει αφιερωμένο πίνακα διαγνωστικών προγραμματιστή στις Settings.
-   **Developer Provider Diagnostic (Long Request)**:
    *   **Diagnostic Call Mode**: επιλέξτε τη διαδρομή εκτέλεσης ανά δοκιμή. Οι OpenAI-compatible πάροχοι υποστηρίζουν επιπλέον forced modes (`direct streaming`, `direct buffered`, `requestUrl-only`) πέρα από τα runtime modes.
    *   **Run Diagnostic**: εκτελεί μία δοκιμή μεγάλου αιτήματος με τον επιλεγμένο τρόπο κλήσης και γράφει `Notemd_Provider_Diagnostic_*.txt` στη ρίζα του vault.
    *   **Run Stability Test**: επαναλαμβάνει τη δοκιμή για διαμορφώσιμο αριθμό εκτελέσεων (1-10) χρησιμοποιώντας τον επιλεγμένο τρόπο κλήσης και αποθηκεύει συγκεντρωτική αναφορά σταθερότητας.
    *   **Diagnostic Timeout**: διαμορφώσιμο timeout ανά εκτέλεση (15-3600 δευτερόλεπτα).
    *   **Why Use It**: ταχύτερο από τη χειροκίνητη αναπαραγωγή όταν ένας πάροχος περνά το "Test connection" αλλά αποτυγχάνει σε πραγματικές μακρόχρονες εργασίες (για παράδειγμα, μετάφραση σε αργά gateway).

-   **Use Custom Prompt for [Task Name]**: (ορατό μόνο όταν η παραπάνω επιλογή είναι ενεργοποιημένη)
    *   Για κάθε υποστηριζόμενη εργασία ("Add Links", "Generate from Title", "Research & Summarize", "Extract Concepts"), μπορείτε να ενεργοποιείτε ή να απενεργοποιείτε ξεχωριστά το προσαρμοσμένο prompt σας.
    *   **Disabled**: αυτή η συγκεκριμένη εργασία θα χρησιμοποιεί το προεπιλεγμένο prompt.
    *   **Enabled**: αυτή η εργασία θα χρησιμοποιεί το κείμενο που παρέχετε στο αντίστοιχο πεδίο "Custom Prompt" παρακάτω.

-   **Custom Prompt Text Area**: (ορατό μόνο όταν το προσαρμοσμένο prompt μιας εργασίας είναι ενεργοποιημένο)
    *   **Default Prompt Display**: για αναφορά, το πρόσθετο εμφανίζει το προεπιλεγμένο prompt που θα χρησιμοποιούσε κανονικά για την εργασία. Μπορείτε να χρησιμοποιήσετε το κουμπί **"Copy Default Prompt"** για να αντιγράψετε αυτό το κείμενο ως αφετηρία για το δικό σας προσαρμοσμένο prompt.
    *   **Custom Prompt Input**: εδώ γράφετε τις δικές σας οδηγίες για το LLM.
    *   **Placeholders**: μπορείτε (και πρέπει) να χρησιμοποιείτε ειδικά placeholders στο prompt σας, τα οποία το πρόσθετο θα αντικαταστήσει με πραγματικό περιεχόμενο πριν στείλει το αίτημα στο LLM. Ανατρέξτε στο προεπιλεγμένο prompt για να δείτε ποια placeholders είναι διαθέσιμα για κάθε εργασία. Τα συνήθη placeholders περιλαμβάνουν:
        *   `{TITLE}`: ο τίτλος της τρέχουσας σημείωσης.
        *   `{RESEARCH_CONTEXT_SECTION}`: το περιεχόμενο που συλλέχθηκε από την έρευνα ιστού.
        *   `{USER_PROMPT}`: το περιεχόμενο της σημείωσης που επεξεργάζεται.

#### Πεδίο ελέγχου διπλοτύπων
-   **Duplicate Check Scope Mode**: ελέγχει ποια αρχεία συγκρίνονται με τις σημειώσεις στον Concept Note Folder για πιθανά διπλότυπα.
    *   **Entire Vault (Default)**: συγκρίνει τα concept notes με όλες τις άλλες σημειώσεις στο vault (εξαιρείται το ίδιο το Concept Note Folder).
    *   **Include Specific Folders Only**: συγκρίνει τα concept notes μόνο με τις σημειώσεις που βρίσκονται στους φακέλους που παρατίθενται παρακάτω.
    *   **Exclude Specific Folders**: συγκρίνει τα concept notes με όλες τις σημειώσεις *εκτός* από εκείνες μέσα στους φακέλους που παρατίθενται παρακάτω (και επίσης εξαιρώντας το Concept Note Folder).
    *   **Concept Folder Only**: συγκρίνει τα concept notes μόνο με *άλλες σημειώσεις μέσα στο Concept Note Folder*. Αυτό βοηθά στον εντοπισμό διπλοτύπων αποκλειστικά μέσα στις παραγόμενες έννοιες.
-   **Include/Exclude Folders**: (ορατό μόνο αν η λειτουργία είναι `Include` ή `Exclude`) εισαγάγετε τις *σχετικές διαδρομές* των φακέλων που θέλετε να συμπεριλάβετε ή να εξαιρέσετε, **μία διαδρομή ανά γραμμή**. Οι διαδρομές είναι ευαίσθητες σε πεζά/κεφαλαία και χρησιμοποιούν `/` ως διαχωριστικό (π.χ. `Reference Material/Papers` ή `Daily Notes`). Αυτοί οι φάκελοι δεν μπορούν να είναι ίδιοι ή να βρίσκονται μέσα στο Concept Note Folder.

-   **Tavily API Key**: (ορατό μόνο αν έχει επιλεγεί Tavily) εισαγάγετε το API key σας από το [tavily.com](https://tavily.com/).
-   **Tavily Max Results**: (ορατό μόνο αν έχει επιλεγεί Tavily) ο μέγιστος αριθμός αποτελεσμάτων που πρέπει να επιστρέψει το Tavily (1-20). Προεπιλογή: 5.
-   **Tavily Search Depth**: (ορατό μόνο αν έχει επιλεγεί Tavily) επιλέξτε `basic` (προεπιλογή) ή `advanced`. Σημείωση: το `advanced` δίνει καλύτερα αποτελέσματα αλλά κοστίζει 2 API credits ανά αναζήτηση αντί για 1.
-   **DuckDuckGo Max Results**: (ορατό μόνο αν έχει επιλεγεί DuckDuckGo) ο μέγιστος αριθμός αποτελεσμάτων αναζήτησης προς ανάλυση (1-10). Προεπιλογή: 5.

-   **Max Research Content Tokens**: κατά προσέγγιση μέγιστος αριθμός token από τα συνδυασμένα αποτελέσματα έρευνας ιστού (snippets/ανακτημένο περιεχόμενο) που θα συμπεριληφθούν στο prompt σύνοψης. Βοηθά στη διαχείριση του μεγέθους του παραθύρου συμφραζομένων και του κόστους. (Προεπιλογή: 3000)

## Οδηγός χρήσης

### Γρήγορες ροές εργασίας και πλευρική στήλη

-   Ανοίξτε την πλευρική στήλη Notemd για να αποκτήσετε πρόσβαση σε ομαδοποιημένες ενότητες ενεργειών για βασική επεξεργασία, δημιουργία, μετάφραση, γνώση και βοηθητικά εργαλεία.
-   Χρησιμοποιήστε την περιοχή **Quick Workflows** στην κορυφή της πλευρικής στήλης για να εκκινείτε προσαρμοσμένα κουμπιά πολλαπλών βημάτων.
-   Η προεπιλεγμένη ροή **One-Click Extract** εκτελεί `Process File (Add Links)` -> `Batch Generate from Titles` -> `Batch Mermaid Fix`.
-   Η πρόοδος της ροής εργασίας, τα logs ανά βήμα και οι αποτυχίες εμφανίζονται στην πλευρική στήλη, με pinned footer που προστατεύει τη μπάρα προόδου και την περιοχή log από το να συμπιεστούν όταν επεκτείνονται οι ενότητες.
-   Η κάρτα προόδου διατηρεί ευανάγνωστα με μια ματιά το κείμενο κατάστασης, το ειδικό ποσοστό και τον υπολειπόμενο χρόνο, ενώ οι ίδιες προσαρμοσμένες ροές εργασίας μπορούν να επαναδιαμορφωθούν από τις ρυθμίσεις.

### Αρχική επεξεργασία (προσθήκη wiki-links)
Αυτή είναι η βασική λειτουργικότητα που εστιάζει στον εντοπισμό εννοιών και την προσθήκη `[[wiki-links]]`.

**Σημαντικό:** αυτή η διαδικασία λειτουργεί μόνο σε αρχεία `.md` ή `.txt`. Μπορείτε να μετατρέψετε δωρεάν PDF αρχεία σε MD χρησιμοποιώντας το [Mineru](https://github.com/opendatalab/MinerU) πριν από την περαιτέρω επεξεργασία.

1.  **Χρήση της πλευρικής στήλης**:
    *   Ανοίξτε την πλευρική στήλη Notemd (εικονίδιο ραβδιού ή παλέτα εντολών).
    *   Ανοίξτε το αρχείο `.md` ή `.txt`.
    *   Κάντε κλικ στο **"Process File (Add Links)"**.
    *   Για επεξεργασία φακέλου: κάντε κλικ στο **"Process Folder (Add Links)"**, επιλέξτε τον φάκελο και κάντε κλικ στο "Process".
    *   Η πρόοδος εμφανίζεται στην πλευρική στήλη. Μπορείτε να ακυρώσετε την εργασία χρησιμοποιώντας το κουμπί "Cancel Processing" στην πλευρική στήλη.
    *   *Σημείωση για επεξεργασία φακέλου:* τα αρχεία επεξεργάζονται στο παρασκήνιο χωρίς να ανοίγουν στον editor.

<img width="618" height="154" alt="image" src="https://github.com/user-attachments/assets/fcfbcc9e-3c80-4e84-b9bb-e3a5cd66acaa" />

2.  **Χρήση της παλέτας εντολών** (`Ctrl+P` ή `Cmd+P`):
    *   **Single File**: ανοίξτε το αρχείο και εκτελέστε το `Notemd: Process Current File`.
    *   **Folder**: εκτελέστε το `Notemd: Process Folder`, μετά επιλέξτε τον φάκελο. Τα αρχεία επεξεργάζονται στο παρασκήνιο χωρίς να ανοίγουν στον editor.
    *   Για ενέργειες από την παλέτα εντολών εμφανίζεται modal προόδου, το οποίο περιλαμβάνει και κουμπί ακύρωσης.
    *   *Σημείωση:* το πρόσθετο αφαιρεί αυτόματα τυχόν αρχικές γραμμές `\boxed{` και τελικές γραμμές `}` από το τελικό επεξεργασμένο περιεχόμενο πριν από την αποθήκευση.

### Νέες δυνατότητες

1.  **Summarise as Mermaid diagram**:
    *   Ανοίξτε τη σημείωση που θέλετε να συνοψίσετε.
    *   Εκτελέστε την εντολή `Notemd: Summarise as Mermaid diagram` (μέσω παλέτας εντολών ή κουμπιού στην πλευρική στήλη).
    *   Το πρόσθετο θα δημιουργήσει νέα σημείωση με το διάγραμμα Mermaid.

2.  **Translate Note/Selection**:
    *   Επιλέξτε κείμενο σε μια σημείωση για να μεταφράσετε μόνο το επιλεγμένο τμήμα, ή καλέστε την εντολή χωρίς επιλογή για να μεταφράσετε ολόκληρη τη σημείωση.
    *   Εκτελέστε την εντολή `Notemd: Translate Note/Selection` (μέσω παλέτας εντολών ή κουμπιού στην πλευρική στήλη).
    *   Θα εμφανιστεί modal που σας επιτρέπει να επιβεβαιώσετε ή να αλλάξετε το **Target Language** (με προεπιλογή την τιμή που ορίζεται στη Διαμόρφωση).
    *   Το πρόσθετο χρησιμοποιεί τον διαμορφωμένο **LLM Provider** (με βάση τις ρυθμίσεις Multi-Model) για να εκτελέσει τη μετάφραση.
    *   Το μεταφρασμένο περιεχόμενο αποθηκεύεται στη διαμορφωμένη **Translation Save Path** με το κατάλληλο suffix και ανοίγει σε **νέο pane στα δεξιά** του αρχικού περιεχομένου για εύκολη σύγκριση.
    *   Μπορείτε να ακυρώσετε αυτή την εργασία από το κουμπί της πλευρικής στήλης ή το κουμπί ακύρωσης του modal.
3.  **Batch Translate**:
    *   Εκτελέστε την εντολή `Notemd: Batch Translate Folder` από την παλέτα εντολών και επιλέξτε φάκελο, ή κάντε δεξί κλικ σε φάκελο στον file explorer και επιλέξτε "Batch translate this folder".
    *   Το πρόσθετο θα μεταφράσει όλα τα Markdown αρχεία στον επιλεγμένο φάκελο.
    *   Τα μεταφρασμένα αρχεία αποθηκεύονται στη διαμορφωμένη translation path αλλά δεν ανοίγουν αυτόματα.
    *   Αυτή η διαδικασία μπορεί να ακυρωθεί μέσω του modal προόδου.

Τα αρχεία επεξεργάζονται στο παρασκήνιο χωρίς να ανοίγουν στον editor.

    *   Η εντολή αυτή σέβεται τη ρύθμιση **"Enable Research in 'Generate from Title'"** για κάθε σημείωση που επεξεργάζεται.
    *   Μπορείτε να ακυρώσετε αυτή την εργασία από το κουμπί της πλευρικής στήλης ή το κουμπί ακύρωσης του modal.

7.  **Extract Concepts (Pure Mode)**:
    *   Αυτή η δυνατότητα σας επιτρέπει να εξάγετε έννοιες από έγγραφο και να δημιουργείτε τα αντίστοιχα concept notes *χωρίς* να τροποποιείτε το αρχικό αρχείο. Είναι ιδανική για γρήγορο γέμισμα της βάσης γνώσης σας από ένα σύνολο εγγράφων.
    *   **Single File**: ανοίξτε ένα αρχείο και εκτελέστε την εντολή `Notemd: Extract concepts (create concept notes only)` από την παλέτα εντολών ή κάντε κλικ στο κουμπί **"Extract concepts (current file)"** στην πλευρική στήλη.
    *   **Folder**: εκτελέστε την εντολή `Notemd: Batch extract concepts from folder` από την παλέτα εντολών ή κάντε κλικ στο κουμπί **"Extract concepts (folder)"** στην πλευρική στήλη, και έπειτα επιλέξτε φάκελο για να επεξεργαστείτε όλες τις σημειώσεις του.
    *   Το πρόσθετο θα διαβάσει τα αρχεία, θα εντοπίσει έννοιες και θα δημιουργήσει νέες σημειώσεις για αυτές μέσα στον καθορισμένο **Concept Note Folder**, αφήνοντας τα αρχικά αρχεία ανέγγιχτα.

8.  **Create Wiki-Link & Generate Note from Selection**:
    *   Αυτή η ισχυρή εντολή απλοποιεί τη διαδικασία δημιουργίας και συμπλήρωσης νέων concept notes.
    *   Επιλέξτε μια λέξη ή φράση στον editor.
    *   Εκτελέστε την εντολή `Notemd: Create Wiki-Link & Generate Note from Selection` (συνιστάται να της αναθέσετε συντόμευση, όπως `Cmd+Shift+W`).
    *   Το πρόσθετο θα:
        1.  Αντικαταστήσει το επιλεγμένο κείμενο με ένα `[[wiki-link]]`.
        2.  Ελέγξει αν υπάρχει ήδη σημείωση με αυτόν τον τίτλο μέσα στον **Concept Note Folder**.
        3.  Αν υπάρχει, θα προσθέσει backlink προς την τρέχουσα σημείωση.
        4.  Αν δεν υπάρχει, θα δημιουργήσει νέα, κενή σημείωση.
        5.  Έπειτα θα εκτελέσει αυτόματα την εντολή **"Generate Content from Title"** πάνω στη νέα ή υπάρχουσα σημείωση, γεμίζοντάς την με περιεχόμενο που έχει παραχθεί από AI.

9.  **Extract Concepts and Generate Titles**:
    *   Αυτή η εντολή συνδέει δύο ισχυρές λειτουργίες σε μία απλοποιημένη ροή εργασίας.
    *   Εκτελέστε την εντολή `Notemd: Extract Concepts and Generate Titles` από την παλέτα εντολών (συνιστάται να της αναθέσετε συντόμευση).
    *   Το πρόσθετο θα:
        1.  Εκτελέσει πρώτα την εργασία **"Extract concepts (current file)"** πάνω στο τρέχον ενεργό αρχείο.
        2.  Στη συνέχεια θα εκτελέσει αυτόματα την εργασία **"Batch generate from titles"** στον φάκελο που έχετε ρυθμίσει στις ρυθμίσεις ως **Concept note folder path**.
    *   Αυτό σας επιτρέπει να γεμίσετε αρχικά τη βάση γνώσης σας με νέες έννοιες από ένα έγγραφο πηγής και αμέσως μετά να αναπτύξετε αυτά τα νέα concept notes με AI-generated περιεχόμενο σε ένα μόνο βήμα.

10. **Extract Specific Original Text**:
    *   Διαμορφώστε τις ερωτήσεις σας στις ρυθμίσεις, στην ενότητα "Extract Specific Original Text".
    *   Χρησιμοποιήστε το κουμπί "Extract Specific Original Text" στην πλευρική στήλη για να επεξεργαστείτε το ενεργό αρχείο.
    *   **Merged Mode**: επιτρέπει ταχύτερη επεξεργασία στέλνοντας όλες τις ερωτήσεις σε ένα prompt.
    *   **Translation**: μεταφράζει προαιρετικά το εξαγμένο κείμενο στη διαμορφωμένη γλώσσα σας.
    *   **Custom Output**: διαμορφώστε πού και πώς θα αποθηκεύεται το εξαγμένο αρχείο.

11. **Batch Mermaid Fix**:
    *   Χρησιμοποιήστε το κουμπί "Batch Mermaid Fix" στην πλευρική στήλη για να σαρώσετε έναν φάκελο και να διορθώσετε κοινά Mermaid syntax errors.
    *   Το πρόσθετο θα αναφέρει τυχόν αρχεία που εξακολουθούν να περιέχουν σφάλματα σε αρχείο `mermaid_error_{foldername}.md`.
    *   Προαιρετικά ρυθμίστε το πρόσθετο να μετακινεί αυτά τα προβληματικά αρχεία σε ξεχωριστό φάκελο για έλεγχο.

## Χρήση δικτύου και διαχείριση δεδομένων

Το Notemd εκτελείται τοπικά μέσα στο Obsidian, αλλά ορισμένες λειτουργίες αποστέλλουν εξερχόμενα αιτήματα.

### Κλήσεις προς παρόχους LLM (παραμετροποιήσιμες)

- Ενεργοποιητής: επεξεργασία αρχείων, δημιουργία, μετάφραση, σύνοψη έρευνας, σύνοψη Mermaid και ενέργειες σύνδεσης/διαγνωστικών.
- Endpoint: τα διαμορφωμένα provider base URL(s) σας στις ρυθμίσεις του Notemd.
- Δεδομένα που αποστέλλονται: το κείμενο του prompt και το περιεχόμενο της εργασίας που απαιτείται για την επεξεργασία.
- Σημείωση διαχείρισης δεδομένων: τα API keys διαμορφώνονται τοπικά στις ρυθμίσεις του πρόσθετου και χρησιμοποιούνται για την υπογραφή αιτημάτων από τη συσκευή σας.

### Κλήσεις έρευνας ιστού (προαιρετικές)

- Ενεργοποιητής: όταν η έρευνα ιστού είναι ενεργοποιημένη και έχει επιλεγεί πάροχος αναζήτησης.
- Endpoint: Tavily API ή DuckDuckGo endpoints.
- Δεδομένα που αποστέλλονται: το ερευνητικό query σας και τα απαραίτητα request metadata.

### Διαγνωστικά προγραμματιστών και debug logs (προαιρετικά)

- Ενεργοποιητής: API debug mode και ενέργειες developer diagnostics.
- Αποθήκευση: τα diagnostic και error logs γράφονται στη ρίζα του vault σας (π.χ. `Notemd_Provider_Diagnostic_*.txt` και `Notemd_Error_Log_*.txt`).
- Σημείωση κινδύνου: τα logs μπορεί να περιέχουν αποσπάσματα request/response. Ελέγξτε τα logs πριν τα μοιραστείτε δημόσια.

### Τοπική αποθήκευση

- Η διαμόρφωση του πρόσθετου αποθηκεύεται στο `.obsidian/plugins/notemd/data.json`.
- Τα παραγόμενα αρχεία, οι αναφορές και τα προαιρετικά logs αποθηκεύονται στο vault σας σύμφωνα με τις ρυθμίσεις σας.

## Αντιμετώπιση προβλημάτων

### Συχνά ζητήματα
-   **Το πρόσθετο δεν φορτώνει**: βεβαιωθείτε ότι τα `manifest.json`, `main.js`, `styles.css` βρίσκονται στον σωστό φάκελο (`<Vault>/.obsidian/plugins/notemd/`) και επανεκκινήστε το Obsidian. Ελέγξτε το Developer Console (`Ctrl+Shift+I` ή `Cmd+Option+I`) για σφάλματα κατά την εκκίνηση.
-   **Αποτυχίες επεξεργασίας / API Errors**:
    1.  **Ελέγξτε τη μορφή αρχείου**: βεβαιωθείτε ότι το αρχείο που προσπαθείτε να επεξεργαστείτε ή να ελέγξετε έχει επέκταση `.md` ή `.txt`. Το Notemd υποστηρίζει προς το παρόν μόνο αυτές τις μορφές κειμένου.
    2.  Χρησιμοποιήστε την εντολή/το κουμπί "Test LLM Connection" για να επαληθεύσετε τις ρυθμίσεις του ενεργού παρόχου.
    3.  Ελέγξτε ξανά τα API Key, Base URL, Model Name και API Version (για Azure). Βεβαιωθείτε ότι το API key είναι σωστό και διαθέτει επαρκή credits/permissions.
    4.  Βεβαιωθείτε ότι ο τοπικός σας LLM server (LMStudio, Ollama) εκτελείται και ότι το Base URL είναι σωστό (π.χ. `http://localhost:1234/v1` για LMStudio).
    5.  Ελέγξτε τη σύνδεσή σας στο διαδίκτυο για cloud παρόχους.
    6.  **Για σφάλματα επεξεργασίας ενός αρχείου:** ελέγξτε το Developer Console για λεπτομερή μηνύματα σφάλματος. Αν χρειάζεται, αντιγράψτε τα μέσω του κουμπιού στο error modal.
    7.  **Για σφάλματα μαζικής επεξεργασίας:** ελέγξτε το αρχείο `error_processing_filename.log` στη ρίζα του vault για λεπτομερή μηνύματα σφάλματος για κάθε αποτυχημένο αρχείο. Το Developer Console ή το error modal μπορεί να εμφανίζουν μόνο σύνοψη ή γενικό batch error.
    8.  **Αυτόματα αρχεία καταγραφής σφαλμάτων:** αν μια διαδικασία αποτύχει, το πρόσθετο αποθηκεύει αυτόματα ένα λεπτομερές αρχείο log με όνομα `Notemd_Error_Log_[Timestamp].txt` στον ριζικό κατάλογο του vault. Αυτό το αρχείο περιέχει το μήνυμα σφάλματος, ίχνος στοίβας και logs συνεδρίας. Αν αντιμετωπίζετε επίμονα προβλήματα, ελέγξτε αυτό το αρχείο. Η ενεργοποίηση του "API Error Debugging Mode" στις ρυθμίσεις θα εμπλουτίσει αυτό το log με ακόμη πιο αναλυτικά API response δεδομένα.
    9.  **Διαγνωστικά μεγάλων αιτημάτων σε πραγματικό endpoint (Developer)**:
        - Εντός πρόσθετου (συνιστάται πρώτο): χρησιμοποιήστε το **Settings -> Notemd -> Developer provider diagnostic (long request)** για να εκτελέσετε runtime probe στον ενεργό πάροχο και να παραχθεί `Notemd_Provider_Diagnostic_*.txt` στη ρίζα του vault.
        - Διαδρομή CLI (εκτός Obsidian runtime): για αναπαραγώγιμη σύγκριση συμπεριφοράς buffered και streaming σε επίπεδο endpoint, χρησιμοποιήστε:
        ```bash
        npm run diagnose:llm -- \
          --transport openai-compatible \
          --provider-name OpenRouter \
          --base-url https://openrouter.ai/api/v1 \
          --api-key "$OPENROUTER_API_KEY" \
          --model anthropic/claude-3.7-sonnet \
          --prompt-file ./tmp/prompt.txt \
          --content-file ./tmp/content.txt \
          --mode compare \
          --timeout-ms 360000 \
          --output ./tmp/openrouter-diagnostic.txt
        ```
        Η παραγόμενη αναφορά περιέχει χρονισμούς ανά προσπάθεια (`First Byte`, `Duration`), καθαρισμένα request metadata, response headers, raw/partial body fragments, parsed stream fragments και transport-layer failure points.
-   **Προβλήματα σύνδεσης LM Studio/Ollama**:
    *   **Test Connection Fails**: βεβαιωθείτε ότι ο τοπικός server (LM Studio ή Ollama) εκτελείται και ότι το σωστό μοντέλο είναι φορτωμένο/διαθέσιμο.
    *   **CORS Errors (Ollama σε Windows)**: αν συναντήσετε σφάλματα CORS (Cross-Origin Resource Sharing) όταν χρησιμοποιείτε Ollama σε Windows, ίσως χρειαστεί να ορίσετε τη μεταβλητή περιβάλλοντος `OLLAMA_ORIGINS`. Μπορείτε να το κάνετε εκτελώντας `set OLLAMA_ORIGINS=*` στο command prompt πριν ξεκινήσετε το Ollama. Αυτό επιτρέπει αιτήματα από οποιοδήποτε origin.
    *   **Enable CORS in LM Studio**: για το LM Studio μπορείτε να ενεργοποιήσετε το CORS απευθείας από τις ρυθμίσεις server, κάτι που μπορεί να είναι απαραίτητο αν το Obsidian τρέχει σε browser ή έχει αυστηρές πολιτικές origin.
-   **Σφάλματα δημιουργίας φακέλων ("File name cannot contain...")**:
    *   Αυτό συνήθως σημαίνει ότι η διαδρομή που δόθηκε στις ρυθμίσεις (**Processed File Folder Path** ή **Concept Note Folder Path**) είναι άκυρη *για το Obsidian*.
    *   **Βεβαιωθείτε ότι χρησιμοποιείτε σχετικές διαδρομές** (π.χ. `Processed`, `Notes/Concepts`) και **όχι απόλυτες διαδρομές** (π.χ. `C:\Users\...`, `/Users/...`).
    *   Ελέγξτε για μη έγκυρους χαρακτήρες: `* " \ / < > : | ? # ^ [ ]`. Σημειώστε ότι το `\` είναι άκυρο ακόμη και στα Windows για διαδρομές του Obsidian. Χρησιμοποιήστε `/` ως διαχωριστικό διαδρομής.
-   **Προβλήματα απόδοσης**: η επεξεργασία μεγάλων αρχείων ή πολλών αρχείων μπορεί να χρειαστεί χρόνο. Μειώστε τη ρύθμιση "Chunk Word Count" για πιθανώς ταχύτερες (αλλά περισσότερες) API calls. Δοκιμάστε διαφορετικό πάροχο LLM ή διαφορετικό μοντέλο.
-   **Απρόσμενη διασύνδεση**: η ποιότητα της διασύνδεσης εξαρτάται σε μεγάλο βαθμό από το LLM και το prompt. Πειραματιστείτε με διαφορετικά μοντέλα ή ρυθμίσεις θερμοκρασίας.
