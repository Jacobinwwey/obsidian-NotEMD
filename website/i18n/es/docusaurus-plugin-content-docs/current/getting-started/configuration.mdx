# Configuración

<TLDR>
**La configuración Notemd abarca 6 áreas principales:** (1) proveedores LLM y claves API, (2) selección del modelo por tarea, (3) rutas de salida y plantillas, (4) idioma UI e idioma de salida, (5) servicios de búsqueda (Tavily/DuckDuckGo), (6) flujos de trabajo personalizados. Todos los ajustes se encuentran en Obsidian Ajustes → Notemd. Los cambios surten efecto de inmediato.
</TLDR>

## Resumen de configuraciones

Acceso: **Obsidian Ajustes (⚙️) → Notemd**

### Secciones principales

1. **Configuración del proveedor** — claves API, URLs base, modelos
2. **Configuración de tareas** — Selección de modelo por tarea
3. **Configuración de salida** — Dónde guardar los archivos generados
4. **Ajustes de idioma** — UI e idiomas de salida
5. **Integración de Búsqueda** — configuración de Tavily/DuckDuckGo
6. **Flujos de trabajo** — Cadenas de botones personalizadas
7. **Avanzado** — Diagnóstico, procesamiento por lotes, fragmentación

---

## 1. Configuración del proveedor

### Agregar un proveedor

1. **Seleccionar proveedor** del menú desplegable (más de 30 disponibles)
2. **Introduce la tecla API**
3. **Opcional**: Establecer una base personalizada URL (para proxies/puertas de enlace)
4. **Haga clic en "Obtener lista de modelos"** para verificar la conexión
5. **Seleccionar modelo predeterminado**

### Proveedores comunes

#### OpenAI
```
Provider: OpenAI
API Key: sk-...
Base URL: https://api.openai.com/v1 (default)
Model: gpt-4o-mini
```

#### Anthropic (Claude)
```
Provider: Anthropic
API Key: sk-ant-...
Model: claude-sonnet-4-6
```

#### Ollama (Local)
```
Provider: Ollama
Base URL: http://localhost:11434 (default)
Model: llama3.1
API Key: (not required)
```

#### DeepSeek
```
Provider: DeepSeek
API Key: sk-...
Model: deepseek-chat
```

### Múltiples proveedores

Puede configurar varios proveedores e intercambiarlos según la tarea:
- **Proveedor principal**: Se utiliza como solución de respaldo
- **Proveedores específicos de tarea**: Sobrescribir para tareas concretas

## 2. Modelos específicos para tareas

Configure modelos diferentes para tareas distintas a fin de optimizar costo/calidad:

### Tipos de tareas

| Tarea | Modelo predeterminado | Propósito |
|------|--------------|---------|
| **Vinculación** | Proveedor principal | Identificar conceptos para enlaces de wiki |
| **Generación de conceptos** | Proveedor principal | Extraer y describir conceptos |
| **Investigación** | Proveedor principal | Resumir los resultados de búsqueda en la web |
| **Traducción** | Proveedor principal | Traducir texto |
| **Generación de diagramas** | Proveedor principal | Generar diagramas Mermaid/Canvas |

### Ejemplo: Configuración optimizada en costos

```
Linking: gpt-4o-mini (fast, many calls)
Concept Generation: claude-sonnet-4-6 (quality extraction)
Research: gpt-4o (web-grounded)
Translation: gemini-flash (multilingual)
Diagram: ollama/llama3.1 (free, local)
```

### Cómo configurar

1. **Ajustes → Notemd → Ajustes de tarea**
2. Para cada tarea, elige:
   - ✅ **Usar proveedor principal** (predeterminado)
   - 🔄 **Usar proveedor personalizado** → seleccionar proveedor + modelo

## 3. Configuración de salida

### Salida de notas conceptuales

**Dónde se guardan las notas de concepto nuevas:**

```
Settings → Output → Concept Folder
Default: concepts/
```

**Plantilla:**
```
Settings → Output → Concept Template
Default:
---
tags: [concept]
created: {{date}}
---

# {{concept}}

{{description}}

## Backlinks
{{backlinks}}
```

### Opciones de estructura de carpetas

**Opción 1: Plano (todos los conceptos en una sola carpeta)**
```
vault/
└── concepts/
    ├── Machine Learning.md
    ├── Neural Networks.md
    └── Deep Learning.md
```

**Opción 2: Jerárquico (por carpeta de origen)**
```
vault/
├── papers/
│   └── concepts/
│       └── Transformer.md
└── notes/
    └── concepts/
        └── Attention Mechanism.md
```

Configurar: **Ajustes → Salida → Usar carpetas jerárquicas**

### Salida del diagrama

**Dónde se guardan los diagramas:**
```
Settings → Output → Diagram Folder
Default: diagrams/
```

**Formato:**
- `.mmd` (Mermaid fuente)
- `.canvas` (JSON Canvas)
- `.html` (Vega-Lite)

## 4. Ajustes de idioma

### UI Idioma

**Idioma de la interfaz** (botones, menús, diálogos):

```
Settings → Language → UI Language
Options: 21+ languages
Default: System language
```

Se admite: inglés, 简体中文, 繁體中文, 日本語, 한국어, Español, Français, Deutsch, Русский, العربية y más.

### Idioma de salida

**Idioma del contenido generado** (notas conceptuales, resúmenes de investigación):

```
Settings → Language → Output Language
Options: Same as UI languages
Default: Same as UI
```

**Caso de uso:** UI en chino + salida en inglés para investigación bilingüe.

### Destino de la traducción

**Idioma de destino predeterminado** para las tareas de traducción:

```
Settings → Translation → Target Language
Default: English
```

## 5. Integración de búsquedas

### Tavily (Recomendado)

**Servicio de búsqueda optimizado para IA:**

```
Settings → Search → Service: Tavily
API Key: tvly-...
```

Obtener clave: [tavily.com](https://tavily.com/)

**Características:**
- Resultados de alta calidad
- Resúmenes optimizados por IA
- Citaciones de fuentes

### DuckDuckGo (Experimental)

**No se necesita la clave API:**

```
Settings → Search → Service: DuckDuckGo
```

**Nota:** Calidad inferior, sin personalización.

## 6. Configuración del flujo de trabajo

### Creación de flujos de trabajo personalizados

**Los flujos de trabajo combinan varias tareas en un único botón.**

1. **Ajustes → Flujos de trabajo → Agregar flujo de trabajo**
2. **Nombre**: "Flujo de Análisis de Papel"
3. **DSL**: `process-current-add-links>extract-concepts-current>research-and-summarize>summarize-as-mermaid`
4. Guardar

Aparece un nuevo botón en la barra lateral izquierda.

### Sintaxis de DSL

**Formato:** `action1>action2>action3`

**Acciones disponibles:**
- `process-current-add-links` — Agregar enlaces wiki a la nota actual
- `extract-concepts-current` — Generar notas de concepto
- `research-and-summarize` — Búsqueda en la web + resumen
- `translate-current` — Traducir nota actual
- `summarize-as-mermaid` — Generar diagrama Mermaid
- `process-folder-add-links` — Procesar carpetas por lotes

**Flujos de trabajo de ejemplo:**

**Extracción rápida de conceptos:**
```
process-current-add-links>extract-concepts-current
```

**Investigación profunda:**
```
process-current-add-links>research-and-summarize>extract-concepts-current
```

**Análisis multilingüe:**
```
translate-current>process-current-add-links>extract-concepts-current
```

## 7. Configuración avanzada

### Procesamiento por lotes

**Procesar múltiples archivos en paralelo:**

```
Settings → Advanced → Batch Processing
Concurrency: 3 (default)
```

Mayor = más rápido, pero más llamadas a API.

### Fragmentación inteligente

**Para notas largas, divídalas en fragmentos:**

```
Settings → Advanced → Smart Chunking
Chunk Size: 2000 words (default)
Overlap: 200 words (default)
```

Evita errores por límite de tokens.

### Panel de Diagnóstico

**Ver registros detallados:**

```
Settings → Advanced → Enable Diagnostics
```

Programas:
- API solicitudes/respuestas
- Uso de tokens
- Tiempos de procesamiento
- Errores

### Detección de conceptos duplicados

**Evitar conceptos casi idénticos:**

```
Settings → Advanced → Synonym Suppression
Threshold: 0.8 (default, 0-1)
```

Mayor = fusión más agresiva.

## Perfiles de configuración

### Configuración de exportación/importación

**Compartir configuraciones entre dispositivos:**

1. **Ajustes → Avanzado → Exportar configuración**
2. Guarda `notemd-config.json`
3. En el dispositivo nuevo: **Importar configuración**

**Qué está incluido:**
- Claves del proveedor API (opcional)
- Selecciones de modelo
- Rutas de salida
- Flujos de trabajo personalizados

```
Provider: OpenAI (gpt-4o-mini)
Concept Folder: concepts/
UI Language: System
Output Language: System
Workflow: "Study Flow" (add-links>extract-concepts)
```

### Para investigadores

```
Providers:
  - Linking: DeepSeek (cheap, many calls)
  - Concepts: Claude Sonnet (high quality)
  - Research: GPT-4o (web-grounded)
Concept Folder: notes/concepts/
Hierarchical: Yes
Workflow: "Paper Flow" (add-links>extract>research>diagram)
```

### Para usuarios preocupados por la privacidad

```
Provider: Ollama (llama3.1)
All tasks: Local model
Search: DuckDuckGo (no API)
Diagnostics: Off
```

### Para trabajo bilingüe

```
UI Language: Chinese
Output Language: English
Translation Target: Chinese
Provider: Gemini (multilingual)
```

## Solución de problemas de configuración

### Ajustes no guardados
- Verificar los permisos del archivo en `.obsidian/plugins/notemd/`
- Prueba: Deshabilitar el complemento → Volver a habilitarlo

### API Clave inválida
- Sin espacios adicionales
- Verificar fecha de vencimiento
- Verificar créditos en el panel del proveedor

### Los modelos no se están cargando.
- Haga clic en "Obtener lista de modelos" para actualizarla
- Verificar la conectividad de red
- Prueba con diferentes proveedores
