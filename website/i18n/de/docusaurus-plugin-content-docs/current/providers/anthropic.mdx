# Anthropic Anbieter

<TLDR>
**Anthropic verwendet den nativen Messages API-Transport, nicht den mit OpenAI kompatiblen Transport.** Die aktuelle Voreinstellung verwendet standardmäßig `claude-3-5-sonnet-20240620`, `https://api.anthropic.com` und `temperature: 0.5`. Behandeln Sie es beim Debuggen als eigenständige Protokolloberfläche.

Dies ist ein Teil des [Obsidian AI-Know-how-Management-Leitfadens](/docs/pillar-ai-knowledge).
</TLDR>

## Einrichtung

Erstellen Sie einen Anthropic API-Schlüssel, fügen Sie ein `Anthropic`-Anbieterprofil hinzu und belassen Sie den Base URL am nativen Endpunkt – es sei denn, Sie leiten den Verkehr über ein spezielles Gatewayprofil. Wählen Sie im Feld „Modell“ das Claude-Modell-ID, das Sie verwenden möchten; die Standard-Einstellung hält die bestehende Produktbasis stabil.

Anthropic eignet sich hervorragend für hochwertige Zusammenfassungen, Forschungssynthesen und Konzeptextraktion. Wenn Sie es mit günstigeren Anbietern kombinieren, verwenden Sie es nur bei Aufgaben, bei denen die Qualität der Ergebnisse von Vorteil ist.

## Endpunkt und Authentifizierung

| Feld | Aktuelle Voreinstellung |
|-------|----------------|
| Transport | `anthropic` |
| API Schlüssel | Erforderlich |
| Base URL | `https://api.anthropic.com` |
| Standardmodell | `claude-3-5-sonnet-20240620` |
| Temperatur | `0.5` |
| Verbindungsprüfung | Nur Chat |

Die Laufzeit sendet Anfragen an `/v1/messages` mit `x-api-key` und `anthropic-version: 2023-06-01`. Konfigurieren Sie unter dieser Einstellung keinen OpenAI-stiligen `/v1/chat/completions`-Endpunkt; verwenden Sie stattdessen eine OpenAI-kompatible Gateway-Einstellung.

## Modellentdeckung

Anthropic verwendet im Provider-Register den `anthropic-models`-Entdeckungsmodus. Wenn Ihre Schlüssel oder Ihr Konto keine Modelle auflisten können, speichern Sie eine bekannte Modell-ID im Profil und überprüfen Sie dies durch den Verbindungstest.

Der Transport ist nicht mit OpenAI-kompatiblen Anbietern austauschbar. Ein Claude-Modell, das über OpenRouter, LiteLLM oder ein anderes Gateway geleitet wird, gehört in das Profil dieses Gateways und nicht in das native Anthropic-Profil.

## Fehlerbehebung

- `401` oder `403`: Überprüfen Sie den Schlüsselbereich, den Zugriff auf die Arbeitsumgebung, die Abrechnung sowie ob der Schlüssel zur erwarteten Anthropic Arbeitsumgebung gehört.
- `404`: Die Basis URL ist falsch oder die ausgewählte Modell-ID steht dem Konto nicht zur Verfügung.
- Fehler bei Gateway-ähnlichen Claude-Modell-IDen: Verschieben Sie das Profil zum entsprechenden Gateway-Anbieter, da das native Anthropic native Modellnamen erwartet.
- Lange Anfragen scheitern spät: Verringern Sie die Ausgabegröße des Modells pro Aufgabe oder verwenden Sie ein Modell mit einer höheren bekannten Obergrenze für Ausgabetoken.

## Wann verwenden

Verwenden Sie Anthropic, wenn bei der Synthese, in Forschungsnotizen oder bei strukturiertem Schreiben die Qualität wichtiger ist als die reine Kostenersparnis. Verwenden Sie es nicht als allgemeinen Ersatz, der mit OpenAI kompatibel ist; der getrennte Transport ist schließlich das Ziel, und das Mischen der Protokolle erschwert die Diagnose.
