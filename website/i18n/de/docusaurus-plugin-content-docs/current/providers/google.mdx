# Google Anbieter

<TLDR>
**Google verwendet den nativen Gemini Generative Language API-Transport.** Die aktuelle Voreinstellung verwendet standardmäßig `gemini-2.0-flash-exp`, `https://generativelanguage.googleapis.com/v1` und `temperature: 0.5`. Es handelt sich nicht um ein OpenAI-kompatibles Profil.

Dies ist ein Teil des [Obsidian AI-Know-how-Management-Leitfadens](/docs/pillar-ai-knowledge).
</TLDR>

## Einrichtung

Erstellen Sie einen Google AI Studio oder Google Cloud API-Schlüssel mit Gemini-Zugriff, fügen Sie ein `Google`-Anbieterprofil hinzu und behalten Sie die native Generative Language Base URL bei. Verwenden Sie das Standard-Gemini Flash-Profil für schnelle Übersetzungen und umfassende Notizwerkzeuge, und ersetzen Sie die Modelle pro Aufgabe, wenn Sie ein größeres oder leistungsstärkeres Gemini-Modell benötigen.

Vermeiden Sie es, Gemini-Modell-IDen in einem OpenAI-kompatiblen Profil zu platzieren, es sei denn, Sie verwenden ausdrücklich eine Gateway-Lösung, die Gemini über ein OpenAI-kompatibles API bereitstellt.

## Endpunkt und Authentifizierung

| Feld | Aktuelle Voreinstellung |
|-------|----------------|
| Transport | `google` |
| API Schlüssel | Erforderlich |
| Base URL | `https://generativelanguage.googleapis.com/v1` |
| Standardmodell | `gemini-2.0-flash-exp` |
| Temperatur | `0.5` |
| Verbindungsprüfung | Nur Chat |

Die Laufzeitruoutine ruft `/models/{model}:generateContent?key={apiKey}` auf. Der Schlüssel wird als Abfrageparameter übergeben, da die native Gemini-Endpunktverbindung im Plugin so konfiguriert ist.

## Modellentdeckung

Google verwendet den `google-models`-Entdeckungsmodus. Die Entdeckung sollte Gemini-Modelle zurückgeben, die für den API-Schlüssel verfügbar sind; wenn der Selektor leer ist, prüfen Sie den Schlüssel gegen denselben Base URL und bestätigen Sie, dass die Modellfamilie für dieses Konto aktiviert ist.

Die bekannten Obergrenzen für Ausgabetoken sind im Anbieterregister modellabhängig. Wenn eine lange Übersetzung oder Diagramm-Erklärung abgeschnitten wird, wählen Sie ein Gemini-Modell mit einer höheren unterstützten Ausgabogrenze oder verringern Sie die gewünschte Ausgabegröße.

## Fehlerbehebung

- `400`: Die Modell-ID stimmt in der Regel nicht mit dem nativen Gemini-Route überein oder die Anfrage überschreitet die vom Modell unterstützte Struktur.
- `401` oder `403`: Überprüfen Sie die Schlüsselbeschränkungen, die Aktivierung des Projekts/API sowie ob der Schlüssel auf Gemini zugreifen kann.
- `404`: Die Modell-ID ist in der konfigurierten API-Version nicht verfügbar.
- Beispiele für Gateways scheitern unter diesem Profil: Verwenden Sie stattdessen die voreingestellte Gateway-Einstellung anstelle des nativen Google-Anbieters.

## Wann verwenden

Verwenden Sie Google für schnelle Übersetzungen, die Bereinigung großer Mengen an Notizen sowie Workflows, in denen das Kosten-Leistungs-Verhältnis von Gemini besser ist als bei direktem Einsatz von OpenAI oder Anthropic. Wählen Sie einen anderen Anbieter für Workflows, die eine mit OpenAI kompatible Verarbeitung von Reasoning-Modellen erfordern.
