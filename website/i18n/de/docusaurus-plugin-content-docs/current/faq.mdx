## Installation und Einrichtung

### Wie installiere ich Notemd in Obsidian?

Öffnen Sie Obsidian **Einstellungen → Community-Plugins → Durchsuchen** → suchen Sie nach **"Notemd"** → **Installieren** → **Aktivieren**.

**Anforderungen:**
- Obsidian 0.15.0+
- Unterstützt auf Desktop (Windows, macOS, Linux) und Mobil (iOS, Android)

**Manuelle Installation:**
1. Laden Sie `main.js`, `styles.css`, `manifest.json` von [GitHub Releases](https://github.com/Jacobinwwey/obsidian-NotEMD/releases) herunter
2. Platzieren Sie sie in `<your-vault>/.obsidian/plugins/notemd/`
3. Lade Obsidian neu und aktiviere den Plugin

**Statistiken:** Mehr als 4.000 Downloads, 44 Veröffentlichungen, lizenziert unter MIT.

### Wie konfiguriere ich verschiedene Modelle für unterschiedliche Aufgaben?

In den **Notemd Einstellungen** hat jede Aufgabe unabhängige Selektoren für Anbieter/Modelle:

| Aufgabe | Beispiellkonfiguration |
|------|----------------------|
| **Verlinken** | DeepSeek (schnell, günstig) |
| **Konzepterstellung** | Claude Opus (hohe Qualität) |
| **Forschung** | Perplexity oder GPT-4o (web-basiert) |
| **Übersetzung** | Gemini Flash (multimodal) |
| **Diagrammgenerierung** | Ollama (lokal, privat) |

Diese pro-Aufgabe-Optimierung balanciert Geschwindigkeit, Kosten und Qualität.

## Kernfunktionen

### Inwiefern unterscheidet sich Notemd von ChatGPT bei der Lesung von Papierdokumenten?

| Abmessung | ChatGPT | Notemd |
|-----------|---------|--------|
| **Ausgabepfad** | Chatverlauf (vorübergehend) | Obsidian Vault (dauerhafte Dateien) |
| **Format des Ergebnisses** | Textantwort | Wiki-Links, Konzeptnotizen, Diagramme, Übersetzungen |
| **Langfristiger Wert** | Verschwindet nach der Sitzung | Sammelt sich in der Wissensdatenbank. |
| **Wiederverwendbarkeit** | Muss erneut abfragen | Die Dateien bleiben erhalten, der Link entwickelt sich weiter. |

**Verwenden Sie ChatGPT für:** Schnelle Fragen und Antworten, einmalige Erklärungen
**Verwenden Sie Notemd für:** Den Aufbau einer dauerhaften, strukturierten Wissensdatenbank über Wochen/Monate

### Wie fügt Notemd Wiki-Links zu Notizen hinzu?

**Prozess:**
1. Sie führen „Datei verarbeiten (Verlinkungen hinzufügen)“ in einer Notiz aus
2. Notemd sendet den Text an Ihren konfigurierten LLM
3. Der LLM identifiziert Schlüsselkonzepte im Kontext
4. Notemd fügt `[[wiki-links]]` bei jedem Vorkommen des Konzepts ein
5. Optional erstellt es automatisch Konzeptnotizen in einem von Ihnen angegebenen Ordner.

**Eigenschaften:**
- Synonymunterdrückung (vermeidet Duplikate von „transformer“ und „transformers“)
- Backlinks von Konzeptnotizen zur Quelle
- Integrität des Links bei Umbenennen/Löschen
- Reiner Extraktionsmodus (Konzepte extrahieren, ohne das Original zu verändern)

### Was ist der Ein-Klick-Arbeitsablauf von Notemd?

**Einst-Klick-Arbeitsabläufe** verknüpfen mehrere Aktionen in einer **einzigen Seitenleisten-Schaltfläche**.

**Standard-Arbeitsablauf:** `One-Click Extract`
```
process-current-add-links > extract-concepts-current > research-and-summarize > summarize-as-mermaid
```

**Ergebnis:** Mit einem Klick wird ausgeführt:
1. Füge Wiki-Links zur aktuellen Notiz hinzu
2. Erstelle Konzeptnotizen für neue Begriffe
3. Forschung und Hinzufügen von Hintergrundinformationen
4. Komprimieren in ein Mermaid Diagramm

**Benutzerdefinierte Workflows:** Definieren Sie Ihre eigenen im Einstellungen-Menü im DSL-Format. Jeder Workflow wird zu einem wiederverwendbaren Seitenleisten-Button.

### Wie vergleicht sich Notemd mit Copilot oder Smart Connections?

Diese Plugins lösen verschiedene Probleme:

- **Copilot** – Gespräch voran. Sie chatten mit einer KI über Ihre Notizen. Die Antworten der KI bleiben im Chatfenster; sie ändern Ihren Safe nicht.
- **Smart Connections** – Entdeckung zuerst. Es findet semantisch ähnliche Notizen mithilfe von Embeddings. Es erstellt oder modifiziert keinen Inhalt.
- **Notemd** — Zuerst schreiben. Die KI verarbeitet Ihre Notizen und erstellt strukturierte Ergebnisse (Wiki-Links, Konzeptnotizen, Forschungsergebnisse, Diagramme, Übersetzungen) direkt als dauerhafte Dateien in Ihrem Vault.

| Was Sie benötigen | Empfohlener Plugin |
|----------------|-------------------|
| Mit KI über Notizen chatten | Copilot |
| Ähnliche/verwandte Notizen finden | Smart Connections |
| Automatische Einbindung von Wiki-Links und Erstellung eines Wissensgraphen | **Notemd** |
| Konzeptnotizen mit Backlinks erstellen | **Notemd** |
| Erstellen Sie Diagramme aus Notizinhalten | **Notemd** |
| Erforsche ein Thema und füge Zusammenfassungen hinzu | **Notemd** |

Alle drei können zusammen verwendet werden – sie ergänzen sich statt miteinander zu konkurrieren.

## Forschung & Übersetzung

### Welche Suchdienste funktionieren mit der Notemd-Forschung?

| Service | API Schlüssel erforderlich | Qualität | Notizen |
|---------|-----------------|---------|-------|
| **Tavily** | Ja | Hoch | Empfohlene, von KI optimierte Suche |
| **DuckDuckGo** | Nein | Experimentell | Keine Einrichtung, geringere Qualität |

**So funktioniert „Forschung und Zusammenfassung“:**
1. Sie wählen ein Thema aus oder führen es in der aktuellen Notiz aus.
2. Notemd fragt den Suchdienst ab
3. Ergebnisse → Zusammenfassung von LLM
4. Zusammenfassung wurde Ihrer Notiz mit Quellenangaben hinzugefügt

**Ergebnis:** Dauerhafter Forschungsverlauf in Ihrem Tresor, nicht in Browsertabellen.

### Unterstützt Notemd Chinesisch oder andere Sprachen?

**UI Sprachen:** Mehr als 21 unterstützt
- Englisch, Chinesisch (vereinfacht), Chinesisch (traditionell), Japanisch, Koreanisch
- Arabisch, Deutsch, Französisch, Spanisch, Russisch, Portugiesisch, Italienisch
- Nein, Bengali, Thailändisch, Vietnamesisch, Türkisch, Polnisch, Tschechisch und mehr

**Sprache der Aufgabenergebnisse:** Unabhängige Konfiguration
- UI auf Chinesisch + Ausgabe auf Englisch: ✅
- UI auf Englisch + Ausgabe auf Chinesisch: ✅

**Anwendungsfall:** Zweisprachige Forscher können UI in der Muttersprache beibehalten, während sie englische Notizen für die Veröffentlichung erstellen.

## PDF & Akademische Arbeitsabläufe

### Kann ich Notemd für das Lesen auf Papier/PDF verwenden?

**Ja.** Workflow:
1. **Konvertiere PDF → Markdown** mit [MinerU](https://github.com/opendatalab/MinerU) (empfohlen für wissenschaftliche Arbeiten)
2. Öffnen Sie die Markdown-Datei in Obsidian
3. Ausführen von Notemd Aufgaben:
   - **Wiki-Links hinzufügen** zu Schlüsselbegriffen
   - **Konzeptnotizen erstellen** für unbekannte Fachbegriffe
   - **Ursprünglicher Text extrahieren** über benutzerdefinierte Fragen (z. B. „Welche Einschränkungen gibt es?“)
   - **Forschungshintergrund** zu zitierten Methoden/Datensätzen
   - Übersetzen Sie schwierige Abschnitte
   - **Komprimieren zu Mermaid** Diagramm zur strukturellen Verständnis

**Version 1.9.1+:** Extraktion der Kapitelstruktur mit TOC-Unterstützung

**Ergebnis:** Das Lesen Ihres Artikels wird zu einem strukturierten, verlinkbaren Wissensobjekt in Ihrem Vault.

## Diagramme & Visualisierung

### Welche Graphen/Diagrammtypen kann Notemd erzeugen?

| Typen | Support | Funktionen |
|------|---------|----------|
| **Mermaid** | Vollständig | Flussdiagramm, Sequenz, Klasse, Zustand, ER-Diagramm, Gantt-Diagramm, automatische Syntaxkorrektur, Batch-Validierung |
| **JSON Canvas** | Vollständig | Obsidian nativ Layouts für Canvas |
| **Vega-Lite** | Vollständig | Datencharts, Zeitreihen, Balkendiagramme, Streudiagramme, anpassbar |
| **HTML** | Fallback | Für nicht unterstützte Diagrammtypen |
| **Bearbeitbare HTML/SVG** | Explizites Ziel | Selbstständiger Inline-Modul SVG mit semantischen Anmerkungen im Draw.io-Stil |
| **Draw.io / Drawnix Artefakte** | Grenze für Export | Deterministischer XML/JSON aus `SemanticFigureModel`; keine Abhängigkeit von der Editor-Laufzeit |

**Mermaid Auto-Fix:** Tiefgehender Debug-Modus zur Reparatur:
- Zerbrochene Pfeile (`-->`, `-.->`, `==>`)
- Ungültige Labels, Kommentare
- Syntaxfehler im Subgraphen
- Formnotation (`[`, `(`, `{`, `((`)
- Pfeichensymbole in Etiketten
- Anhänge notieren

**Workflow:**
1. Führen Sie „Summarize as Mermaid diagram“ auf einer Notiz aus
2. Voransicht im Modalfenster
3. Exportieren als SVG oder PNG
4. Oder speichern als `.mmd`-Datei im Tresor

## Fehlerbehebung

### Meine LLM-Anrufe funktionieren nicht. Was sollte ich überprüfen?

1. **Ist der API-Schlüssel gültig?** Überprüfen Sie Einstellungen → Anbieterkonfiguration
2. **Ist der Modellname korrekt?** Verwenden Sie die Schaltfläche „获取模型列表“ (Get Model List) in den Einstellungen.
3. **Ist das Netzwerk erreichbar?** Einige Anbieter erfordern eine VPN.
4. **Rate Limits?** Überprüfen Sie das Dashboard des Anbieters
5. **Diagnose:** Führen Sie im Einstellungenmenü den „Verbindungs-Test“ aus, um detaillierte Fehlerprotokolle zu erhalten

### Notemd ist langsam. Wie kann ich es beschleunigen?

1. **Schnellere Modelle verwenden:** DeepSeek, Gemini Flash, GPT-4o-mini
2. **Größe des Chunks verringern:** Einstellungen → Intelligentes Chunking → Wortanzahl reduzieren
3. **Parallelverarbeitung:** Einstellungen → Batchverarbeitung → Konkurrenz erhöhen
4. **Lokale Modelle:** Ollama für Aufgaben, die keine Websuche benötigen

### Kann Notemd zu 100 % offline arbeiten?

**Ja, mit Ollama.** Keine API-Einstellung, kein Internet, keine Daten verlassen Ihren Rechner:

1. Installieren Sie [Ollama](https://ollama.com) und führen Sie `ollama serve` aus
2. Wählen Sie in den Einstellungen von Notemd **Ollama** als Ihren Anbieter aus
3. Modell herunterladen: `ollama pull llama3` (oder qwen2.5, gemma2 usw.)
4. Alle Funktionen funktionieren: Wiki-Links, Konzeptnotizen, Übersetzung, Diagramme

**DuckDuckGo Research** funktioniert auch ohne eine API-Schlüssel (experimentelle Qualität).

Mit Cloud-Anbietern (OpenAI, Anthropic, Google, DeepSeek usw.) wird der Inhalt Ihrer Notiz als API-Eingabe gesendet. Prüfen Sie die Datenschutzrichtlinien jedes Anbieters für Details zur Datenverarbeitung.

### Wird meine Daten in die Cloud gesendet?

**Es hängt von Ihrem Anbieter ab:**

| Typ des Anbieters | Wurden Daten in die Cloud gesendet? | Notizen |
|--------------|---------------------|-------|
| **Ollama (lokal)** | Nein | Alles bleibt auf Ihrem Gerät. |
| **LMStudio (lokal)** | Nein | Genau wie Ollama |
| **DeepSeek, OpenAI, usw.** | Ja | Hinweis: Der Inhalt wird als API-Eingabe zur Verarbeitung gesendet |
| **OpenRouter, AIHubMix** | Ja | Gateway, das zu Cloud-Anbietern weiterleitet |

Notemd sammelt selbst **keine Telemetriedaten, keine Analysedaten und keine Benutzerdaten**. Der Plugin ist vollständig open source (MIT). Die einzigen Daten, die Ihre Maschine verlassen, sind die, die Sie explizit an Ihren konfigurierten LLM-Anbieter senden.

### Welches Obsidian AI-Plugin eignet sich am besten für Forschungszwecke?

Es hängt von Ihrem Forschungsworkflow ab:

| Benötige | Plugin | Warum |
|------|--------|-----|
| Suche nach verwandten Artikeln/Notizen | Smart Connections | Entdeckung der semantischen Ähnlichkeit |
| Chatten Sie über Ihre Quellen | Copilot | Gespräch im Vault-Kontext |
| Konzepte für Auto-Links + Notizen erstellen | **Notemd** | KI erkennt Konzepte, fügt Links ein und erstellt Notizen |
| Webrecherche + Zusammenfassung in Notizen | **Notemd** | Tavily/DuckDuckGo suchen → LLM Zusammenfassung → an die Notiz angehängt |
| Diagramme aus Forschungsergebnissen erstellen | **Notemd** | Mermaid/Canvas/Vega-Lite aus dem Notizinhalt |

**Empfohlene Kombination:** Smart Connections (Entdeckung) + Notemd (Strukturierung) + Copilot (Diskussion).
