# Diagramme erstellen

<TLDR>
Wählen Sie **welchen Diagrammtyp** Sie benötigen, anschließend das entsprechende **bearbeitbare Quellformat**. Prüfen Sie das Ergebnis in Obsidian und exportieren Sie es als SVG, PNG oder PDF.
</TLDR>

## Die drei verfügbaren Optionen verstehen

| Option | Was wird damit gesteuert | Beispiele |
|---|---|---|
| Diagrammtyp | Bedeutung des Diagramms | Flussdiagramm, Sequenzdiagramm, Schaltplan, Datenchart |
| Quellformat | Das bearbeitbare oder wiederverwendbare Dateiformat, das von Notemd erstellt wird | Mermaid, JSON Canvas, Draw.io, Drawnix, CircuitikZ |
| Exportformat | Das Bild oder Dokument, das zum Teilen verwendet wird | SVG, PNG, PDF |

Draw.io und Drawnix sind **Quellformate**, keine Diagrammtypen. CircuitikZ ist ein Quellformat, das speziell für Schaltpläne entwickelt wurde. Dasselbe Vorschaubild kann in mehrere visuelle Formate exportiert werden, daher beschränkt die Auswahl von CircuitikZ Sie nicht auf SVG.

## Ein Quellformat auswählen

| Format | Am besten geeignet für | Quelldatei |
|---|---|---|
| Mermaid | Diagramme, die direkt im Markdown gespeichert werden | Mermaid-Codeblock |
| JSON Canvas | Workflow‑Modelle, die direkt in Obsidian verwendet werden | `.canvas` |
| Draw.io | Spätere Bearbeitung in diagrams.net oder kompatiblen Tools | `.drawio` |
| Drawnix | Handgezeichnete oder Whiteboard‑ähnliche Bearbeitungen | `.drawnix` |
| CircuitikZ | Strukturierte elektrische Schaltungen und Transistorschaltungen | `.tex` |

Lassen Sie **Preferred source format** auf **Auto** stehen, wenn Sie kein bestimmtes Editor‑Tool benötigen. Notemd wählt dann automatisch ein kompatibles Format für den ausgewählten Diagrammtyp aus.

## Ein Diagramm erstellen und exportieren

1. Öffnen Sie die Notiz oder wählen Sie den Text aus, den Sie visualisieren möchten.
2. Wählen Sie **Generate diagram** im Seitenbereich oder in der Befehlsleiste von Notemd aus.
3. Wählen Sie einen **Diagrammtyp** aus oder lassen Sie ihn auf **Auto-detect** stehen.
4. Wählen Sie ein **Quellformat** aus oder lassen Sie es auf **Auto** stehen.
5. Prüfen Sie die Vorschau.
6. Wählen Sie **Save source file**, **Export SVG**, **Export PNG** oder **Export PDF**.

PNG und PDF verwenden die in den Notemd‑Einstellungen konfigurierte Bildauflösung (PPI). SVG bleibt ein Vektorbild und wird nicht von diesem PPI‑Wert beeinflusst.

## Schaltpläne

Für unterstützte Schaltplananfragen wählen Sie **Circuit diagram** zusammen mit einer **CircuitikZ Quelldatei**. Zunächst erstellt Notemd eine strukturierte Beschreibung des Schaltplans und schreibt anschließend die CircuitikZ‑Quelle. Dies ist zuverlässiger, als vom Modell eine unbeschränkte LaTeX‑Datei erstellen zu lassen.

Die aktuelle Unterstützung für CircuitikZ ist absichtlich eingeschränkt. Am besten funktionieren allgemein verfügbare NMOS‑Verstärker sowie unterstützte CMOS‑Bausteine. Überprüfen Sie immer die Vorschau und die Quelle, bevor Sie einen generierten Schaltplan in technischen Arbeiten oder Publikationen verwenden.

Die SVG‑Vorschau wird von Notemd als Hilfsmittel zur Überprüfung erstellt. Sie ist kein Nachweis dafür, dass eine lokale LaTeX- oder TikZJax‑Installation die `.tex`-Datei erfolgreich kompiliert hat.

## Fehlerbehebung

### Das Ergebnis ist der falsche Diagrammtyp

Wählen Sie den Diagrammtyp explizit aus, anstatt auf **Auto-detect** zu setzen. Für Schaltpläne wählen Sie **Circuit diagram** und verwenden Sie den Befehl **Generate diagram**, nicht mehr den veralteten Befehl **Summarise as Mermaid diagram**.

### Ein Quellformat eignet sich nicht

Setzen Sie das Quellformat wieder auf **Auto** oder wählen Sie ein Format, das für den jeweiligen Diagrammtyp geeignet ist. CircuitikZ dient ausschließlich für Schaltpläne; Draw.io und Drawnix sind für unterstützte strukturierte Diagramme vorgesehen.

### Die Export‑Schaltflächen fehlen

SVG, PNG und PDF erfordern eine visuelle Vorschau. Wenn nur der Quelltext vorhanden ist, speichern Sie die Quelldatei und öffnen Sie sie in dem entsprechenden externen Tool.

### Eine CircuitikZ-Datei lässt sich extern nicht kompilieren

Stellen Sie sicher, dass Ihre LaTeX‑Umgebung das Paket `circuitikz` enthält. Der Plugin enthält keine vollständige LaTeX‑Distribution.
