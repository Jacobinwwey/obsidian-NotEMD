# Konfiguration

<TLDR>
**Notemd Die Konfiguration umfasst 6 Hauptbereiche:** (1) LLM-Anbieter und API-Schlüssel, (2) Auswahl des Modells pro Aufgabe, (3) Ausgabepfade und Vorlagen, (4) UI-Sprache und Ausgabesprache, (5) Suchdienste (Tavily/DuckDuckGo), (6) Benutzerdefinierte Workflows. Alle Einstellungen befinden sich in Obsidian Einstellungen → Notemd. Änderungen treten sofort in Kraft.
</TLDR>

## Überblick über Einstellungen

Zugriff: **Obsidian Einstellungen (⚙️) → Notemd**

### Hauptabschnitte

1. **Anbieterkonfiguration** — API-Schlüssel, Basis URLs, Modelle
2. **Aufgaben-Einstellungen** — Modellauswahl pro Aufgabe
3. **Ausgabekonfiguration** – Wo die generierten Dateien gespeichert werden sollen
4. **Spracheinstellungen** — UI und Ausgabesprachen
5. **Suchintegration** – Einrichtung von Tavily/DuckDuckGo
6. **Workflows** – Benutzerdefinierte Schaltflächenketten
7. **Erweitert** — Diagnose, Batch-Verarbeitung, Chunking

---

## 1. Konfiguration des Anbieters

### Einen Anbieter hinzufügen

1. **Anbieter auswählen** aus dem Dropdown-Menü (mehr als 30 verfügbar)
2. **Drücken Sie die API-Taste**
3. **Optional**: Legen Sie eine benutzerdefinierte Basis URL fest (für Proxye/Gateways)
4. Klicken Sie auf „Modellliste anfordern“, um die Verbindung zu überprüfen
5. **Standardmodell auswählen**

### Gängige Anbieter

#### OpenAI
```
Provider: OpenAI
API Key: sk-...
Base URL: https://api.openai.com/v1 (default)
Model: gpt-4o-mini
```

#### Anthropic (Claude)
```
Provider: Anthropic
API Key: sk-ant-...
Model: claude-sonnet-4-6
```

#### Ollama (Lokal)
```
Provider: Ollama
Base URL: http://localhost:11434 (default)
Model: llama3.1
API Key: (not required)
```

#### DeepSeek
```
Provider: DeepSeek
API Key: sk-...
Model: deepseek-chat
```

### Mehrere Anbieter

Sie können mehrere Anbieter konfigurieren und je nach Aufgabe zwischen ihnen wechseln:
- **Hauptanbieter**: Wird als Ersatz verwendet
- **Aufgabenbezogene Anbieter**: Überschreiben für spezifische Aufgaben

## 2. modell spezifisch für Aufgaben

Konfigurieren Sie verschiedene Modelle für unterschiedliche Aufgaben, um Kosten und Qualität zu optimieren:

### Aufgabentypen

| Aufgabe | Standardmodell | Zweck |
|------|--------------|---------|
| **Verlinken** | Hauptanbieter | Identifiziere Konzepte für Wiki-Links |
| **Konzepterstellung** | Hauptanbieter | Konzepte extrahieren und beschreiben |
| **Forschung** | Hauptanbieter | Zusammenfassung der Web-Suchergebnisse |
| **Übersetzung** | Hauptanbieter | Übersetzen Sie den Text |
| **Diagrammgenerierung** | Hauptanbieter | Erstelle Mermaid/Canvas-Diagramme |

### Beispiel: kosteneffiziente Einrichtung

```
Linking: gpt-4o-mini (fast, many calls)
Concept Generation: claude-sonnet-4-6 (quality extraction)
Research: gpt-4o (web-grounded)
Translation: gemini-flash (multilingual)
Diagram: ollama/llama3.1 (free, local)
```

### Wie konfiguriere ich

1. **Einstellungen → Notemd → Aufgabeneinstellungen**
2. Für jede Aufgabe wählen Sie aus:
   - ✅ **Verwenden Sie den Hauptanbieter** (Standard)
   - 🔄 **Benutzerdefinierter Anbieter verwenden** → Anbieter + Modell auswählen

## 3. Ausgabekonfiguration

### Ausgabe der Konzeptnotizen

**Wo neue Konzeptnotizen gespeichert werden:**

```
Settings → Output → Concept Folder
Default: concepts/
```

**Vorlage:**
```
Settings → Output → Concept Template
Default:
---
tags: [concept]
created: {{date}}
---

# {{concept}}

{{description}}

## Backlinks
{{backlinks}}
```

### Optionen für die Ordnerstruktur

**Option 1: Flach (alle Konzepte in einer einzigen Ordner)**
```
vault/
└── concepts/
    ├── Machine Learning.md
    ├── Neural Networks.md
    └── Deep Learning.md
```

**Option 2: Hierarchisch (nach Quellordner)**
```
vault/
├── papers/
│   └── concepts/
│       └── Transformer.md
└── notes/
    └── concepts/
        └── Attention Mechanism.md
```

Konfigurieren: **Einstellungen → Ausgabe → Hierarchische Ordner verwenden**

### Diagramm-Ausgabe

**Wo Diagramme gespeichert werden:**
```
Settings → Output → Diagram Folder
Default: diagrams/
```

**Format:**
- `.mmd` (Mermaid Quelle)
- `.canvas` (JSON Canvas)
- `.html` (Vega-Lite)

## 4. Spracheinstellungen

### UI Sprache

**Sprache der Benutzeroberfläche** (Schaltflächen, Menüs, Dialoge):

```
Settings → Language → UI Language
Options: 21+ languages
Default: System language
```

Unterstützt: Englisch, 简体中文, 繁體中文, 日本語, 한국어, Español, Français, Deutsch, Русский, العربية und weitere Sprachen.

### Ausgabesprache

**Sprache der generierten Inhalte** (Konzeptnotizen, Forschungszusammenfassungen):

```
Settings → Language → Output Language
Options: Same as UI languages
Default: Same as UI
```

**Anwendungsfall:** Chinesisch UI + englische Ausgabe für zweisprachige Forschung.

### Übersetzungsziel

**StandardzielSprache** für Übersetzungsaufgaben:

```
Settings → Translation → Target Language
Default: English
```

## 5. Suchintegrierung

### Tavily (Empfohlen)

**AI-gestützter Suchdienst mit Optimierung:**

```
Settings → Search → Service: Tavily
API Key: tvly-...
```

Schlüssel abrufen: [tavily.com](https://tavily.com/)

**Eigenschaften:**
- Hochwertige Ergebnisse
- Von KI optimierte Zusammenfassungen
- Quellenangaben

### DuckDuckGo (Experimentell)

**Kein API-Schlüssel erforderlich:**

```
Settings → Search → Service: DuckDuckGo
```

**Hinweis:** Geringere Qualität, keine Anpassungsmöglichkeiten.

## 6. Konfiguration des Workflows

### Erstellen von benutzerdefinierten Workflows

**Workflows verknüpfen mehrere Aufgaben zu einer einzigen Schaltfläche.**

1. **Einstellungen → Workflows → Workflow hinzufügen**
2. **Name**: „Paper Analysis Flow“
3. **DSL**: `process-current-add-links>extract-concepts-current>research-and-summarize>summarize-as-mermaid`
4. **Speichern**

Ein neuer Button erscheint im linken Seitenbereich.

### DSL-Syntax

**Format:** `action1>action2>action3`

**Verfügbare Aktionen:**
- `process-current-add-links` — Wiki-Links zur aktuellen Notiz hinzufügen
- `extract-concepts-current` — Konzeptnotizen erstellen
- `research-and-summarize` — Web-Recherche + Zusammenfassung
- `translate-current` — Übersetze die aktuelle Notiz
- `summarize-as-mermaid` — Erstelle ein Mermaid-Diagramm
- `process-folder-add-links` — Batch-Verarbeitung von Ordnern

**Beispiele für Arbeitsabläufe:**

**Schnelle Konzeptextraktion:**
```
process-current-add-links>extract-concepts-current
```

**Tiefergehende Recherche:**
```
process-current-add-links>research-and-summarize>extract-concepts-current
```

**Mehrsprachige Analyse:**
```
translate-current>process-current-add-links>extract-concepts-current
```

## 7. Erweiterte Einstellungen

### Batchverarbeitung

**Mehrere Dateien parallel verarbeiten:**

```
Settings → Advanced → Batch Processing
Concurrency: 3 (default)
```

Höher = schneller, aber mehr API Aufrufe.

### Intelligente Teilung

**Für lange Notizen: Teilen Sie sie in Abschnitte auf:**

```
Settings → Advanced → Smart Chunking
Chunk Size: 2000 words (default)
Overlap: 200 words (default)
```

Verhindert Fehler aufgrund der Token-Limitierung.

### Diagnosepanel

**Detaillierte Protokolle ansehen:**

```
Settings → Advanced → Enable Diagnostics
```

Anzeige:
- API Anfragen/Antworten
- Tokenverwendung
- Verarbeitungsdauer
- Fehler

### Erkennung doppelter Konzepte

**Vermeiden Sie nahezu identische Konzepte:**

```
Settings → Advanced → Synonym Suppression
Threshold: 0.8 (default, 0-1)
```

Höher = aggressivere Zusammenführung.

## Konfigurationsprofile

### Einstellungen für Export/Import

**Konfigurationen über Geräte hinweg teilen:**

1. **Einstellungen → Erweitert → Konfiguration exportieren**
2. Speichert `notemd-config.json`
3. Auf neuem Gerät: **Konfiguration importieren**

**Was enthalten ist:**
- Provider API Schlüssel (optional)
- Modellauswahlen
- Ausgabepfade
- Benutzerdefinierte Workflows

## Fehlerbehebung der Konfiguration

### Einstellungen nicht gespeichert
- Überprüfe die Dateiberechtigungen in `.obsidian/plugins/notemd/`
- Versuchen Sie: Plugin deaktivieren → Wieder aktivieren

### API Schlüssel ungültig
- Keine zusätzlichen Leerzeichen
- Überprüfe Ablaufdatum
- Überprüfen Sie die Credits im Anbieter-Dashboard

### Modelle werden nicht geladen.
- Klicken Sie auf „Modellliste abrufen“, um sie zu aktualisieren
- Überprüfe die Netzwerkverbindung
- Versuchen Sie einen anderen Anbieter.
