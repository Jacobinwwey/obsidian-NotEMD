# 自定义 Prompt

<TLDR>
**Notemd 允许你为每一种任务类型覆盖默认 LLM prompt。** 自定义 prompt 能控制模型优先关注什么：你可以把概念抽取限制在某个学科，改变链接风格，或调整翻译语气。`{{content}}`、`{{concept}}` 等 prompt variables 会在运行时替换。focused learning domain 会把你的学习领域前置到 prompt 中，用来自动约束抽取结果。

这是 [Obsidian AI Knowledge Management Guide](/docs/pillar-ai-knowledge) 的一部分。
</TLDR>

## 概览

Notemd 内置 prompt 面向通用知识工作，跨领域表现稳定，但具体领域通常需要更强约束。自定义 prompt 让你明确告诉 LLM 应该如何处理当前任务，例如只抽取医学术语、只链接命名实体，或用正式语气翻译。

自定义 prompt 不会替换整个 system prompt。它覆盖或增强的是**任务特定指令部分**，同时保留 Notemd 解析所依赖的结构化输出格式，例如 `CONCEPT:` 行和 wiki-link 语法。

## 工作机制

### Prompt 架构

每个 Notemd 任务都遵循同一类 prompt 结构：

```text
[System role: You are a helpful assistant...]
[Domain constraint: Relevant Fields: [{domain}]]   <-- focused learning（可选）
[Custom task prompt]                                <-- 你的覆盖指令（可选）
[Output format instructions]                        <-- 始终保留
[Content: {{content}}]                              <-- 运行时替换
```

1. **Base system prompt** -- 设置 assistant persona，不能覆盖。
2. **Focused learning domain** -- 将 `Relevant Fields: [medicine]` 这类领域约束加到抽取 prompt 前。它是设置项，不是裸 prompt。
3. **Custom task prompt** -- 替换默认任务说明，例如抽取哪些概念、链接强度如何。
4. **Output format** -- Notemd 始终追加结构规则，保证响应可解析。
5. **Content** -- 真实笔记内容，通过 `{{content}}` 在运行时替换。

### Prompt 变量

变量会在执行时替换。不同任务支持的变量不同：

| Variable | Tasks | Replaced With |
|----------|-------|---------------|
| `{{content}}` | All | 当前笔记或选区文本 |
| `{{concept}}` | Concept note template | 抽取出的概念名 |
| `{{description}}` | Concept note template | LLM 生成的概念描述 |
| `{{backlinks}}` | Concept note template | 带链接的来源笔记列表 |
| `{{date}}` | Concept note template | 当前日期（YYYY-MM-DD） |
| `{{language}}` | Translation | 目标输出语言 |
| `{{source}}` | Concept note template | 来源笔记 basename |

### 聚焦学习领域

focused learning 是领域约束的快捷方式。你不需要在每个自定义 prompt 中重复写领域，只需要在设置里配置 `focusedLearningDomain`：

```text
Settings -> Advanced -> Focused Learning Domain
Example: "medicine" or "computer science" or "Chinese philosophy"
```

这会把 `Relevant Fields: [medicine]` 加到抽取 prompt 前，让 LLM 优先保留领域相关概念，并压制过于泛化的名词。它比完整自定义 prompt 更轻量，并可覆盖抽取类任务。

## 配置

| Setting | Default | Effect |
|---------|---------|--------|
| `customAddLinksPrompt` | `''` | 覆盖 add-links 任务 prompt |
| `customExtractConceptsPrompt` | `''` | 覆盖概念抽取 prompt |
| `customResearchPrompt` | `''` | 覆盖 research + summarize prompt |
| `customTranslatePrompt` | `''` | 覆盖翻译 prompt |
| `customDiagramPrompt` | `''` | 覆盖图表生成 prompt |
| `focusedLearningDomain` | `''` | 给抽取和链接 prompt 前置学科上下文 |
| `extractConceptsMinimalTemplate` | `true` | 使用 title-only 概念笔记模板 |
| `customConceptNoteTemplate` | `''` | 用完整变量支持覆盖概念笔记模板 |

当自定义 prompt 字段为空时，Notemd 使用内置默认 prompt。

## 示例

你是医学研究者，正在从临床试验笔记中抽取概念。默认 prompt 可能会抽出医学术语以外的普通名词，导致概念文件夹变乱。

**Solution:** 设置 focused learning domain，并配置自定义抽取 prompt。

1. 将 `focusedLearningDomain` 设为 `medicine`
2. 将 `customExtractConceptsPrompt` 设为：

```text
Extract only clinically relevant concepts: diseases, drug names,
anatomical terms, measurement endpoints, and statistical methods.
Ignore generic English nouns, brand names, and institutional names.
Prioritize MeSH terminology where applicable.
```

对临床笔记运行 "Extract concepts" 后，LLM 会输出：

```text
CONCEPT: Randomized Controlled Trial
CONCEPT: Double-Blind Method
CONCEPT: Myocardial Infarction
CONCEPT: Creatine Kinase-MB
CONCEPT: Kaplan-Meier Estimate
```

而不是同时抽出 "hospital"、"month" 这类无用泛词。

## 建议

- **先从 focused learning domain 开始** -- 很多场景只需要一个领域设置，不必写完整 prompt。
- **保留输出格式** -- 不要删掉 "one per line" 或 `CONCEPT:` prefix 这类格式要求；Notemd parser 依赖它们。
- **先用单篇笔记测试** -- 自定义 prompt 可能改变抽取宽度，批处理前先验证一篇。
- **保持 prompt 简洁** -- 过长 prompt 会消耗 token，也会稀释模型关注点。
