# LLM Providers

<TLDR>
**Notemd 支持 36 个 LLM providers，覆盖 4 类 transport。** 所有 API 调用都使用原始 HTTP，不依赖 SDK；支持自动重试、transport fallback 和 5 分钟响应缓存。按任务选择模型可以让简单任务使用便宜模型，研究和图表等复杂任务使用更强模型。本地 providers（Ollama、LMStudio）不需要 API key。
</TLDR>

这是 [Obsidian AI 知识管理指南](../pillar-ai-knowledge) 的一部分。

## Per-Task Model Selection

Notemd 支持为每个任务独立选择 provider 和模型。开启方式：`useMultiModelSettings: true`。

| 任务 | Settings key | 默认 Provider |
|---|---|---|
| Add Links | `addLinksProvider` / `addLinksModel` | DeepSeek |
| Research | `researchProvider` / `researchModel` | DeepSeek |
| Generate from Title | `generateTitleProvider` / `generateTitleModel` | DeepSeek |
| Translation | `translateProvider` / `translateModel` | DeepSeek |
| Diagram Generation | `summarizeToMermaidProvider` / `Model` | DeepSeek |
| Concept Extraction | `extractConceptsProvider` / `Model` | DeepSeek |
| Original Text Extraction | `extractOriginalTextProvider` / `Model` | DeepSeek |

### 解析顺序

1. 如果开启 `useMultiModelSettings`，使用任务级 provider + model
2. 如果任务模型为空，回退到 provider 默认模型
3. 如果任务 provider 无效，回退到全局 `activeProvider`
4. 如果关闭 `useMultiModelSettings`，所有任务使用全局 `activeProvider`

### 推荐组合

```text
# 研究者：研究任务用强模型，其余任务用低成本模型
activeProvider: DeepSeek
extractConceptsProvider: DeepSeek
researchProvider: OpenAI
summarizeToMermaidProvider: DeepSeek

# 隐私优先：全本地
activeProvider: Ollama
addLinksProvider: Ollama
extractConceptsProvider: Ollama

# 双语预算方案
activeProvider: DeepSeek
translateProvider: Google
researchProvider: SiliconFlow
```

## API Call Architecture

### Transport Layers

1. **Obsidian `requestUrl`**：主路径，所有平台可用
2. **Desktop HTTP**：Node.js `http` / `https`，适合 streaming 和长请求
3. **Web `fetch`**：Node 模块不可用时的 fallback

插件会在临时网络错误时自动切换 transport。

### Retry Logic

- **Normal mode**：单次请求；临时失败时自动升级到 stable retry
- **Stable retry mode**：最多 4 次尝试，间隔 5 秒
- 会触发 retry 的临时错误：`err_connection_closed`、`err_connection_reset`、`err_timed_out`、`econnreset`、`socket hang up`、`network request failed`
- 不重试：HTTP 400、401、403、404

### Response Caching

5 分钟内存 LRU 缓存，key 由 provider、model、temperature、prompt 和 content 组成。相同请求在 5 分钟内会直接返回缓存结果。

### Reasoning Model Handling

不支持 `system` role 的模型（DeepSeek R1、OpenAI o1/o3/o4/gpt-5）会把 system + user prompt 合并到一个 user message。DeepSeek thinking 开启时会加 `{ thinking: { type: 'enabled' } }`，并校验最大输出 token 不低于 8000。

### Token Estimation

`estimateTokens(text)` 使用 `Math.ceil(text.length / 4)`，按 4 字符约等于 1 token 估算。它用于在请求前控制 `maxResearchContentTokens`。

## Quick Start

1. **选择 provider**：DeepSeek、OpenAI 或 Ollama
2. **输入 API key**：Settings → Notemd → Provider → API Key
3. **测试连接**：点击 test button
4. **可选：按任务选择模型**：开启 `useMultiModelSettings`

Ollama 不需要 API key，只要确认 `ollama serve` 正在运行。
