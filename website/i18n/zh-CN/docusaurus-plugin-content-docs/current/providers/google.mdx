# Google 提供商

<TLDR>
**Google 使用的是原生的 Gemini 生成语言 API 传输机制。** 当前的预设默认为 `gemini-2.0-flash-exp`、`https://generativelanguage.googleapis.com/v1` 和 `temperature: 0.5`。它不是兼容 OpenAI 的配置文件。

这是[Obsidian AI知识管理指南](/docs/pillar-ai-knowledge)的一部分。
</TLDR>

## 设置

创建一个具有 Gemini 访问权限的 Google AI Studio 或 Google 云 API 密钥，添加一个 `Google` 提供商配置文件，并保留原生的生成式语言基础 URL。使用默认的 Gemini Flash 配置文件以实现快速翻译和广泛的笔记处理功能，而在需要更强大或功能更完善的 Gemini 模型时，则可针对特定任务覆盖该模型。

除非您明确使用的是能够通过 OpenAI 兼容的 API 接口来访问 Gemini 的网关，否则请避免在 OpenAI 兼容的配置文件中放置 Gemini 模型 ID。

## 端点与身份验证

| 字段 | 当前预设 |
|-------|----------------|
| 传输 | `google` |
| API 键 | 必需 |
| 基础 URL | `https://generativelanguage.googleapis.com/v1` |
| 默认模型 | `gemini-2.0-flash-exp` |
| 温度 | `0.5` |
| 连接测试 | 仅聊天模式 |

运行时调用了 `/models/{model}:generateContent?key={apiKey}`。该键作为查询参数传递，因为插件中的原生 Gemini 接口就是以这种方式连接的。

## 模型发现

Google 使用 `google-models` 发现模式。该发现功能应返回对 API 密钥可用的 Gemini 模型；如果选择器为空，则需使用相同的 Base URL 对该密钥进行测试，并确认该账户已启用相应的模型系列。

在提供者注册表中，已知的输出token上限与模型类型相关。如果长篇的翻译或图表说明被截断，请选择支持更高输出上限的Gemini模型，或降低所需的输出长度。

## 故障排除

- `400`：模型编号通常与原生 Gemini 路由不匹配，或者请求的格式超出了该模型支持的规格。
- `401` 或 `403`：验证密钥限制、项目/API 的启用状态，以及该密钥是否能够访问 Gemini。
- `404`：在配置的 API 版本中无法获取模型编号。
- 使用此配置文件时，网关示例会失败：请改用预置的网关，而非默认的 Google 提供商。

## 何时使用

在需要快速翻译、大量笔记整理，以及 Gemini 的成本/性能比优于直接使用 OpenAI 或 Anthropic 的工作流中，可使用 Google。而对于那些需要支持 OpenAI 格式推理模型的工作流，则建议选择其他服务提供商。
