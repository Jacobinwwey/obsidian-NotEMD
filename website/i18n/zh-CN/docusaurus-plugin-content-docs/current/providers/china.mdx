# 中国供应商

<TLDR>
**Notemd** 通过主要兼容 OpenAI 的端点，支持中国地区及中国境内的服务提供商。目前的列表包括 DeepSeek、Qwen、Qwen Code、Doubao、Moonshot、Xiaomi MiMo、GLM、MiniMax、Baidu Qianfan、SiliconFlow、Huawei Cloud MaaS 和 Z AI。其优势在于覆盖范围广且成本可控，而风险则在于服务提供商特定的模型命名方式及账户访问限制。

这是[Obsidian AI知识管理指南](/docs/pillar-ai-knowledge)的一部分。
</TLDR>

## 设置

创建提供商账户，生成一个 API 密钥，然后选择对应的 Notemd 提供商预设，而非使用通用的 OpenAI 配置文件。该预设包含默认的 Base URL、模型、API 测试模式、模型发现行为以及任何特定于提供商的 UI 字段。

在此应积极采用按任务选择模型的方式。例如，在需要大量推理的工作流中使用 DeepSeek 或 GLM，在处理广泛的中文内容时使用 Qwen 或 Moonshot，而当相关账户支持编码模型时，则将编码任务分配给 Qwen Code。

这些提供商中的大多数都使用兼容 OpenAI 的共享运行时，但它们的账户和模型语义有所不同。除非您已确认该账户的备用端点，否则请保留特定于预设的 Base URL。

## 模型发现

当提供方支持列表功能时，DeepSeek、Qwen、Qwen Code、Doubao、Moonshot、Xiaomi MiMo、GLM、Z AI、MiniMax、Baidu Qianfan 和 SiliconFlow 会使用与 OpenAI 兼容的模型发现机制。由于 ModelArts 的部署元数据与通用的 `/v1/models` 列表并不相同，因此 Huawei Cloud MaaS 会采用 `huaweicloud-modelarts-models` 模式。

不要将发现失败视为聊天不可行的证据。有些服务提供商在限制模型列表权限的情况下仍允许聊天。在这种情况下，应在配置文件中明确指定模型ID，并通过连接测试进行验证。

## 故障排除

- 认证成功但聊天失败：模型名称可能超出了该账户的权限范围或所在区域。
- 使用普通模型名称时 Doubao 会失败：如果部署有要求，请在模型字段中使用 Ark 端点 ID。
- 推理响应被截断：请检查提供商规定的最大输出token数以及Notemd所对应的模型感知型最大token设置。
- 区域网络配置各不相同：如果保险库机器位于中国境内，则应使用提供商的中国的端点；只有当账户和网络路径均有效时，才可使用国际端点。

## 何时使用

当区域延迟、支付/账户访问、中文内容质量或成本是主要限制因素时，应使用中国境内的服务提供商。如果需要更简单的协议结构或更高的跨服务商调试一致性，建议选择直接的 OpenAI、Anthropic、Google 或本地服务提供商。
