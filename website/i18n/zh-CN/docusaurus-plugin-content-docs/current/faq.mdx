## 安装与设置

### 如何在 Obsidian 中安装 Notemd？

打开 Obsidian **设置 → 社区插件 → 浏览** → 搜索 **“Notemd"** → **安装** → **启用**。

**要求：**
- Obsidian 0.15.0+
- 在桌面端（Windows、macOS、Linux）和移动端（iOS、Android）上均支持。

**手动安装：**
1. 从 [GitHub Releases](https://github.com/Jacobinwwey/obsidian-NotEMD/releases) 下载 `main.js`、`styles.css`、`manifest.json`
2. 把它们放到 `<your-vault>/.obsidian/plugins/notemd/` 中。
3. 重新加载 Obsidian 并启用该插件

**统计信息：** 下载量超过4千次，共发布44个版本，采用MIT许可证。

**网关：**

- Hugging Face，Vercel AI Gateway，Requesty

### Notemd 能与本地的 LLMs 配合使用吗？

**是的，完全支持。** Notemd 可以与以下内容配合使用：
- **Ollama**（原生集成）
- **LM Studio**（通过 OpenAI 兼容的端点）
- **vLLM、LocalAI、text-generation-webui**（通过 OpenAI 兼容模式）

所有任务均使用本地模型完成：链接、概念生成、翻译、研究以及图表生成。

**混搭示例：**
- Ollama（本地）用于链接 → 速度快、私密且免费
- 用于研究的 OpenAI → 高质量，支持网页搜索集成
- Gemini翻译功能 → 多模态支持

### 如何为不同的任务配置不同的模型？

在**Notemd 设置**中，每个任务都有独立的提供者/模型选择器：

| 任务 | 示例配置 |
|------|----------------------|
| **链接** | DeepSeek（快速、便宜） |
| **概念生成** | Claude Opus（高质量版） |
| **研究** | Perplexity 或 GPT-4o（基于网络的版本） |
| **翻译** | Gemini Flash（多模态） |
| **图表生成** | Ollama（本地，私有） |

这种按任务进行的优化能够在速度、成本和质量之间取得平衡。

## 核心功能

### 在阅读纸质文档方面，Notemd 与 ChatGPT 有什么区别？

| 尺寸 | ChatGPT | Notemd |
|-----------|---------|--------|
| **输出位置** | 聊天记录（临时） | Obsidian vault（持久化文件） |
| **结果格式** | 文本回复 | 维基链接、概念说明、图表、翻译内容 |
| **长期价值** | 会话结束后消失 | 累积到知识库中 |
| **可重用性** | 必须重新查询 | 文件保留，链接不变，持续演进 |

**使用 ChatGPT 的场景：** 快速问答、一次性解释
**请使用 Notemd 来：** 在数周或数月的时间里构建一个持久且结构化的知识库

### Notemd 是如何为笔记添加维基链接的？

**处理流程：**
1. 你在笔记上运行了“处理文件（添加链接）”功能。
2. Notemd 会将文本发送到您配置的 LLM。
3. LLM 用于标识上下文中的关键概念
4. Notemd 会在每个概念出现的位置插入 `[[wiki-links]]`
5. 可选地，它会在您指定的文件夹中自动创建概念笔记。

**功能特性：**
- 同义词抑制（避免“transformer”与“transformers”的重复）
- 从概念说明到来源的反向链接
- 重命名/删除时的链接完整性
- 纯提取模式（提取概念而不修改原文）

### 什么是 Notemd 一键工作流？

**一键工作流**可将多个操作串联到一个**单个侧边栏按钮**中。

**默认工作流：** `One-Click Extract`
```
process-current-add-links > extract-concepts-current > research-and-summarize > summarize-as-mermaid
```

**结果：** 点击一次即可运行：
1. 在当前笔记中添加维基链接
2. 为新术语生成概念说明。
3. 研究并补充背景信息
4. 压缩为 Mermaid 图表

**自定义工作流：** 可以通过 DSL 格式在设置中定义自己的工作流。每个工作流都会变成一个可重复使用的侧边栏按钮。

### Notemd与Copilot或Smart Connections相比如何？

这些插件可以解决不同的问题：

- **Copilot** — 以对话为核心。您可以与人工智能讨论您的笔记，人工智能的回复会显示在聊天面板中，而不会修改您的存储库。
- **Smart Connections** — 以发现为核心。它通过嵌入技术查找语义上相似的笔记，不会创建或修改内容。
- **Notemd** — 先撰写。AI会处理您的笔记，并将结构化的结果（维基链接、概念说明、研究资料、图表、翻译内容）直接作为持久性文件保存到您的存储库中。

| 你需要什么 | 推荐插件 |
|----------------|-------------------|
| 与AI聊聊笔记 | Copilot |
| 查找相似/相关的笔记 | Smart Connections |
| 自动插入维基链接并构建知识图谱 | **Notemd** |
| 生成包含反向链接的概念说明文档 | **Notemd** |
| 根据笔记内容创建图表 | **Notemd** |
| 研究一个主题并附上总结 | **Notemd** |

这三个可以一起使用——它们是互补关系，而非竞争关系。

## 研究与翻译

### 哪些搜索服务可以与 Notemd 研究功能配合使用？

| 服务 | API 需要密钥 | 质量 | 备注 |
|---------|-----------------|---------|-------|
| **Tavily** | 是的 | 高 | 推荐：经过人工智能优化的搜索功能 |
| **DuckDuckGo** | 不行 | 实验性版本 | 无需设置，质量较低 |

**“研究与总结”功能的工作原理：**
1. 您选择一个主题或在其当前笔记上运行它。
2. Notemd 向搜索服务发起查询
3. 结果 → LLM 摘要
4. 摘要已附在您的笔记中，并标注了来源引用

**结果：** 研究记录会保存在您的保险库中，而非浏览器标签页里。

### Notemd 是否支持中文或其他语言？

**UI 支持的语言：** 21种以上
- 英语, 中文 (简体), 中文 (繁體), 日本语, 韩国语
- 阿拉伯语、德语、法语、西班牙语、俄语、葡萄牙语、意大利语
- 印地语、孟加拉语、泰语、越南语、土耳其语、波兰语、捷克语等更多语言

**任务输出语言：** 独立配置
- UI 的中文翻译 + 英文输出：✅
- 英语中的UI + 中文输出：✅

**使用场景：** 双语研究人员在为出版物生成英文笔记时，可以保留 UI 的原文语言版本。

## PDF与学术工作流程

### 我可以使用 Notemd 来进行纸质/PDF 阅读吗？

**是的。** 工作流：
1. 使用 [MinerU](https://github.com/opendatalab/MinerU) 将 PDF **转换为 Markdown**（适用于学术论文）
2. 在 Obsidian 中打开 Markdown 文件
3. 运行 Notemd 任务：
   - 在关键术语中**添加维基链接**
   - **为不熟悉的术语生成概念说明**
   - 通过自定义问题**提取原文内容**（例如：“有哪些局限性？”）
   - 引用方法/数据集的**研究背景**
   - **翻译**难懂的部分
   - **压缩为 Mermaid** 图表以便理解结构

**版本 1.9.1+：** 支持目录功能的章节结构提取功能

**结果：** 您的论文阅读内容将转化为存储在您知识库中的结构化、可链接的知识资源。

## 图表与可视化

### Notemd 能生成哪些类型的图表/图示？

| 输入类型 | 支持 | 功能特性 |
|------|---------|----------|
| **Mermaid** | 完整版 | 流程图、时序图、类图、状态图、实体关系图、甘特图、语法自动修正、批量验证 |
| **JSON Canvas** | 完整版 | Obsidian 原生画布布局 |
| **Vega-Lite** | 完整版 | 数据图表、时间序列图、柱状图、散点图，可自定义 |
| **HTML** | 备用方案 | 对于不支持的图表类型 |
| **可编辑的 HTML/SVG** | 明确的目标 | 自带功能的内联 SVG，带有语义化的 Draw.io 风格注释 |
| **Draw.io / Drawnix 构建产物** | 导出边界 | 基于 `SemanticFigureModel` 的确定性 XML/JSON；无需编辑器运行时依赖 |

**Mermaid 自动修复：** 深度调试模式进行修复：
- 损坏的箭头（`-->`, `-.->`, `==>`）
- 标签无效，注释错误
- 子图语法错误
- 形状标记（`[`, `(`, `{`, `((`）
- 标签中的管道字符
- 注意附件

**工作流程：**
1. 在笔记上运行“Summarize as Mermaid diagram”命令。
2. 在模态框中预览
3. 导出为 SVG 或 PNG 格式
4. 或将其保存为保险库中的 `.mmd` 文件

## 故障排除

### 我的 LLM 调用失败了。我应该检查什么？

1. **API 密钥有效吗？** 请检查“设置”→“提供商配置”。
2. **模型名称正确吗？** 请在设置中使用“获取模型列表”按钮。
3. **可以访问网络吗？**某些提供商要求使用VPN。
4. **速率限制？**查看提供商控制面板。
5. **诊断信息：** 在设置中运行“连接测试”以获取详细的错误日志。

### Notemd 运行速度很慢。我该如何加快它的速度？

1. **使用更快的模型：** DeepSeek、Gemini Flash、GPT-4o-mini
2. **减小分块大小：** 设置 → 智能分块 → 降低字数限制
3. **并行处理：** 设置 → 批量处理 → 增加并发数
4. **本地模型：** 用于无需网络搜索的任务的 Ollama

### Notemd 能够完全离线运行吗？

**可以，使用 Ollama 即可。** 没有 API 密钥，无需网络连接，数据也不会离开您的机器：

1. 安装 [Ollama](https://ollama.com)，然后运行 `ollama serve`
2. 在 Notemd 设置中，选择 **Ollama** 作为您的服务提供商
3. 拉取模型：`ollama pull llama3`（或 qwen2.5、gemma2 等）
4. 所有功能均正常运行：wiki链接、概念笔记、翻译、图表

**DuckDuckGo研究**在没有 API密钥的情况下也能运行（效果为实验性水平）。

通过云服务提供商（OpenAI、Anthropic、Google、DeepSeek等），您的笔记内容会以API的形式被发送。请查看各提供商的隐私政策以了解数据处理细节。

### 我的数据会被发送到云端吗？

**这取决于您的服务提供商：**

| 提供商类型 | 数据被发送到云端了吗？ | 备注 |
|--------------|---------------------|-------|
| **Ollama（本地）** | 不行 | 所有内容都会保留在您的机器上。 |
| **LMStudio（本地）** | 不行 | 与 Ollama 相同 |
| **DeepSeek、OpenAI 等。** | 是的 | 请将内容作为 API 输入以供处理 |
| **OpenRouter, AIHubMix** | 是的 | 用于路由至云服务提供商的网关 |

Notemd 本身不会收集任何遥测数据、分析数据或用户数据。该插件是完全开源的（采用 MIT 许可证）。唯一会离开您设备的数据，就是您主动发送给已配置的 LLM 提供商的内容。

### 哪种 Obsidian AI 插件最适合用于研究？

这取决于您的研究工作流程：

| 需要 | 插件 | 为什么 |
|------|--------|-----|
| 查找相关论文/笔记 | Smart Connections | 语义相似性发现 |
| 聊聊你的信息来源吧 | Copilot | 与保险库上下文的对话 |
| 自动关联概念 + 生成笔记 | **Notemd** | AI识别概念，插入链接，创建笔记 |
| 网络调研 + 撰写总结笔记 | **Notemd** | Tavily/DuckDuckGo 搜索 → LLM 摘要 → 追加到笔记中 |
| 根据研究生成图表 | **Notemd** | 来自笔记内容的 Mermaid/Canvas/Vega-Lite |

**推荐组合：** Smart Connections（发现）+ Notemd（结构化）+ Copilot（讨论）。
