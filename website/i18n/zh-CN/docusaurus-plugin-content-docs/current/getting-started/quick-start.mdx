# 快速入门指南

<TLDR>
**5分钟内即可开始使用 Notemd。**只需三步：(1) 添加 LLM API 密钥（或使用本地 Ollama），(2) 打开笔记并运行“处理文件（添加链接）”功能，(3) 在侧边栏中体验一键工作流。支持30多种服务提供商，无需编写代码，搭配 Ollama 即可离线使用。
</TLDR>

## 步骤1：配置您的第一个 LLM 提供商（2分钟）

### 选项A：云服务提供商（OpenAI、Anthropic等）

1. 打开**设置** → **Notemd**
2. 在**LLM 提供商配置**下方，从下拉列表中选择一名提供商
3. 输入您的 **API 密钥**
4. （可选）点击“获取模型列表”以查看可用模型
5. 为每种任务类型选择一个模型

**热门选择：**
- **OpenAI**：`gpt-4o-mini`（快速、价格实惠）
- **Anthropic**：`claude-sonnet-4-6`（高质量）
- **DeepSeek**：`deepseek-chat`（价格非常实惠）

### 选项 B：本地 Ollama（无需 API 密钥）

1. 安装 [Ollama](https://ollama.com/)
2. 下载模型：`ollama pull llama3.1`
3. 在 Notemd 设置中，选择 **“Ollama"** 作为提供方
4. 模型名称：`llama3.1`（或您下载的任何模型）

## 第2步：尝试您的第一个任务（1分钟）

### 在笔记中添加维基链接

1. 打开任何包含有用内容的笔记
2. 在编辑器中右键点击
3. 选择**“处理文件（添加链接）”**
4. 稍等几秒。
5. 关键概念现在就是 `[[wiki-linked]]`！

**示例：**

**之前：**
```
Machine learning models use neural networks to process data.
```

**之后：**
```
[[Machine learning]] models use [[neural networks]] to process data.
```

## 第3步：探索功能（2分钟）

### 提取概念笔记

1. 右键点击 → **“从当前文件提取概念”**
2. Notemd 会为每个概念创建单独的文件。
3. 检查您配置的概念文件夹（默认路径：`concepts/`）
4. 每份概念说明都包含指向您原始说明的回链。

### 研究与总结

1. 在笔记中高亮显示某个主题
2. 右键点击 → **“研究与总结”**
3. Notemd 查询网络搜索（Tavily 或 DuckDuckGo）
4. 摘要功能会将来源信息附加到您的笔记中

### 一键工作流

1. 查看**左侧侧边栏**中的工作流按钮
2. 点击任意笔记上的**“一键提取”**
3. 它的运行流程为：添加链接 → 提取概念 → 进行研究 → 生成图表
4. 一键搞定！

## 常见的初始任务

### 任务1：处理论文/文章

```
1. Paste article content into a note
2. Run "Process file (add links)"
3. Run "Extract concepts"
4. Run "Research and summarize" on unfamiliar terms
5. Run "Summarize as Mermaid diagram" for visual structure
```

### 任务2：翻译笔记

```
1. Highlight text to translate
2. Right-click → "Translate to target language"
3. Set target language in settings first
```

### 任务3：创建知识图谱

```
1. Process 5-10 related notes with "add links"
2. Run "Extract concepts" on each
3. Open Graph view to see connections
4. Your knowledge graph is forming!
```

## 定制技巧

### 按任务选择模型

你可以为不同的任务使用不同的模型：

| 任务 | 推荐型号 | 为什么 |
|------|------------------|-----|
| 添加链接 | `gpt-4o-mini`，`deepseek-chat` | 快速，大量调用 |
| 提取概念 | `claude-sonnet-4-6`，`gpt-4o` | 质量提取 |
| 研究 | `perplexity`，`gpt-4o` | 基于网页的 |
| 翻译 | `gemini-flash`，`gpt-4o` | 多语言支持 |
| 图表 | `claude-opus`，本地模型 | 复杂推理 |

配置位置：**设置 → Notemd → 任务专用模型**

### 自定义工作流

创建你自己的按钮：

1. 设置 → Notemd → 工作流
2. 点击“添加工作流”
3. 输入 DSL：`process-current-add-links>extract-concepts-current>research-and-summarize`
4. 命名为：“我的研究流程”
5. 侧边栏中出现了一个新按钮。

## 故障排除

### “API 密钥无效”
- 检查键是否正确（不能有额外空格）
- 验证账户是否有余额
- 使用设置中的“连接测试”按钮进行测试

### “未找到模型”
- 点击“获取模型列表”即可查看可用的模型。
- 某些提供商要求使用特定的型号名称（例如 `gpt-4o`，而非 `gpt-4`）。

### 链接没有显示出来
- 确保笔记内容足够丰富（超过50个字）。
- 试试更强大的模型吧。
- 检查 LLM 调用是否成功（请查看诊断面板）

### 性能缓慢
- 使用更快的模型（`gpt-4o-mini` 而非 `gpt-4o`）
- 在设置中减小块大小
- 为批处理任务启用并行处理
