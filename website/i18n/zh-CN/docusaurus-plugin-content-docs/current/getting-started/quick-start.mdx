# 快速开始指南

<TLDR>
**5 分钟开始使用 Notemd。** 三步：配置 LLM API key（或本地 Ollama），打开笔记运行 "Process file (add links)"，再试用侧边栏一键工作流。支持 30+ providers，无需写代码；配合 Ollama 可离线运行。
</TLDR>

## 步骤 1：配置第一个 LLM Provider（2 分钟）

### 选项 A：云端 Provider（OpenAI、Anthropic 等）

1. 打开 **Settings** → **Notemd**
2. 在 **LLM Provider Configuration** 中选择 provider
3. 输入 **API Key**
4. 可选：点击 **"Get Model List"** 查看可用模型
5. 为不同任务选择模型

常见选择：

- **OpenAI**：`gpt-4o-mini`，速度快、成本低
- **Anthropic**：`claude-sonnet-4-6`，质量高
- **DeepSeek**：`deepseek-chat`，成本很低

### 选项 B：本地 Ollama（无需 API Key）

1. 安装 [Ollama](https://ollama.com/)
2. 下载模型：`ollama pull llama3.1`
3. 在 Notemd 设置中选择 **Ollama**
4. 模型名填写 `llama3.1`，或你已下载的其他模型

## 步骤 2：尝试第一个任务（1 分钟）

### 快速开始指南：核心说明

1. 打开一篇有实际内容的笔记
2. 在编辑器中右键
3. 选择 **"Process file (add links)"**
4. 等待几秒
5. 关键概念会变成 `[[wiki-links]]`

示例：

**处理前：**

```text
Machine learning models use neural networks to process data.
```

**处理后：**

```text
[[Machine learning]] models use [[neural networks]] to process data.
```

## 步骤 3：探索功能（2 分钟）

### 快速开始指南：核心说明

1. 右键 → **"Extract concepts from current file"**
2. Notemd 为每个概念创建独立文件
3. 查看配置的概念目录，默认是 `concepts/`
4. 每个概念笔记都会回链到原始笔记

### 研究并总结

1. 在笔记中选中一个主题
2. 右键 → **"Research and summarize"**
3. Notemd 调用 Tavily 或 DuckDuckGo
4. 摘要会带来源追加到当前笔记

### 一键工作流

1. 查看 **左侧边栏** 的工作流按钮
2. 在任意笔记上点击 **"One-Click Extract"**
3. 它会依次运行：add links → extract concepts → research → generate diagram
4. 一个按钮完成整条流程

## 常见首批任务

### 任务 1：处理论文或文章

```text
1. 把文章内容粘到一篇笔记
2. 运行 "Process file (add links)"
3. 运行 "Extract concepts"
4. 对陌生术语运行 "Research and summarize"
5. 运行 "Summarize as Mermaid diagram" 生成结构图
```

### 任务 2：翻译笔记

```text
1. 选中要翻译的文本
2. 右键 → "Translate to target language"
3. 先在设置里配置 target language
```

### 任务 3：创建知识图谱

```text
1. 用 "add links" 处理 5-10 篇相关笔记
2. 对每篇运行 "Extract concepts"
3. 打开 Graph view 查看连接
4. 知识图谱开始成形
```

## 快速开始指南：后续建议

### 按任务选择模型

不同任务可以使用不同模型：

| 任务 | 推荐模型 | 原因 |
|---|---|---|
| Add Links | `gpt-4o-mini`, `deepseek-chat` | 快，调用次数多 |
| Extract Concepts | `claude-sonnet-4-6`, `gpt-4o` | 概念抽取质量更重要 |
| Research | `perplexity`, `gpt-4o` | 需要可靠的 web-grounded 汇总 |
| Translation | `gemini-flash`, `gpt-4o` | 多语言能力好 |
| Diagrams | `claude-opus`, local model | 需要复杂推理 |

配置入口：**Settings → Notemd → Task-Specific Models**

### 自定义工作流

创建自己的按钮：

1. Settings → Notemd → Workflows
2. 点击 "Add Workflow"
3. 输入 DSL：`process-current-add-links>extract-concepts-current>research-and-summarize`
4. 命名为 "My Research Flow"
5. 侧边栏会出现新的按钮

## 故障排查

### 快速开始指南：配置与模型

- 检查 key 是否正确，尤其是前后空格
- 确认账号有额度
- 在设置里使用 "Connection Test"

### 快速开始指南：配置与模型

- 点击 "Get Model List" 查看可用模型
- 部分 provider 要求精确模型名，例如 `gpt-4o` 不是 `gpt-4`

### 快速开始指南：核心说明

- 确认笔记内容足够多，建议超过 50 词
- 换用能力更强的模型
- 检查 LLM 调用是否成功，可看 diagnostics panel

### 快速开始指南：故障排查

- 使用更快模型，例如 `gpt-4o-mini`
- 调小 chunk size
- 批处理时开启 parallel processing
