# 配置

<TLDR>
**Notemd**配置涵盖了6个主要方面：(1) LLM提供方和API密钥，(2) 每个任务的模型选择，(3) 输出路径与模板，(4) UI语言及输出语言，(5) 搜索服务(Tavily/DuckDuckGo)，(6) 自定义工作流。所有设置均在Obsidian的“设置”→“Notemd”中。更改会立即生效。
</TLDR>

## 设置概览

访问路径：**Obsidian 设置（⚙️）→ Notemd**

### 主要章节

1. **提供程序配置** — API 密钥、基础 URLs、模型
2. **任务设置** — 每个任务的模型选择
3. **输出配置** — 生成文件的保存位置
4. **语言设置** — UI 与输出语言
5. **搜索集成** — Tavily/DuckDuckGo 设置
6. **工作流** — 自定义按钮链
7. **高级** — 诊断、批量处理、分块处理

---

## 1. 提供商配置

### 添加提供程序

1. 从下拉列表中选择提供商（共有30多种可选）。
2. **输入 API 键**
3. **可选**：设置自定义基础地址 URL（用于代理/网关）
4. **点击“获取模型列表”**以验证连接状态
5. **选择默认模型**

### 常用提供商

#### OpenAI
```
Provider: OpenAI
API Key: sk-...
Base URL: https://api.openai.com/v1 (default)
Model: gpt-4o-mini
```

#### Anthropic（Claude）
```
Provider: Anthropic
API Key: sk-ant-...
Model: claude-sonnet-4-6
```

#### Ollama（本地）
```
Provider: Ollama
Base URL: http://localhost:11434 (default)
Model: llama3.1
API Key: (not required)
```

#### DeepSeek
```
Provider: DeepSeek
API Key: sk-...
Model: deepseek-chat
```

### 多个提供商

您可以配置多个提供者，并根据任务在它们之间切换：
- **主要提供商**：用作备用选项
- **特定任务提供程序**：为特定任务进行覆盖设置

## 2. 任务专用模型

为不同的任务配置不同的模型，以优化成本与质量：

### 任务类型

| 任务 | 默认模型 | 用途 |
|------|--------------|---------|
| **链接** | 主要提供商 | 确定维基链接的相关概念 |
| **概念生成** | 主要提供商 | 提取并描述相关概念 |
| **研究** | 主要提供商 | 总结网页搜索结果 |
| **翻译** | 主要提供商 | 翻译文本 |
| **图表生成** | 主要提供商 | 生成 Mermaid/Canvas 图表 |

### 示例：成本优化配置

```
Linking: gpt-4o-mini (fast, many calls)
Concept Generation: claude-sonnet-4-6 (quality extraction)
Research: gpt-4o (web-grounded)
Translation: gemini-flash (multilingual)
Diagram: ollama/llama3.1 (free, local)
```

### 如何进行配置

1. **设置 → Notemd → 任务设置**
2. 对于每个任务，请选择：
   - ✅ **使用主提供商**（默认）
   - 🔄 **使用自定义提供程序** → 选择提供程序和模型

## 3. 输出配置

### 概念说明输出

**新概念文档的保存位置：**

```
Settings → Output → Concept Folder
Default: concepts/
```

**模板：**
```
Settings → Output → Concept Template
Default:
---
tags: [concept]
created: {{date}}
---

# {{concept}}

{{description}}

## Backlinks
{{backlinks}}
```

### 文件夹结构选项

**选项1：扁平式（所有概念放在一个文件夹中）**
```
vault/
└── concepts/
    ├── Machine Learning.md
    ├── Neural Networks.md
    └── Deep Learning.md
```

**选项 2：分层结构（按源文件夹）**
```
vault/
├── papers/
│   └── concepts/
│       └── Transformer.md
└── notes/
    └── concepts/
        └── Attention Mechanism.md
```

配置：**设置 → 输出 → 使用分层文件夹**

### 图表输出

**图表保存位置：**
```
Settings → Output → Diagram Folder
Default: diagrams/
```

**格式：**
- `.mmd` (Mermaid 源码)
- `.canvas` (JSON Canvas)
- `.html` (Vega-Lite)

## 4. 语言设置

### UI 语言

**界面语言**（按钮、菜单、对话框）：

```
Settings → Language → UI Language
Options: 21+ languages
Default: System language
```

支持的语言包括：英语、简体中文、繁体中文、日语、韩语、西班牙语、法语、德语、俄语、阿拉伯语等。

### 输出语言

**生成内容语言**（概念说明、研究总结）：

```
Settings → Language → Output Language
Options: Same as UI languages
Default: Same as UI
```

**使用场景：** 中文 UI + 英文输出，用于双语研究。

### 翻译目标

翻译任务的**默认目标语言**：

```
Settings → Translation → Target Language
Default: English
```

## 5. 搜索集成

### Tavily（推荐）

**AI优化搜索服务：**

```
Settings → Search → Service: Tavily
API Key: tvly-...
```

获取密钥：[tavily.com](https://tavily.com/)

**功能特性：**
- 高质量的结果
- 经过 AI 优化的摘要
- 来源引用

### DuckDuckGo（实验性功能）

**无需 API 键：**

```
Settings → Search → Service: DuckDuckGo
```

**注意：** 图像质量较低，且不支持自定义。

## 6. 工作流配置

### 创建自定义工作流

**工作流可将多个任务串联到一个按钮中。**

1. **设置 → 工作流 → 添加工作流**
2. **名称**：“纸张分析流程”
3. **DSL**：`process-current-add-links>extract-concepts-current>research-and-summarize>summarize-as-mermaid`
4. **保存**

左侧侧边栏中出现了一个新按钮。

### DSL 语法

**格式：** `action1>action2>action3`

**可用操作：**
- `process-current-add-links` — 为当前笔记添加维基链接
- `extract-concepts-current` — 生成概念说明
- `research-and-summarize` — 网络调研 + 摘要
- `translate-current` — 翻译当前笔记
- `summarize-as-mermaid` — 生成 Mermaid 图表
- `process-folder-add-links` — 批量处理文件夹

**示例工作流程：**

**快速概念提取：**
```
process-current-add-links>extract-concepts-current
```

**深入研究：**
```
process-current-add-links>research-and-summarize>extract-concepts-current
```

**多语言分析：**
```
translate-current>process-current-add-links>extract-concepts-current
```

## 7. 高级设置

### 批量处理

**并行处理多个文件：**

```
Settings → Advanced → Batch Processing
Concurrency: 3 (default)
```

数值越高 = 速度越快，但 API 调用次数越多。

### 智能分块

**对于较长的笔记，请将其拆分成多个部分：**

```
Settings → Advanced → Smart Chunking
Chunk Size: 2000 words (default)
Overlap: 200 words (default)
```

防止令牌数量限制错误。

### 诊断面板

**查看详细日志：**

```
Settings → Advanced → Enable Diagnostics
```

显示：
- API 请求/响应
- 令牌使用情况
- 处理时间
- 错误

### 重复概念检测

**避免近乎重复的概念：**

```
Settings → Advanced → Synonym Suppression
Threshold: 0.8 (default, 0-1)
```

数值越高，合并的激进程度就越大。

## 配置文件

### 导出/导入设置

**在设备间共享配置：**

1. **设置 → 高级选项 → 导出配置**
2. 保存 `notemd-config.json`
3. 在新设备上：**导入配置**

**包含内容：**
- 提供方 API 密钥（可选）
- 模型选择
- 输出路径
- 自定义工作流

## 配置故障排除

### 设置未保存
- 检查 `.obsidian/plugins/notemd/` 中的文件权限
- 尝试：禁用插件 → 重新启用

### API 密钥无效
- 不要多余的空白
- 检查有效期
- 在提供商控制面板中验证积分。

### 模型无法加载
- 点击“获取模型列表”以刷新。
- 检查网络连接状态
- 尝试不同的提供商
