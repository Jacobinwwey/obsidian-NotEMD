# Diagramy

<TLDR>
Wybierz **rodzaj diagramu**, który potrzebujesz, a następnie wybierz jego **edytowalny format źródłowy**. Przeglądaj wynik w Obsidian i eksportuj go jako SVG, PNG lub PDF.
</TLDR>

## Zrozumienie trzech opcji

| Opcja | To, co kontroluje | Przykłady |
|---|---|---|
| Rodzaj diagramu | Znaczenie diagramu | Diagram przepływu, diagram sekwencyjny, diagram obwodu, wykres danych |
| Format źródłowy | Edytowalny lub wielokrotnie używalny plik tworzony przez Notemd | Mermaid, JSON Canvas, Draw.io, Drawnix, CircuitikZ |
| Format eksportu | Obraz lub dokument używany do udostępniania | SVG, PNG, PDF |

Draw.io oraz Drawnix to **formaty źródłowe**, a nie rodzaje diagramów. CircuitikZ to format źródłowy przeznaczony specjalnie do diagramów obwodów elektrycznych. Ten sam przegląd można eksportować w kilku formatach wizualnych, więc wybór CircuitikZ nie ogranicza Cię do formatu SVG.

## Wybór formatu źródłowego

| Format | Najlepszy do | Plik źródłowy |
|---|---|---|
| Mermaid | Diagramów umieszczanych bezpośrednio w Markdown | Blok kodu Mermaid |
| JSON Canvas | Pracy z natywnym interfejsem canvas w Obsidian | `.canvas` |
| Draw.io | Edycji późniejszej w diagram.net lub kompatybilnych narzędziach | `.drawio` |
| Drawnix | Edycji ręcznej lub w stylu tablicy | `.drawnix` |
| CircuitikZ | Strukturalnych obwodów elektrycznych i tranzystorowych | `.tex` |

Jeśli nie potrzebujesz konkretnego edytora, pozostaw pole **Preferred source format** ustawione na **Auto**. Notemd sam wybierze odpowiedni format dla wybranego rodzaju diagramu.

## Tworzenie i eksportowanie diagramu

1. Otwórz notatkę lub wybierz tekst, który chcesz zobrazować.
2. Wybierz opcję **Generate diagram** z paska bocznego Notemd lub palety poleceń.
3. Wybierz **rodzaj diagramu**, albo pozostaw ustawienie na **Auto-detect**.
4. Wybierz **format źródłowy**, albo pozostaw ustawienie na **Auto**.
5. Przeglądaj wynik w przeglądarce.
6. Wybierz opcję **Save source file**, **Export SVG**, **Export PNG** lub **Export PDF**.

W przypadku formatów PNG i PDF stosowany jest współczynnik PPI do eksportu obrazu, ustawiony w opcjach Notemd. SVG pozostaje obrazem wektorowym i nie jest podatny na wpływ wartości PPI.

## Diagramy obwodów elektrycznych

Dla zapytań dotyczących obwodów elektrycznych wybierz **Circuit diagram** w połączeniu z plikiem źródłowym typu **CircuitikZ**. Notemd najpierw generuje strukturalny opis obwodu, a następnie zapisuje kod źródłowy w formacie CircuitikZ. To rozwiązanie jest bardziej niezawodne niż proszenie modelu o napisanie nieograniczonego kodu LaTeX.

Obecna obsługa formatu CircuitikZ jest celowo ograniczona. Najlepiej działają powszechnie dostępne wzmacniacze NMOS oraz wspierane elementy CMOS. Zawsze sprawdzaj przegląd oraz kod źródłowy przed użyciem wygenerowanego obwodu w pracach technicznych lub publikacjach.

Przegląd w formacie SVG jest tylko narzędziem pomocniczym generowanym przez Notemd. Nie stanowi on dowodu na to, że lokalna instalacja LaTeX lub TikZJax pomyślnie skompilowała plik `.tex`.

## Rozwiązywanie problemów

### Wynik to niewłaściwy rodzaj diagramu

Wybierz konkretny rodzaj diagramu zamiast polegać na funkcji Auto-detect. W przypadku obwodów elektrycznych wybierz **Circuit diagram** i użyj polecenia **Generate diagram**, a nie starszego polecenia **Summarise as Mermaid diagram**.

### Format źródłowy nie jest odpowiedni

Przywróć ustawienie formatu źródłowego na **Auto**, albo wybierz format przeznaczony do danego rodzaju diagramu. CircuitikZ służy do diagramów obwodów; Draw.io oraz Drawnix są przeznaczone do obsługiwanych diagramów strukturalnych.

### Brak przycisków eksportu

Formaty SVG, PNG i PDF wymagają widocznego przeglądu wizualnego. Jeśli dostępny jest tylko tekst źródłowy, zapisz plik źródłowy i otwórz go w odpowiednim narzędziu zewnętrznym.

### Plik CircuitikZ nie kompiluje się poza programem

Sprawdź, czy Twoje środowisko LaTeX zawiera pakiet `circuitikz`. Ten plugin nie zawiera kompletnego dystrybucji LaTeX.
