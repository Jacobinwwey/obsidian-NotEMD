# Anthropic Fournisseur

<TLDR>
**Anthropic utilise le transport Messages natif API, et non le transport compatible avec OpenAI.** La configuration par défaut actuelle est `claude-3-5-sonnet-20240620`, `https://api.anthropic.com` et `temperature: 0.5`. Considérez-le comme une interface de protocole distincte lors du débogage.

Ceci fait partie du [Obsidian Guide de gestion des connaissances IA](/docs/pillar-ai-knowledge).
</TLDR>

## Installation

Créez une clé Anthropic API, ajoutez un profil de fournisseur `Anthropic`, et laissez le Base URL sur l’endpoint natif à moins que vous ne fassiez route par l’intermédiaire d’un profil de passerelle dédié. Choisissez l’ID du modèle Claude que vous souhaitez utiliser dans le champ modèle ; la valeur par défaut maintient la base du produit existante stable.

Anthropic est un bon candidat pour la synthèse de haute qualité, la synthèse de recherches et l’extraction de concepts. Si vous le combinez avec des fournisseurs moins chers, utilisez-le uniquement pour les tâches qui bénéficient de la qualité de ses résultats.

## Point de terminaison et authentification

| Champ | Préférence actuelle |
|-------|----------------|
| Transport | `anthropic` |
| Clé API | Obligatoire |
| Base URL | `https://api.anthropic.com` |
| Modèle par défaut | `claude-3-5-sonnet-20240620` |
| Température | `0.5` |
| Test de connexion | Seulement chat |

Le temps de exécution envoie des requêtes à `/v1/messages` avec `x-api-key` et `anthropic-version: 2023-06-01`. Ne configurez pas d’endpoint `/v1/chat/completions` de type OpenAI sous ce préreglage ; utilisez plutôt un préreglage de passerelle compatible avec OpenAI.

## Découverte de modèles

Anthropic utilise le mode de découverte `anthropic-models` dans le registre des fournisseurs. Si votre clé ou compte ne peut pas afficher les modèles, conservez un ID de modèle connu dans le profil et validez via le test de connexion.

Le transport n’est pas interchangeable avec les fournisseurs compatibles OpenAI. Un modèle Claude acheminé via OpenRouter, LiteLLM ou un autre passerelle doit faire partie de ce profil de passerelle, et non du profil natif Anthropic.

## Résolution de problèmes

- `401` ou `403` : vérifier la portée de la clé, l’accès à l’espace de travail, la facturation, et déterminer si la clé appartient à l’espace de travail Anthropic attendu.
- `404` : la base URL est incorrecte ou l’ID du modèle sélectionné n’est pas disponible pour le compte.
- Les ID de modèle Claude de type Gateway échouent : déplacez le profil vers le fournisseur de gateway correspondant, car Anthropic natif attend des noms de modèle natifs.
- Les requêtes longues échouent tardivement : réduisez la taille de la sortie du modèle par tâche ou utilisez un modèle ayant une limite supérieure connue en tokens de sortie.

## Quand l'utiliser

Utilisez Anthropic lorsque la qualité de la synthèse, des notes de recherche ou de l’écriture structurée est plus importante que le coût brut. Ne l’utilisez pas comme solution de remplacement générique compatible avec OpenAI ; le transport séparé est justement l’élément clé, et mélanger les protocoles rend le diagnostic plus difficile.
