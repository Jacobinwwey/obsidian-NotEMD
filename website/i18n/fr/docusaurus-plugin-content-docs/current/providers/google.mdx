# Google Fournisseur

<TLDR>
**Google utilise le mécanisme de transport natif pour les langages génératifs Gemini API.** La configuration par défaut actuelle est `gemini-2.0-flash-exp`, `https://generativelanguage.googleapis.com/v1` et `temperature: 0.5`. Il ne s’agit pas d’un profil compatible avec OpenAI.

Ceci fait partie du [Obsidian Guide de gestion des connaissances IA](/docs/pillar-ai-knowledge).
</TLDR>

## Installation

Créez une clé Google AI Studio ou Google Cloud API avec accès à Gemini, ajoutez un profil de fournisseur `Google`, et conservez la base de langage génératif native URL. Utilisez le profil Gemini Flash par défaut pour des traductions rapides et des workflows de notes étendus, puis remplacez les modèles par tâche lorsque vous avez besoin d’un modèle Gemini plus puissant ou plus performant.

Évitez de placer des identifiants de modèle Gemini dans un profil compatible avec OpenAI, sauf si vous utilisez explicitement une passerelle qui expose Gemini via un API compatible avec OpenAI.

## Point de terminaison et authentification

| Champ | Préférence actuelle |
|-------|----------------|
| Transport | `google` |
| Clé API | Obligatoire |
| Base URL | `https://generativelanguage.googleapis.com/v1` |
| Modèle par défaut | `gemini-2.0-flash-exp` |
| Température | `0.5` |
| Test de connexion | Seulement chat |

Le temps de exécution appelle `/models/{model}:generateContent?key={apiKey}`. La clé est transmise en tant que paramètre de requête, car c’est ainsi que l’endpoint natif Gemini est connecté dans le plugin.

## Découverte de modèles

Google utilise le mode de découverte `google-models`. La découverte doit retourner les modèles Gemini disponibles pour la clé API ; si le sélecteur est vide, testez la clé avec le même Base URL et confirmez que la famille de modèles est activée pour ce compte.

Les plafonds connus des tokens de sortie sont pris en compte par le modèle dans le registre des fournisseurs. Si une traduction ou une explication de diagramme trop longue est tronquée, sélectionnez un modèle Gemini disposant d’un plafond de sortie plus élevé ou réduisez la taille de sortie demandée.

## Résolution de problèmes

- `400` : l’ID du modèle ne correspond généralement pas à la route native Gemini ou la requête dépasse la forme prise en charge par le modèle.
- `401` ou `403` : vérifier les restrictions de clé, l’activation du projet/API, et déterminer si la clé peut accéder à Gemini.
- `404` : l’ID du modèle n’est pas disponible dans la version API configurée.
- Les exemples de passerelle échouent avec ce profil : utilisez la préconfiguration de passerelle au lieu du fournisseur natif Google.

## Quand l'utiliser

Utilisez Google pour une traduction rapide, le nettoyage de grandes quantités de notes, ainsi que pour les workflows où le rapport coût/performance de Gemini est supérieur à celui d’OpenAI ou d’Anthropic directs. Préférez un autre fournisseur pour les workflows nécessitant un traitement de modèles de raisonnement compatibles avec OpenAI.
