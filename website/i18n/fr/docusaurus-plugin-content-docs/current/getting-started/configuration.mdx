# Configuration

<TLDR>
**La configuration Notemd couvre 6 domaines principaux :** (1) les fournisseurs LLM et les clés API, (2) le choix du modèle par tâche, (3) les chemins de sortie et les modèles, (4) la langue UI et la langue de sortie, (5) les services de recherche (Tavily/DuckDuckGo), (6) les workflows personnalisés. Tous les paramètres se trouvent dans Obsidian Settings → Notemd. Les modifications prennent effet immédiatement.
</TLDR>

## Aperçu des paramètres

Accès : **Obsidian Paramètres (⚙️) → Notemd**

### Sections principales

1. **Configuration du fournisseur** — clés API, URL de base, modèles
2. **Paramètres de tâche** — Sélection du modèle par tâche
3. **Configuration de sortie** — Où enregistrer les fichiers générés
4. **Paramètres de langue** — UI et langues de sortie
5. **Intégration de recherche** — configuration Tavily/DuckDuckGo
6. **Flux de travail** — Chaînes de boutons personnalisées
7. **Avancé** — Diagnostic, traitement par lots, segmentation en blocs

---

## 1. Configuration du fournisseur

### Ajout d’un fournisseur

1. **Sélectionner un fournisseur** dans la liste déroulante (plus de 30 disponibles)
2. **Appuyez sur la touche API**
3. **Optionnel** : Définir une base personnalisée URL (pour les proxys/gatesways)
4. Cliquez sur « Obtenir la liste des modèles » pour vérifier la connexion
5. **Sélectionner le modèle par défaut**

### Fournisseurs courants

#### OpenAI
```
Provider: OpenAI
API Key: sk-...
Base URL: https://api.openai.com/v1 (default)
Model: gpt-4o-mini
```

#### Anthropic (Claude)
```
Provider: Anthropic
API Key: sk-ant-...
Model: claude-sonnet-4-6
```

#### Ollama (Local)
```
Provider: Ollama
Base URL: http://localhost:11434 (default)
Model: llama3.1
API Key: (not required)
```

#### DeepSeek
```
Provider: DeepSeek
API Key: sk-...
Model: deepseek-chat
```

### Plusieurs fournisseurs

Vous pouvez configurer plusieurs fournisseurs et les alterner selon la tâche :
- **Fournisseur principal** : Utilisé en cas de défaillance
- **Fournisseurs spécifiques aux tâches** : Surcharger pour des tâches spécifiques

## 2. Modèles spécifiques aux tâches

Configurez différents modèles pour différentes tâches afin d’optimiser le rapport coût/qualité :

### Types de tâches

| Tâche | Modèle par défaut | But |
|------|--------------|---------|
| **Lien** | Fournisseur principal | Identifier des concepts pour des liens wiki |
| **Génération de concepts** | Fournisseur principal | Extraire et décrire les concepts |
| **Recherche** | Fournisseur principal | Résumer les résultats de recherche sur le Web |
| **Traduction** | Fournisseur principal | Traduire le texte |
| **Génération de diagrammes** | Fournisseur principal | Générer des diagrammes Mermaid/Canvas |

### Exemple : Configuration optimisée en termes de coûts

```
Linking: gpt-4o-mini (fast, many calls)
Concept Generation: claude-sonnet-4-6 (quality extraction)
Research: gpt-4o (web-grounded)
Translation: gemini-flash (multilingual)
Diagram: ollama/llama3.1 (free, local)
```

### Comment configurer

1. **Paramètres → Notemd → Paramètres de tâche**
2. Pour chaque tâche, choisissez :
   - ✅ **Utiliser le fournisseur principal** (par défaut)
   - 🔄 **Utiliser un fournisseur personnalisé** → sélectionner le fournisseur + le modèle

## 3. Configuration de sortie

### Résultat des notes de concept

**Où sont enregistrées les nouvelles notes de concept :**

```
Settings → Output → Concept Folder
Default: concepts/
```

**Modèle :**
```
Settings → Output → Concept Template
Default:
---
tags: [concept]
created: {{date}}
---

# {{concept}}

{{description}}

## Backlinks
{{backlinks}}
```

### Options de structure de dossier

**Option 1 : Plat (tous les concepts dans un seul dossier)**
```
vault/
└── concepts/
    ├── Machine Learning.md
    ├── Neural Networks.md
    └── Deep Learning.md
```

**Option 2 : Hiérarchique (par dossier de source)**
```
vault/
├── papers/
│   └── concepts/
│       └── Transformer.md
└── notes/
    └── concepts/
        └── Attention Mechanism.md
```

Configurer : **Paramètres → Sortie → Utiliser des dossiers hiérarchiques**

### Sortie du diagramme

**Où les diagrammes sont enregistrés :**
```
Settings → Output → Diagram Folder
Default: diagrams/
```

**Format :**
- `.mmd` (Mermaid source)
- `.canvas` (JSON Canvas)
- `.html` (Vega-Lite)

## 4. Paramètres de langue

### UI Langue

**Langue de l’interface** (boutons, menus, boîtes de dialogue) :

```
Settings → Language → UI Language
Options: 21+ languages
Default: System language
```

Pris en charge : anglais, 简体中文, 繁體中文, japonais, coréen, espagnol, français, allemand, russe, العربية, et bien d’autres.

### Langue de sortie

**Langue du contenu généré** (notes de concept, résumés de recherche) :

```
Settings → Language → Output Language
Options: Same as UI languages
Default: Same as UI
```

**Cas d’usage :** UI en chinois + sortie en anglais pour des recherches bilingues.

### Cible de traduction

**Langue cible par défaut** pour les tâches de traduction :

```
Settings → Translation → Target Language
Default: English
```

## 5. Intégration de la recherche

### Tavily (Recommandé)

**Service de recherche optimisé par l’IA :**

```
Settings → Search → Service: Tavily
API Key: tvly-...
```

Obtenir la clé : [tavily.com](https://tavily.com/)

**Fonctionnalités :**
- Résultats de haute qualité
- Résumés optimisés par l’IA
- Citations de source

### DuckDuckGo (Expérimental)

**Aucune clé API nécessaire :**

```
Settings → Search → Service: DuckDuckGo
```

**Remarque :** Qualité inférieure, sans personnalisation.

## 6. Configuration du flux de travail

### Création de flux de travail personnalisés

**Les workflows associent plusieurs tâches à un seul bouton.**

1. **Paramètres → Flux de travail → Ajouter un flux de travail**
2. **Nom** : « Flux d’analyse de papier »
3. **DSL** : `process-current-add-links>extract-concepts-current>research-and-summarize>summarize-as-mermaid`
4. **Enregistrer**

Un nouveau bouton apparaît dans la barre latérale gauche.

### Syntaxe DSL

**Format :** `action1>action2>action3`

**Actions disponibles :**
- `process-current-add-links` — Ajouter des liens wiki à la note actuelle
- `extract-concepts-current` — Générer des notes de concept
- `research-and-summarize` — Recherche sur le web + résumé
- `translate-current` — Traduire la note actuelle
- `summarize-as-mermaid` — Générer le diagramme Mermaid
- `process-folder-add-links` — Traitement par lots de dossier

**Exemples de flux de travail :**

**Extraction rapide de concepts :**
```
process-current-add-links>extract-concepts-current
```

**Recherche approfondie :**
```
process-current-add-links>research-and-summarize>extract-concepts-current
```

**Analyse multilingue :**
```
translate-current>process-current-add-links>extract-concepts-current
```

## 7. Paramètres avancés

### Traitement par lots

**Traiter plusieurs fichiers en parallèle :**

```
Settings → Advanced → Batch Processing
Concurrency: 3 (default)
```

Plus élevé = plus rapide, mais plus d’appels API.

### Éclatement intelligent

**Pour les notes longues, divisez-les en parties :**

```
Settings → Advanced → Smart Chunking
Chunk Size: 2000 words (default)
Overlap: 200 words (default)
```

Empêche les erreurs de limite de tokens.

### Panneau de diagnostic

**Afficher les journaux détaillés :**

```
Settings → Advanced → Enable Diagnostics
```

Affiche :
- API requêtes/réponses
- Utilisation des tokens
- Délais de traitement
- Erreurs

### Détection de concepts dupliqués

**Éviter les concepts presque identiques :**

```
Settings → Advanced → Synonym Suppression
Threshold: 0.8 (default, 0-1)
```

Plus élevé = fusion plus agressive.

## Profils de configuration

### Paramètres d’exportation/importation

**Partager les configurations entre appareils :**

1. **Paramètres → Avancé → Exporter la configuration**
2. Enregistre `notemd-config.json`
3. Sur le nouveau dispositif : **Importer la configuration**

**Ce qui est inclus :**
- Clés du fournisseur API (facultatif)
- Sélections de modèles
- Chemins de sortie
- Flux de travail personnalisés

## Configurations recommandées

### Pour les étudiants

```
Provider: OpenAI (gpt-4o-mini)
Concept Folder: concepts/
UI Language: System
Output Language: System
Workflow: "Study Flow" (add-links>extract-concepts)
```

### Pour les chercheurs

```
Providers:
  - Linking: DeepSeek (cheap, many calls)
  - Concepts: Claude Sonnet (high quality)
  - Research: GPT-4o (web-grounded)
Concept Folder: notes/concepts/
Hierarchical: Yes
Workflow: "Paper Flow" (add-links>extract>research>diagram)
```

### Pour les utilisateurs soucieux de leur vie privée

```
Provider: Ollama (llama3.1)
All tasks: Local model
Search: DuckDuckGo (no API)
Diagnostics: Off
```

### Pour le travail bilingue

```
UI Language: Chinese
Output Language: English
Translation Target: Chinese
Provider: Gemini (multilingual)
```

## Résolution des problèmes de configuration

### Les paramètres ne sont pas enregistrés
- Vérifier les permissions du fichier dans `.obsidian/plugins/notemd/`
- Essayez : Désactiver le plugin → Réactiver

### API Clé invalide
- Aucun espace supplémentaire
- Vérifier l’expiration
- Vérifier les crédits dans le tableau de bord du fournisseur

### Les modèles ne s’chargent pas.
- Cliquez sur « Obtenir la liste des modèles » pour mettre à jour
- Vérifier la connectivité réseau
- Essayez différents fournisseurs
