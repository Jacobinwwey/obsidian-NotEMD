# Diagrammit

<TLDR>
Valitse **mitä tyyppinen diagrammi sinulle tarvitsee**, sitten valitse sen **muokattavaa lähteenvormattoa**. Esitä tulemus Obsidianissa ja eksportoi se SVG-, PNG- tai PDF-muotoon.
</TLDR>

## Ymmärrä kolme valintaa

| Valinta | Mitä se kontrolloi | Esimerkit |
|---|---|---|
| Diagrammityyppi | Mitä diagrammi tarkoittaa | Flowchart, sequence diagram, circuit diagram, data chart |
| Lähteenvormatti | Muokattavaa tai uudelleen käytettävää tiedostoa, joka Notemd luoo | Mermaid, JSON Canvas, Draw.io, Drawnix, CircuitikZ |
| Eksportivormatti | Kuvaa tai dokumenttia, jota käytetään jakamiseen | SVG, PNG, PDF |

Draw.io ja Drawnix ovat **lähteenvormattot**, ei diagrammityyppit. CircuitikZ on lähteenvormatti, joka on suunniteltu电路diagrammien luomiseen. Saman esityksen voidaan eksportoida useihin visuaalisiin muotoihin, joten CircuitikZ-valinta ei rajoita sinua vain SVG:hen.

## Valitse lähteenvormatti

| Muoto | Parhaiten sopii | Lähteetiedosto |
|---|---|---|
| Mermaid | Diagrammit, jotka pysyvät suoraan Markdownissa | Mermaid-koodiplokki |
| JSON Canvas | Obsidianin omat canvas-toimintatiedostot | `.canvas` |
| Draw.io | Muokkaus myöhemmin diagrammet.net:ssä tai samankaltaisissa ohjelmistoissa | `.drawio` |
| Drawnix | Käsikirjoituksellinen tai tahvallistyylinen muokkaus | `.drawnix` |
| CircuitikZ | Strukturoituja elektrisia ja transistoria koskevia circuiteja | `.tex` |

Jätä **Preferred source format**-valinta **Auto**-tilaan, jos sinulle ei tarvitse konkreettista ohjelmistoa. Notemd valitsee sopivan muodon valittun diagrammityyppin perusteella.

## Luoda ja eksportoida diagrammi

1. Avoa tiedosto tai valitse teksti, jota haluat visualisoida.
2. Valitse **Generate diagram** Notemdin puolekalustosta tai käskupalvelusta.
3. Valitse **diagrammityyppi**, tai jätä se **Auto-detect**-tilaan.
4. Valitse **lähteenvormatti**, tai jätä se **Auto**-tilaan.
5. Tarkista esitys.
6. Valitse **Save source file**, **Export SVG**, **Export PNG** tai **Export PDF**.

PNG ja PDF käyttävät Notemdin asetuksissa määriteltua kuvaneksportin PPI-arvoa. SVG jää vektorikuvaksi eikä se vaikuta PPI-arvoon.

## Circuit-diagrammit

Valitse **Circuit diagram** sekä **CircuitikZ source file** tuettavien circuit-taiotuksissa. Notemd luostaa first a strukturoituun circuit-tulkinnan, sitten kirjoittaa CircuitikZ-lähteetiedoston. Tämä on todennäköisempi kuin pyytää mallia kirjoittamaan rajoitusta mittaamattoman LaTeX -koodin.

Prahtinen CircuitikZ-tukea on suositettavasti rajoitettu. Yleisimmät NMOS-voimistimet ja tuetut CMOS-komponentit toimivat parhaiten. Aseta aina tarkista esitystä ja lähteetiedostoa ennen kuin käytät luodutta circuitia teknisissä tai publikointitöissä.

SVG-esitys on Notemdin luottamuksentekijä, joka tuotetaan esityksen tarkistamiseen. Se ei ole tõendus siitä, että lokalisoinnin LaTeX- tai TikZJax-asennus on `.tex`-tiedoston suorittanut edullisesti.

## Vianhallinta

### Tulemus on väärä diagrammityyppi

Valitse diagrammityyppi selkeästi sijaan siitä, että käytät Auto-detect -toimintaa. Circuit-diagrammin korjaamiseen valitse **Circuit diagram** ja käytä **Generate diagram**-komentoa, ei vanhaa **Summarise as Mermaid diagram**-komentoa.

### Lähteenvormatti ei ole sopiva

Palaa lähteenvormattin **Auto**-tilaan tai valitse muoto, joka on suunniteltu konkreettisen diagrammityyppin käyttöön. CircuitikZ on suunniteltu circuit-diagrammien luomiseen; Draw.io ja Drawnix ovat suunniteltu tuettaviin strukturoituihin diagrammiin.

### Eksportointipainikkeet puuttuvat

SVG-, PNG- ja PDF-muotoihin on tarpeen visuaalista esitystä. Jos on käytettävissä vain lähteeteksti, salaa lähteetiedosto ja avaa se vastaavassa ulkopuolisessa ohjelmistossa.

### CircuitikZ-tiedosto ei suorita ulkopuolisissa ohjelmistoissa

Tarkista, että sinun LaTeX-ympäristösi sisältää `circuitikz`-paketin. Palvelin ei sisällä täysin LaTeX-asennusta.
