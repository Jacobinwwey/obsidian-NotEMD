# แผนภาพ

<TLDR>
เลือก **ประเภทของแผนภาพที่คุณต้องการ** จากนั้นเลือก **รูปแบบไฟล์ต้นฉบับที่สามารถแก้ไขได้** ดูตัวอย่างผลลัพธ์ใน Obsidian แล้วส่งออกเป็นรูปแบบ SVG, PNG หรือ PDF
</TLDR>

## เข้าใจตัวเลือกทั้งสามประเภท

| ตัวเลือก | ควบคุมอะไร | ตัวอย่าง |
|---|---|---|
| ประเภทของแผนภาพ | ความหมายของแผนภาพ | แผนภาพกระแสงาน, แผนภาพลำดับเหตุการณ์, แผนภาพวงจรไฟฟ้า, แผนภูมิข้อมูล |
| รูปแบบไฟล์ต้นฉบับ | ไฟล์ที่สามารถแก้ไขหรือนำกลับมาใช้ซ้ำได้ซึ่ง Notemd จะสร้างขึ้น | Mermaid, JSON Canvas, Draw.io, Drawnix, CircuitikZ |
| รูปแบบการส่งออก | รูปภาพหรือเอกสารที่ใช้สำหรับการแชร์ | SVG, PNG, PDF |

Draw.io และ Drawnix เป็น **รูปแบบไฟล์ต้นฉบับ** ไม่ใช่ประเภทของแผนภาพ CircuitikZ เป็นรูปแบบไฟล์ต้นฉบับที่ออกแบบมาสำหรับแผนภาพวงจรไฟฟ้า คุณสามารถส่งออกตัวอย่างเดียวกันในรูปแบบภาพหลายรูปแบบได้ ดังนั้นการเลือก CircuitikZ จึงไม่ได้จำกัดคุณให้ส่งออกเป็นรูปแบบ SVG เท่านั้น

## เลือกรูปแบบไฟล์ต้นฉบับ

| รูปแบบ | เหมาะสำหรับ | ชื่อไฟล์ต้นฉบับ |
|---|---|---|
| Mermaid | แผนภาพที่เก็บไว้ใน Markdown โดยตรง | บล็อกโค้ด Mermaid |
| JSON Canvas | กระบวนการทำงานบน canvas ของ Obsidian แบบดั้งเดิม | `.canvas` |
| Draw.io | การแก้ไขต่อใน diagrams.net หรือเครื่องมือที่รองรับ | `.drawio` |
| Drawnix | การแก้ไขแบบวาดด้วยมือหรือสไตล์กระดานไวท์บอร์ด | `.drawnix` |
| CircuitikZ | วงจรไฟฟ้าและวงจรทรานซิสเตอร์ที่มีโครงสร้างชัดเจน | `.tex` |

หากคุณไม่ต้องการใช้โปรแกรมแก้ไขเฉพาะใด ให้ตั้งค่า **Preferred source format** เป็น **Auto** Notemd จะเลือกรูปแบบที่เหมาะสมกับประเภทของแผนภาพที่คุณเลือกให้โดยอัตโนมัติ

## สร้างและส่งออกแผนภาพ

1. เปิดบันทึกหรือเลือกข้อความที่คุณต้องการแสดงเป็นแผนภาพ
2. เลือก **Generate diagram** จากแถบข้าง Notemd หรือแผงคำสั่ง
3. เลือก **ประเภทของแผนภาพ** หรือปล่อยให้เป็น **Auto-detect**
4. เลือก **รูปแบบไฟล์ต้นฉบับ** หรือปล่อยให้เป็น **Auto**
5. ตรวจสอบตัวอย่างผลลัพธ์
6. เลือก **Save source file**, **Export SVG**, **Export PNG** หรือ **Export PDF**

รูปแบบ PNG และ PDF จะใช้ค่า PPI สำหรับการส่งออกรูปภาพที่ตั้งค่าไว้ในการตั้งค่าของ Notemd ส่วน SVG ยังคงเป็นรูปภาพแบบเวกเตอร์ จึงไม่ได้รับผลกระทบจากค่า PPI

## แผนภาพวงจรไฟฟ้า

สำหรับการสร้างแผนภาพวงจรไฟฟ้าที่รองรับ ให้เลือก **Circuit diagram** พร้อมกับ **CircuitikZ source file** Notemd จะสร้างคำอธิบายวงจรที่มีโครงสร้างชัดเจนก่อน แล้วจึงเขียนไฟล์ต้นฉบับในรูปแบบ CircuitikZ วิธีนี้จะน่าเชื่อถือกว่าการให้โมเดลเขียนโค้ด LaTeX แบบไม่มีข้อจำกัด

ปัจจุบันความสามารถในการรองรับ CircuitikZ ถูกจำกัดไว้โดยเจตนา โดยจะทำงานได้ดีที่สุดกับตัวขยาย NMOS ที่มีแหล่งข้อมูลมาตรฐานและบล็อกสร้าง CMOS ที่รองรับ ควรตรวจสอบตัวอย่างผลลัพธ์และไฟล์ต้นฉบับเสมอก่อนนำแผนภาพที่สร้างขึ้นไปใช้ในงานด้านเทคนิคหรือการเผยแพร่

ตัวอย่างผลลัพธ์ในรูปแบบ SVG เป็นเพียงเครื่องมือช่วยตรวจสอบที่ Notemd สร้างขึ้นมาเท่านั้น ไม่ใช่หลักฐานยืนยันว่าไฟล์ `.tex` ถูกคอมไพล์สำเร็จโดยการติดตั้ง LaTeX หรือ TikZJax ในเครื่องของคุณ

## การแก้ไขปัญหา

### ผลลัพธ์เป็นประเภทของแผนภาพที่ผิด

ให้เลือกประเภทของแผนภาพโดยตรงแทนการใช้โหมด Auto-detect สำหรับแผนภาพวงจรไฟฟ้า ให้เลือก **Circuit diagram** และใช้คำสั่ง **Generate diagram** ไม่ใช่คำสั่งเก่า **Summarise as Mermaid diagram**

### รูปแบบไฟล์ต้นฉบับไม่เหมาะสม

ให้กลับไปตั้งค่ารูปแบบไฟล์ต้นฉบับเป็น **Auto** หรือเลือกรูปแบบที่เหมาะสมกับประเภทของแผนภาพนั้นๆ CircuitikZ ใช้สำหรับแผนภาพวงจรไฟฟ้าเท่านั้น ส่วน Draw.io และ Drawnix ถูกออกแบบมาสำหรับแผนภาพที่มีโครงสร้างที่รองรับเท่านั้น

### ปุ่มสำหรับการส่งออกหายไป

รูปแบบ SVG, PNG และ PDF จำเป็นต้องมีตัวอย่างผลลัพธ์ในรูปภาพ หากมีเพียงข้อความต้นฉบับเท่านั้น ให้บันทึกไฟล์ต้นฉบับแล้วเปิดในเครื่องมือภายนอกที่เหมาะสม

### ไฟล์ CircuitikZ ไม่สามารถคอมไพล์ได้ภายนอก

ตรวจสอบให้แน่ใจว่าสภาพแวดล้อม LaTeX ของคุณมีแพ็กเกจ `circuitikz` รวมอยู่ด้วย เนื่องจากปลั๊กอินนี้ไม่ได้รวมการติดตั้ง LaTeX แบบครบถ้วนมาให้
