# 구성 설정

<TLDR>
**Notemd 설정은 6가지 주요 영역을 포함합니다:** (1) LLM 제공자 및 API 키, (2) 작업별 모델 선택, (3) 출력 경로 및 템플릿, (4) UI 언어와 출력 언어, (5) 검색 서비스 (Tavily/DuckDuckGo), (6) 사용자 정의 워크플로우. 모든 설정은 Obsidian 설정 → Notemd에 있습니다. 변경 사항은 즉시 적용됩니다.
</TLDR>

## 설정 개요

접근 방법: **Obsidian 설정 (⚙️) → Notemd**

### 주요 섹션들

1. **제공자 설정** — API 키, 기본 URLs, 모델들
2. **작업 설정** — 작업별 모델 선택
3. **출력 설정** — 생성된 파일을 저장할 위치
4. **언어 설정** — UI 및 출력 언어
5. **검색 통합** — Tavily/DuckDuckGo 설정
6. **워크플로우** — 사용자 정의 버튼 체인
7. **고급** — 진단, 일괄 처리, 청크 처리

---

## 1. 제공업체 설정

### 프로바이더 추가

1. 드롭다운에서 **제공업체 선택**하기 (30개 이상 사용 가능)
2. **API 키를 입력하세요**
3. **선택 사항**: 사용자 지정 기본값 URL 설정 (프록시/게이트웨이용)
4. **“모델 목록 가져오기”를 클릭**하여 연결 상태를 확인하세요.
5. **기본 모델 선택**

### 일반 제공업체

#### OpenAI
```
Provider: OpenAI
API Key: sk-...
Base URL: https://api.openai.com/v1 (default)
Model: gpt-4o-mini
```

#### Anthropic (Claude)
```
Provider: Anthropic
API Key: sk-ant-...
Model: claude-sonnet-4-6
```

#### Ollama (로컬)
```
Provider: Ollama
Base URL: http://localhost:11434 (default)
Model: llama3.1
API Key: (not required)
```

#### DeepSeek
```
Provider: DeepSeek
API Key: sk-...
Model: deepseek-chat
```

### 여러 공급업체

여러 공급자를 설정하고 작업별로 그 사이를 전환할 수 있습니다:
- **주 공급업체**: 백업으로 사용됨
- **작업별 프로바이더**: 특정 작업에 대해 재정의하기

## 2. 작업별 모델

비용과 품질을 최적화하기 위해 각기 다른 작업에 적합한 모델을 구성합니다:

### 작업 유형

| 작업 | 기본 모델 | 목적 |
|------|--------------|---------|
| **링크 연결** | 주요 공급업체 | 위키 링크를 위한 개념들을 파악하세요 |
| **컨셉트 생성** | 주요 공급업체 | 개념을 추출하고 설명합니다. |
| **연구** | 주요 공급업체 | 웹 검색 결과 요약하기 |
| **번역** | 주요 공급업체 | 텍스트를 번역해 주세요 |
| **다이어그램 생성** | 주요 공급업체 | Mermaid/Canvas 다이어그램 생성 |

### 예시: 비용 최적화 설정

```
Linking: gpt-4o-mini (fast, many calls)
Concept Generation: claude-sonnet-4-6 (quality extraction)
Research: gpt-4o (web-grounded)
Translation: gemini-flash (multilingual)
Diagram: ollama/llama3.1 (free, local)
```

### 어떻게 설정하나요?

1. **설정 → Notemd → 작업 설정**
2. 각 작업에 대해 다음 중 하나를 선택하세요:
   - ✅ **기본 제공업체 사용** (기본값)
   - 🔄 **사용자 지정 프로바이더 사용** → 프로바이더 및 모델 선택

## 3. 출력 설정

### 개념 노트 출력

**새로운 콘셉트 노트가 저장되는 위치:**

```
Settings → Output → Concept Folder
Default: concepts/
```

**템플릿:**
```
Settings → Output → Concept Template
Default:
---
tags: [concept]
created: {{date}}
---

# {{concept}}

{{description}}

## Backlinks
{{backlinks}}
```

### 폴더 구조 옵션

**옵션 1: 평면형 (모든 개념을 하나의 폴더에 저장)**
```
vault/
└── concepts/
    ├── Machine Learning.md
    ├── Neural Networks.md
    └── Deep Learning.md
```

**옵션 2: 계층적 구조(소스 폴더별)**
```
vault/
├── papers/
│   └── concepts/
│       └── Transformer.md
└── notes/
    └── concepts/
        └── Attention Mechanism.md
```

구성: **설정 → 출력 → 계층적 폴더 사용**

### 다이어그램 출력

**다이어그램 저장 위치:**
```
Settings → Output → Diagram Folder
Default: diagrams/
```

**형식:**
- `.mmd` (Mermaid 소스)
- `.canvas` (JSON Canvas)
- `.html` (Vega-Lite)

## 4. 언어 설정

### UI 언어

**인터페이스 언어** (버튼, 메뉴, 대화상자):

```
Settings → Language → UI Language
Options: 21+ languages
Default: System language
```

지원 언어: 영어, 简体中文, 繁體中文, 일본어, 한국어, Español, Français, Deutsch, Русский, العربية 및 그 외 여러 언어.

### 출력 언어

**생성된 콘텐츠 언어** (개념 노트, 연구 요약):

```
Settings → Language → Output Language
Options: Same as UI languages
Default: Same as UI
```

**사용 사례:** 중국어 UI + 영어 출력을 통한 이중 언어 연구.

### 번역 대상

번역 작업의 **기본 대상 언어**:

```
Settings → Translation → Target Language
Default: English
```

## 5. 검색 통합

### Tavily (권장)

**AI 최적화 검색 서비스:**

```
Settings → Search → Service: Tavily
API Key: tvly-...
```

키 가져오기: [tavily.com](https://tavily.com/)

**기능:**
- 고품질의 결과
- AI 최적화 요약
- 출처 인용

### DuckDuckGo (실험적)

**API 키는 필요하지 않습니다:**

```
Settings → Search → Service: DuckDuckGo
```

**참고:** 품질이 낮으며, 사용자 지정이 불가능합니다.

## 6. 워크플로우 설정

### 커스텀 워크플로우 생성

**워크플로우는 여러 작업을 하나의 버튼에 연결합니다.**

1. **설정 → 워크플로우 → 워크플로우 추가**
2. **이름**: "Paper Analysis Flow"
3. **DSL**: `process-current-add-links>extract-concepts-current>research-and-summarize>summarize-as-mermaid`
4. **저장**

왼쪽 사이드바에 새로운 버튼이 나타납니다.

### DSL 구문

**형식:** `action1>action2>action3`

**사용 가능한 작업:**
- `process-current-add-links` — 현재 노트에 위키 링크 추가
- `extract-concepts-current` — 콘셉트 노트 생성
- `research-and-summarize` — 웹 조사 + 요약
- `translate-current` — 현재 노트 번역하기
- `summarize-as-mermaid` — Mermaid 다이어그램 생성
- `process-folder-add-links` — 폴더 일괄 처리

**예시 워크플로우:**

**빠른 개념 추출:**
```
process-current-add-links>extract-concepts-current
```

**심층 연구:**
```
process-current-add-links>research-and-summarize>extract-concepts-current
```

**다국어 분석:**
```
translate-current>process-current-add-links>extract-concepts-current
```

## 7. 고급 설정

### 배치 처리

**여러 파일을 병렬로 처리하기:**

```
Settings → Advanced → Batch Processing
Concurrency: 3 (default)
```

더 높을수록 = 더 빠르지만 API 호출 횟수가 더 많아집니다.

### 스마트 청킹

**긴 메모의 경우, 여러 부분으로 나누세요:**

```
Settings → Advanced → Smart Chunking
Chunk Size: 2000 words (default)
Overlap: 200 words (default)
```

토큰 제한 오류를 방지합니다.

### 진단 패널

**상세 로그 보기:**

```
Settings → Advanced → Enable Diagnostics
```

표시됨:
- API 요청/응답
- 토큰 사용량
- 처리 시간
- 오류

### 중복 개념 탐지

**거의 동일한 개념은 피하십시오:**

```
Settings → Advanced → Synonym Suppression
Threshold: 0.8 (default, 0-1)
```

값이 높을수록 더 공격적인 병합이 이루어집니다.

## 구성 문제 해결

### 설정이 저장되지 않았습니다.
- `.obsidian/plugins/notemd/`의 파일 권한을 확인하세요.
- 시도해 보세요: 플러그인 비활성화 → 다시 활성화

### API 키가 유효하지 않습니다.
- 여분의 공백 없음
- 만기일 확인
- 제공업체 대시보드에서 크레딧을 확인하세요

### 모델이 로드되지 않습니다.
- “모델 목록 가져오기”를 클릭하여 새로 고침하세요.
- 네트워크 연결 상태를 확인하세요.
- 다른 제공업체를 시도해 보세요.
