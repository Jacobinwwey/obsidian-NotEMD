# 빠른 시작 가이드

<TLDR>
**5분 만에 Notemd를 사용해 보세요.** 단 3단계만 거치면 됩니다: (1) LLM API 키를 추가하거나 로컬 Ollama을 사용합니다, (2) 노트를 열고 “파일 처리(링크 추가)” 명령을 실행합니다, (3) 사이드바에서 원클릭 워크플로우를 살펴봅니다. 30개 이상의 제공업체를 지원하며, 코딩이 필요하지 않습니다. Ollama를 사용하면 오프라인에서도 작동합니다.
</TLDR>

## 1단계: 첫 번째 LLM 제공업체를 설정합니다 (2분)

### 옵션 A: 클라우드 제공업체 (OpenAI, Anthropic 등)

1. **설정** → **Notemd**를 엽니다.
2. **LLM Provider Configuration** 아래에서 드롭다운 목록에서 제공업체를 선택하세요.
3. **API 키**를 입력하세요.
4. (선택 사항) 사용 가능한 모델을 보려면 **“Get Model List”**를 클릭하세요.
5. 각 작업 유형에 맞는 모델을 선택하세요

**인기 있는 선택지:**
- **OpenAI**: `gpt-4o-mini` (빠르고, 저렴함)
- **Anthropic**: `claude-sonnet-4-6` (고품질)
- **DeepSeek**: `deepseek-chat` (매우 저렴함)

### 옵션 B: 로컬 Ollama (API 키 필요 없음)

1. [Ollama](https://ollama.com/)를 설치하세요.
2. 모델 다운로드: `ollama pull llama3.1`
3. Notemd 설정에서 제공업체로 **"Ollama"**를 선택하세요
4. 모델 이름: `llama3.1` (또는 사용자가 다운로드한 모든 모델)

## 2단계: 첫 번째 작업 시도하기 (1분)

### 노트에 위키 링크 추가하기

1. 의미 있는 내용이 포함된 노트를 열어보세요.
2. 편집기에서 마우스 오른쪽 버튼 클릭
3. **"파일 처리(링크 추가)"**를 선택하세요.
4. 몇 초 기다려 주세요.
5. 주요 개념들은 이제 `[[wiki-linked]]`입니다!

**예시:**

**이전:**
```
Machine learning models use neural networks to process data.
```

**완료 후:**
```
[[Machine learning]] models use [[neural networks]] to process data.
```

## 3단계: 기능 탐색 (2분)

### 개념 노트 추출

1. 마우스 오른쪽 버튼 클릭 → **"현재 파일에서 개념 추출"**
2. Notemd는 각 개념마다 별도의 파일을 생성합니다.
3. 구성된 콘셉트 폴더(기본값: `concepts/`)를 확인해 주세요.
4. 각 콘셉트 노트에는 원본 노트로 연결되는 백링크가 있습니다.

### 조사 및 요약

1. 노트에서 주제를 강조 표시하세요
2. 마우스 오른쪽 버튼 클릭 → **"조사 및 요약"**
3. Notemd은 웹 검색을 수행합니다 (Tavily 또는 DuckDuckGo)
4. 요약 내용이 출처와 함께 노트에 추가됩니다.

### 원클릭 워크플로우

1. 워크플로우 버튼이 있는지 **왼쪽 사이드바**를 확인해 보세요.
2. 어떤 노트에서든 **"One-Click Extract"**를 클릭하세요
3. 실행 과정은 다음과 같습니다: 링크 추가 → 개념 추출 → 조사 → 다이어그램 생성
4. 한 번의 클릭으로 모든 것이 해결됩니다!

## 일반적인 초기 작업들

### 과제 1: 논문/기사 처리

```
1. Paste article content into a note
2. Run "Process file (add links)"
3. Run "Extract concepts"
4. Run "Research and summarize" on unfamiliar terms
5. Run "Summarize as Mermaid diagram" for visual structure
```

### 과제 2: 노트 번역

```
1. Highlight text to translate
2. Right-click → "Translate to target language"
3. Set target language in settings first
```

### 과제 3: 지식 맵 생성

```
1. Process 5-10 related notes with "add links"
2. Run "Extract concepts" on each
3. Open Graph view to see connections
4. Your knowledge graph is forming!
```

## 맞춤 설정 팁

### 작업별 모델 선택

다양한 작업에 맞는 서로 다른 모델을 사용할 수 있습니다:

| 작업 | 추천 모델 | 왜요? |
|------|------------------|-----|
| 링크 추가 | `gpt-4o-mini`, `deepseek-chat` | 빠르고, 많은 호출 |
| 개념 추출 | `claude-sonnet-4-6`, `gpt-4o` | 품질 추출 |
| 조사 | `perplexity`, `gpt-4o` | 웹 기반 |
| 번역 | `gemini-flash`, `gpt-4o` | 다국어 |
| 다이어그램 | `claude-opus`, 로컬 모델 | 복잡한 추론 |

구성 위치: **Settings → Notemd → Task-Specific Models**

### 커스텀 워크플로우

자신만의 버튼 만들기:

1. 설정 → Notemd → 워크플로우
2. “워크플로우 추가”를 클릭하세요.
3. DSL 입력: `process-current-add-links>extract-concepts-current>research-and-summarize`
4. 이름 지정: "My Research Flow"
5. 사이드바에 새로운 버튼이 나타납니다.

## 문제 해결

### "API 키가 유효하지 않습니다."
- 키가 올바른지 확인하세요(여분의 공백 없음).
- 계정에 크레딧이 있는지 확인하세요.
- 설정의 "Connection Test" 버튼을 사용해 테스트하세요

### "모델을 찾을 수 없습니다."
- 사용 가능한 모델을 보려면 “Get Model List”를 클릭하세요.
- 일부 제공업체는 특정 모델 이름을 요구합니다(예: `gpt-4o`이지 `gpt-4`은 아님).

### 링크가 표시되지 않습니다.
- 메모에 충분한 내용이 포함되도록 하세요(50단어 이상).
- 더 강력한 모델을 사용해 보세요.
- LLM 호출이 성공했는지 확인하십시오(진단 패널 참조).

### 느린 성능
- 더 빠른 모델을 사용하세요(`gpt-4o-mini` 대신 `gpt-4o` 사용).
- 설정에서 청크 크기를 줄이기
- 배치 작업에 대한 병렬 처리를 활성화합니다.
