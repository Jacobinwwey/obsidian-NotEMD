# 다이어그램

<TLDR>
원하는 **다이어그램 유형**을 선택한 다음, 해당 유형에 맞는 **편집 가능한 소스 형식**을 골라주세요. Obsidian에서 결과를 미리 보고 SVG, PNG 또는 PDF 형태로 내보낼 수 있습니다.
</TLDR>

## 세 가지 선택 옵션 이해하기

| 선택 옵션 | 제어하는 항목 | 예시 |
|---|---|---|
| 다이어그램 유형 | 다이어그램이 나타내는 의미 | 흐름도, 시퀀스 다이어그램, 회로도, 데이터 차트 |
| 소스 형식 | Notemd가 생성하는 편집 가능하거나 재사용 가능한 파일 | Mermaid, JSON Canvas, Draw.io, Drawnix, CircuitikZ |
| 내보내기 형식 | 공유할 때 사용되는 이미지 또는 문서 형태 | SVG, PNG, PDF |

Draw.io와 Drawnix는 다이어그램 유형이 아닌 **소스 형식**에 해당합니다. CircuitikZ는 회로도를 그리기 위해 설계된 소스 형식입니다. 동일한 미리 보기 결과를 여러 가지 시각적 형식으로 내보낼 수 있으므로 CircuitikZ를 선택해도 반드시 SVG만 사용해야 하는 것은 아닙니다.

## 소스 형식 선택하기

| 형식 | 가장 적합한 용도 | 소스 파일 |
|---|---|---|
| Mermaid | Markdown 내에 직접 포함되는 다이어그램 | Mermaid 코드 블록 |
| JSON Canvas | Obsidian의 기본 캔버스 기반 작업 흐름 | `.canvas` |
| Draw.io | diagrams.net이나 호환 가능한 도구에서 추가로 편집 | `.drawio` |
| Drawnix | 손으로 그리거나 화이트보드 스타일로 편집하는 경우 | `.drawnix` |
| CircuitikZ | 구조화된 전기 회로 및 트랜지스터 회로 | `.tex` |

특정 편집기를 사용하고 싶지 않다면 **Preferred source format**을 **Auto** 상태로 두세요. Notemd는 선택한 다이어그램 유형에 맞는 호환 가능한 형식을 자동으로 선택합니다.

## 다이어그램 생성 및 내보내기

1. 시각화하고자 하는 노트를 열거나 해당 텍스트를 선택하세요.
2. Notemd 사이드바나 커맨드 팔레트에서 **Generate diagram**을 선택하세요.
3. **다이어그램 유형**을 선택하거나 **Auto-detect** 상태로 두세요.
4. **소스 형식**을 선택하거나 **Auto** 상태로 두세요.
5. 미리 보기 결과를 확인하세요.
6. **Save source file**, **Export SVG**, **Export PNG** 또는 **Export PDF** 중 원하는 옵션을 선택하세요.

PNG와 PDF의 경우 Notemd 설정에 지정된 이미지 내보내기 PPI 값이 적용됩니다. SVG는 벡터 이미지이므로 PPI 값의 영향을 받지 않습니다.

## 회로도

지원되는 회로 관련 요청의 경우 **Circuit diagram**을 선택하고 **CircuitikZ source file**을 사용하세요. Notemd는 먼저 구조화된 회로 설명을 생성한 다음 CircuitikZ 소스 코드를 작성합니다. 이 방식은 모델에게 제한 없이 LaTeX를 작성하도록 요청하는 것보다 훨씬 안정적입니다.

현재 CircuitikZ의 지원 범위는 의도적으로 제한되어 있습니다. 일반적인 NMOS 증폭기와 지원되는 CMOS 구성 요소들이 가장 잘 작동합니다. 기술 문서나 출판물에 생성된 회로를 사용하기 전에 반드시 미리 보기 결과와 소스 코드를 꼼꼼히 확인해 주세요.

SVG 미리 보기는 Notemd가 생성한 검토용 보조 도구일 뿐입니다. 이것이 로컬에 설치된 LaTeX나 TikZJax 환경에서 `.tex` 파일이 정상적으로 컴파일되었다는 증거는 아닙니다.

## 문제 해결

### 결과가 원하는 다이어그램 유형이 아닌 경우

**Auto-detect**를 사용하는 대신 다이어그램 유형을 명시적으로 선택해 주세요. 회로도의 경우 **Circuit diagram**을 선택하고 기존의 **Summarise as Mermaid diagram** 명령 대신 **Generate diagram** 명령을 사용해야 합니다.

### 소스 형식이 적합하지 않은 경우

소스 형식을 다시 **Auto** 상태로 설정하거나, 해당 다이어그램 유형에 맞는 형식을 선택하세요. CircuitikZ는 회로도용이며, Draw.io와 Drawnix는 지원되는 구조화된 다이어그램용으로 설계되었습니다.

### 내보내기 버튼이 표시되지 않는 경우

SVG, PNG, PDF를 내보내려면 시각적 미리 보기가 필요합니다. 소스 텍스트만 제공된다면 소스 파일을 저장한 후 해당 형식에 맞는 외부 도구에서 열어주세요.

### CircuitikZ 파일이 외부에서 컴파일되지 않는 경우

자신의 LaTeX 환경에 `circuitikz` 패키지가 포함되어 있는지 확인해 주세요. 이 플러그인에는 완전한 LaTeX 배포판이 포함되어 있지 않습니다.
