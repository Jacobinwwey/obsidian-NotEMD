# 개념 노트

<TLDR>
**Notemd는 메모에서 핵심 개념을 추출하여 독립적인 개념 노트 파일을 생성합니다.** LLM는 핵심 용어를 식별하고, 설정 가능한 폴더에 각 개념당 하나의 `.md` 파일을 생성하며, 필요에 따라 원본 메모로의 백링크를 추가합니다. 동의어 제거, 작업별 모델 선택, 파일명 정리 기능 및 4단계 중복 제거 엔진을 지원합니다. 이 기능은 [Obsidian AI 지식 관리 가이드](/docs/pillar-ai-knowledge)의 일부입니다.
</TLDR>

## 개요

컨셉 노트는 연결된 지식 베이스의 기본 구성 요소입니다. 노트를 처리할 때, Notemd:

1. **구조화된 추출 프롬프트를 사용하여 텍스트를 LLM로 전송합니다**
2. **LLM 응답에서 `CONCEPT:` 줄들을 파싱합니다**
3. 지정된 폴더에 각 개념당 하나의 `.md` 파일을 생성합니다.
4. (활성화될 경우) 출처 노트에 **백링크를 추가**합니다.

## 작동 원리

### 추출 파이프라인

```
Source Note → Split into Chunks → LLM per Chunk → Parse CONCEPT: Lines
    → Deduplicate via Set → Create .md Files → Add Backlinks
    → (Optional: Generate Log)
```

### 추출 프롬프트

LLM는 다음과 같이 지시하는 프롬프트를 받습니다:

- 문서의 주제와 관련된 **명사 또는 명사구**를 추출합니다.
- **구체성 우선시하기** — “Relaxation” 대신 “Dielectric Relaxation” 사용하기
- **단수형으로 정규화** — "models"는 "model"이 됩니다
- **하위 개념 억제** — “유전체 이완”이 나타나면 “이완”도 함께 추출하지 마세요.
- **생략** 참고문헌/참고자료 섹션, 일반 명사, 회사/제품명
- 한 개념씩 줄에 하나씩 출력합니다: `CONCEPT: Term Name`

**동의어 억제** (선택 사항): 프롬프트 앞에 “가능한 한 동의어나 의미적으로 유사한 핵심 개념, 키워드를 추출하지 않도록 하세요”라는 문구를 붙입니다.

**집중 학습 분야** (선택 사항): 특정 학문 분야로 추출을 제한하기 위해 `Relevant Fields: [domain]`를 앞에 붙입니다.

**언어 재지정** (선택 사항): “출력 결과는 반드시 [language] 언어로 되어 있어야 합니다.”라는 문구가 추가됩니다.

### 청킹

긴 노트는 `settings.chunkWordCount`로 나뉩니다. 각 조각은 별도로 전송됩니다. 모든 조각의 개념들은 `Set`에 축적되며, 조각 간에 중복되는 항목들은 자동으로 제거됩니다. 개별 조각에서 발생하는 LLM 오류는 전체 작업을 중단시키지 않습니다.

## 컨셉 노트 작성

### 파일명 규칙

| 규칙 | 동작 방식 |
|------|----------|
| 하이픈/언더스코어 | 공백으로 대체됨 |
| 특수 문자 | 제거된 문자: 백슬래시, 슬래시, 콜론, 별표, 물음표, 따옴표, 각도 괄호, 파이프, 해시, 카렛, 괄호 |
| 공백 | 축소되고 잘림 |
| 최대 길이 | 100자 (잘림 + 다시 제거됨) |
| 빈 결과/모든 점 표시 결과 | 조용히 건너뛰었습니다. |

예시: `A-B: Key*Term`는 `A B KeyTerm.md`가 됩니다.

### 템플릿

**전체 템플릿** (백링크 활성화, 최소 템플릿 비활성화):

```markdown
# Concept Name

## Linked From
- [[SourceNoteBasename]]
```

**최소 템플릿** (`extractConceptsMinimalTemplate: true`일 때):

```markdown
# Concept Name
```

### 기존 노트 업데이트

컨셉트 파일이 이미 존재할 경우:

1. 백링크가 활성화되어 있고 출처 정보가 아직 목록에 없는 경우 백링크를 추가하세요.
2. `## Linked From` 섹션이 존재하는 경우, 다음 `##` 헤더 앞에 삽입합니다.
3. `## Linked From`가 존재하지 않으면 끝에 섹션을 추가합니다.
4. 중복된 백링크는 확인한 후 건너뜁니다.

### 구성 설정

| 설정 | 기본값 | 효과 |
|---------|---------|--------|
| `useCustomConceptNoteFolder` | `true` | 마스터 토글. 꺼짐 = 노트 생성 안 함 |
| `conceptNoteFolder` | `''` (비활성화됨) | 컨셉 노트를 위한 폴더 경로 |
| `extractConceptsMinimalTemplate` | `true` | 제목만 있는 노트 (링크된 항목 없음) |
| `extractConceptsAddBacklink` | `false` | 독립형 추출 중 백링크를 다시 추가하기 |
| `extractConceptsProvider` / `extractConceptsModel` | DeepSeek | 개념 추출을 위한 작업별 LLM |
| `extractConceptsLanguage` | `'en'` | 개념 이름의 출력 언어 |
| `replaceSynonymsDuringConceptExtraction` | `false` | 프롬프트에서 동의어 억제 기능을 활성화합니다. |
| `generateConceptLogFile` | `false` | `.log` 목록에 생성된 개념들을 작성하세요 |

**중요:** `extractConceptsAddBacklink`와 `extractConceptsMinimalTemplate` 설정은 **전용 추출 명령어**에만 영향을 미칩니다. “링크 추가” 명령어는 항상 백링크가 포함된 완전한 템플릿 형태의 노트를 생성합니다.

## 중복 제거

dedup 명령어(`Notemd: Check & remove duplicate concept notes`)는 5단계의 분석을 수행합니다:

### 1단계: 정확한 파일명 일치

구성된 범위 전체에서 대소문자를 구분하지 않는 파일명 비교.

### 2단계: 복수형 감지

접미사 제거: `-ies`는 `-y`로, `-es`와 `-s`는 제거됨. 단수 형태를 스코프 파일과 비교함. 불규칙 복수형(children/child)은 처리하지 않음.

### 3단계: 기호 정규화

적용 대상: `toLowerCase()`, 하이픈/언더스코어를 공백으로 대체하고, 영숫자가 아닌 문자를 제거하며, 공백을 축소합니다. 동일하게 정규화된 항목들은 표시됩니다.

### 4단계: 단어 단위 격리

단어 하나로 된 개념 노트는 여러 단어로 구성된 스코프 파일의 파일명과 비교됩니다. 스코프가 `concept_folder_only`인 경우에는 실행되지 않습니다.

### 5단계: 확인 및 삭제

모든 후보 항목들은 확인 모달 창에 표시됩니다. 승인된 항목들은 시스템 휴지통으로 이동합니다. 각 삭제 작업 사이에서는 해당 작업을 취소할 수 있습니다.

### 스코프 모드

| 모드 | 비교 대상 |
|------|-------------------|
| `vault` (기본값) | concept 폴더를 제외한 모든 vault `.md` 파일들 |
| `concept_folder_only` | 폴더 내의 다른 콘셉트 노트들 |
| `include` | 지정된 폴더들에 있는 파일들 |
| `exclude` | concept 폴더와 목록에 있는 폴더를 제외한 모든 vault 파일 |

## 사용법

### 개념 추출 (독립형)

1. 메모 열기
2. 명령 팔레트에서 **"Notemd: Extract concepts"**를 실행하세요
3. 컨셉 노트는 설정된 폴더에 생성됩니다.

### 콘텐츠 추출 + 생성

**"Notemd: 개념 추출 및 제목 생성"** — 개념을 추출한 후 해당 개념 폴더에 있는 모든 노트에 대해 "제목으로부터 생성" 기능을 실행합니다. 이를 통해 새로 생성된 노트뿐만 아니라 폴더 내의 **모든** 개념 노트에 대한 콘텐츠가 생성됩니다.

### 선택한 내용에서 위키 링크 생성

1. 에디터에서 텍스트 강조 표시
2. **"Notemd: 선택한 항목에서 위키 링크 생성"**을 실행하세요.
3. 텍스트는 `[[...]]` 안에 포함되어 있으며, 개념 문서가 생성됩니다.

## 팁

- **먼저 컨셉 폴더를 설정하세요** — 이것이 없으면 메모가 생성되지 않습니다. 설정하지 않으면 경고 모달이 나타납니다.
- **작업별 모델 사용하기** — 개념 추출은 더 저렴한 모델(GPT-4o-mini, DeepSeek)을 사용하면 효과적입니다. 비싼 모델은 연구나 번역 용도로 아껴두세요.
- 동일한 개념을 지칭하는 여러 용어가 존재하는 기술 분야에서 **동의어 억제** 기능을 활성화합니다.
- **정기적으로 중복 제거** — 일괄 처리 후 중복에 가까운 항목을 찾아내기 위해 중복 제거 명령을 실행하세요.
- 매번 실행할 때 로그 파일은 추가되지 않고 덮어씌워집니다. 이전 결과가 필요하면 복사해 주세요.
