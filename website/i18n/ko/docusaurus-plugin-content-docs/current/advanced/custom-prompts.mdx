# 커스텀 프롬프트

<TLDR>
**Notemd를 사용하면 모든 작업 유형에 대한 기본 LLM 프롬프트를 재정의할 수 있습니다.** 사용자 정의 프롬프트를 통해 모델이 무엇을 우선시할지 제어할 수 있으며, 개념 추출을 특정 분야로 제한하거나 연결 스타일을 변경하거나 번역 톤을 조절할 수도 있습니다. `{{content}}`와 `{{concept}}`과 같은 프롬프트 변수는 실행 시에 대체됩니다. 집중 학습 도메인 기능은 사용자의 연구 분야를 앞에 붙여 추출 결과의 범위를 자동으로 설정해 줍니다.

이것은 [Obsidian AI 지식 관리 가이드](/docs/pillar-ai-knowledge)의 일부입니다.
</TLDR>

## 개요

Notemd의 내장 프롬프트는 일반적인 지식 작업을 위해 작성되었습니다. 이 프롬프트들은 다양한 분야에서 잘 작동하지만, 분야별로 더 개선할 수 있는 여지도 있습니다. 사용자 정의 프롬프트를 통해 LLM에게 자신의 사용 사례에 맞게 어떻게 행동해야 하는지 정확하게 지시할 수 있습니다. 이는 의료 용어만 추출하거나, 명명된 엔티티만 연결하거나, 형식적인 어조로 번역하는 것과 같은 작업을 포함합니다.

커스텀 프롬프트는 전체 시스템 프롬프트를 대체하지 않습니다. 이들은 Notemd이 파싱을 위해 의존하는 구조적 출력 형식(예: `CONCEPT:` 줄, 위키 링크 구문)을 그대로 유지한 채 **작업별 지시 사항 부분**을 덮어쓰거나 보완합니다.

## 작동 원리

### 프롬프트 아키텍처

모든 Notemd 작업은 동일한 프롬프트 구조를 따릅니다:

```
[System role: You are a helpful assistant...]
[Domain constraint: Relevant Fields: [{domain}]]   <-- focused learning (optional)
[Custom task prompt]                                <-- your override (optional)
[Output format instructions]                        <-- always preserved
[Content: {{content}}]                              <-- runtime substitution
```

1. **기본 시스템 프롬프트** -- 어시스턴트의 인격을 설정합니다. 덮어쓸 수 없습니다.
2. **집중 학습 도메인** -- 범위를 제한하기 위해 `Relevant Fields: [medicine / law / computer science / ...]`을 앞에 붙입니다. 이는 원시 프롬프트가 아닌 설정 전환 기능입니다.
3. **커스텀 작업 프롬프트** -- 기본 작업 지침을 대체합니다(예: 어떤 종류의 개념을 추출할지, 얼마나 적극적으로 연결할지).
4. **출력 형식** -- 구조적 규칙 (`CONCEPT:` 줄, 위키 링크 형식). 항상 해석 가능성을 보장하기 위해 Notemd가 뒤에 붙습니다.
5. **내용** -- 실제 메모 텍스트로, 런타임에 `{{content}}`을 통해 대체됩니다.

### 프롬프트 변수

변수는 실행 시점에 대체됩니다. 사용 가능한 변수는 작업에 따라 다릅니다:

| 변수 | 작업 | 대체 값 |
|----------|-------|---------------|
| `{{content}}` | 모두 | 메모 또는 선택된 텍스트 |
| `{{concept}}` | 컨셉 노트 템플릿 | 추출된 개념 이름 |
| `{{description}}` | 컨셉 노트 템플릿 | LLM로 생성된 컨셉 설명 |
| `{{backlinks}}` | 컨셉 노트 템플릿 | 링크가 포함된 출처 노트 목록 |
| `{{date}}` | 컨셉 노트 템플릿 | 현재 날짜 (YYYY-MM-DD) |
| `{{language}}` | 번역 | 목표 출력 언어 |
| `{{source}}` | 컨셉 노트 템플릿 | 출처 메모 파일명 |

### 집중 학습 영역

집중 학습 기능은 도메인 범위 지정을 위한 단축키입니다. 해당 분야를 명시하는 맞춤형 프롬프트를 작성하는 대신, 설정에서 `focusedLearningDomain`를 지정하면 됩니다.

```
Settings → Advanced → Focused Learning Domain
Example: "medicine" or "computer science" or "Chinese philosophy"
```

이 방식은 추출 프롬프트 앞에 `Relevant Fields: [medicine]`를 붙여 LLM이 도메인과 관련된 개념을 우선시하고 일반적인 개념들은 억제하도록 합니다. 이는 완전한 맞춤형 프롬프트보다 가벼우며 모든 추출 작업에서 사용할 수 있습니다.

## 구성 설정

| 설정 | 기본값 | 효과 |
|---------|---------|--------|
| `customAddLinksPrompt` | `''` | 기본적인 add-links 작업 프롬프트를 재정의합니다. |
| `customExtractConceptsPrompt` | `''` | 기본 컨셉 추출 프롬프트를 재정의합니다. |
| `customResearchPrompt` | `''` | 기본 연구 + 요약 프롬프트를 재정의합니다. |
| `customTranslatePrompt` | `''` | 기본 번역 프롬프트를 재정의합니다. |
| `customDiagramPrompt` | `''` | 기본 다이어그램 생성 프롬프트를 재정의합니다. |
| `focusedLearningDomain` | `''` | 추출 및 링크 프롬프트에 디스ци플린 컨텍스트를 앞에 붙이세요 |
| `extractConceptsMinimalTemplate` | `true` | 제목만 사용하는 콘셉트 노트 템플릿 사용 (`{{concept}}` 템플릿 범위에 영향을 줌) |
| `customConceptNoteTemplate` | `''` | 전체 변수 지원 기능이 포함된 콘셉트 노트 파일 템플릿을 재정의합니다. |

**커스텀 프롬프트 필드가 비어 있을 경우, 내장된 기본 프롬프트가 사용됩니다.**

## 예시

당신은 임상 시험 기록에서 개념을 추출하는 의학 연구원입니다. 기본 프롬프트는 의학 용어와 함께 일반 명사까지 추출하여 개념 폴더를 혼잡하게 만듭니다.

**해결 방법:** 집중할 학습 영역과 맞춤형 추출 프롬프트를 설정합니다.

1. `focusedLearningDomain`를 `medicine`로 설정하세요.
2. `customExtractConceptsPrompt`을/를 다음 값으로 설정합니다:

```
Extract only clinically relevant concepts: diseases, drug names,
anatomical terms, measurement endpoints, and statistical methods.
Ignore generic English nouns, brand names, and institutional names.
Prioritize MeSH terminology where applicable.
```

이제 임상 기록에 “Extract concepts”를 실행하면 LLM은 다음과 같은 결과를 생성합니다:

```
CONCEPT: Randomized Controlled Trial
CONCEPT: Double-Blind Method
CONCEPT: Myocardial Infarction
CONCEPT: Creatine Kinase-MB
CONCEPT: Kaplan-Meier Estimate
```

“hospital”나 “month”와 같은 원치 않는 일반적인 용어도 추출될 수 있는 기본 동작 대신.

## 팁

- 완전한 맞춤형 프롬프트를 작성하기 전에 **집중 학습 영역부터 시작**하세요. 단 한 가지 설정만으로도 80%의 사용 사례를 처리할 수 있습니다.
- **출력 형식 유지** -- 사용자 정의 프롬프트에서 “한 줄에 하나”나 “CONCEPT: prefix”와 같은 형식 지정 안내문을 절대 삭제하지 마십시오. Notemd의 파서는 이러한 안내문들에 의존합니다.
- **먼저 하나의 노트로 테스트해 보세요** -- 사용자 정의 프롬프트는 추출 범위에 예상치 못한 영향을 줄 수 있습니다. 일괄 처리하기 전에 먼저 하나의 노트에서 테스트를 진행해 주세요.
- **프롬프트는 간결하게 유지하세요** – 지나치게 긴 맞춤형 프롬프트는 토큰을 많이 소비하여 모델의 집중력을 떨어뜨릴 수 있습니다.
