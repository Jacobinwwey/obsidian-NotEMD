# Google 제공업체

<TLDR>
**Google는 기본 제공되는 Gemini 생성형 언어 API 전송 방식을 사용합니다.** 현재 프리셋은 `gemini-2.0-flash-exp`, `https://generativelanguage.googleapis.com/v1`, `temperature: 0.5`를 기본값으로 사용합니다. 이것은 OpenAI와 호환되는 프로필이 아닙니다.

이것은 [Obsidian AI 지식 관리 가이드](/docs/pillar-ai-knowledge)의 일부입니다.
</TLDR>

## 설치

Gemini 접근 권한이 있는 Google AI Studio 또는 Google 클라우드 API 키를 생성하고, `Google` 공급업체 프로필을 추가한 다음 기본적인 Generative Language Base URL는 그대로 유지합니다. 빠른 번역 및 다양한 노트 작업 흐름을 위해 기본 Gemini Flash 프로필을 사용하고, 더 크거나 더 강력한 Gemini 모델이 필요할 때는 각 작업에 맞게 모델을 재지정합니다.

OpenAI-호환 API를 통해 Gemini를 노출하는 게이트웨이를 명시적으로 사용하지 않는 한, OpenAI-호환 프로필에 Gemini 모델 ID를 포함시키지 마십시오.

## 엔드포인트 및 인증

| 필드 | 현재 프리셋 |
|-------|----------------|
| 전송 | `google` |
| API 키 | 필수 |
| 베이스 URL | `https://generativelanguage.googleapis.com/v1` |
| 기본 모델 | `gemini-2.0-flash-exp` |
| 온도 | `0.5` |
| 연결 테스트 | 채팅 전용 |

런타임이 `/models/{model}:generateContent?key={apiKey}`을 호출합니다. 플러그인 내의 네이티브 Gemini 엔드포인트가 연결된 방식상 키는 쿼리 파라미터로 전달됩니다.

## 모델 발견

Google은 `google-models` 발견 모드를 사용합니다. 발견 기능을 통해 API 키에서 사용할 수 있는 Gemini 모델들이 반환되어야 합니다. 선택기가 비어 있는 경우, 동일한 Base URL를 기준으로 해당 키를 테스트하여 해당 계정에서 모델 패밀리가 활성화되어 있는지 확인해야 합니다.

공급업체 레지스트리에서는 알려진 출력 토큰 한도가 모델에 따라 다릅니다. 긴 번역문이나 다이어그램 설명이 잘린 경우, 더 높은 출력 한도를 지원하는 Gemini 모델을 선택하거나 요청하는 출력 크기를 줄이십시오.

## 문제 해결

- `400`: 모델 ID는 일반적으로 네이티브 Gemini 루트와 일치하지 않거나, 요청이 모델이 지원하는 형태를 초과합니다.
- `401` 또는 `403`: 키 제한 사항, 프로젝트/API 활성화 여부, 그리고 해당 키가 Gemini에 접근할 수 있는지 확인합니다.
- `404`: 구성된 API 버전에서는 모델 ID를 사용할 수 없습니다.
- 이 프로필에서는 게이트웨이 예제들이 작동하지 않습니다. 기본 Google 공급자 대신 게이트웨이 사전 설정을 사용하십시오.

## 언제 사용할까요?

빠른 번역, 대량의 노트 정리, 그리고 Gemini의 비용/성능 균형이 직접적인 OpenAI나 Anthropic보다 우수한 워크플로우의 경우 Google를 사용하십시오. OpenAI와 호환되는 추론 모델 처리가 필요한 워크플로우의 경우에는 다른 제공업체를 선호하십시오.
