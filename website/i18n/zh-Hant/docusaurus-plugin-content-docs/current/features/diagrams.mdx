# 圖表

<TLDR>
先選擇**您需要的圖表類型**，再選擇其**可編輯的原始格式**。在 Obsidian 中預覽結果，之後即可將其匯出為 SVG、PNG 或 PDF。
</TLDR>

## 了解三種選項

| 選項 | 控制內容 | 範例 |
|---|---|---|
| 圖表類型 | 圖表的含義 | 流程圖、序列圖、電路圖、資料圖表 |
| 原始格式 | Notemd 所建立的、可編輯或可重複使用的檔案 | Mermaid、JSON Canvas、Draw.io、Drawnix、CircuitikZ |
| 匯出格式 | 用於分享的圖片或文件格式 | SVG、PNG、PDF |

Draw.io 與 Drawnix 屬於**原始格式**，而非圖表類型。CircuitikZ 是專為電路圖設計的原始格式。同一個預覽結果可以匯出為多種視覺格式，因此選擇 CircuitikZ 並不會限制您只能使用 SVG。

## 選擇原始格式

| 格式 | 最適用場景 | 原始檔案 |
|---|---|---|
| Mermaid | 直接儲存在 Markdown 中的圖表 | Mermaid 程式碼區塊 |
| JSON Canvas | Obsidian 內建的 canvas 工作流程 | `.canvas` |
| Draw.io | 後續在 diagrams.net 或相容工具中進行編輯 | `.drawio` |
| Drawnix | 手繪或白板式的編輯方式 | `.drawnix` |
| CircuitikZ | 有結構的電路與電晶體圖表 | `.tex` |

如果您不需要特定的編輯器，請將**Preferred source format** 設定為 **Auto**。Notemd 會根據所選的圖表類型自動挑選相容的格式。

## 產生並匯出圖表

1. 開啟筆記，或選取您想要轉換為圖表的文字。
2. 從 Notemd 的側邊欄或指令面板中選擇 **Generate diagram**。
3. 選擇一種**圖表類型**，或保持設定為 **Auto-detect**。
4. 選擇一種**原始格式**，或保持設定為 **Auto**。
5. 檢視預覽結果。
6. 選擇 **Save source file**、**Export SVG**、**Export PNG** 或 **Export PDF**。

PNG 與 PDF 會使用在 Notemd 設定中配置的圖片匯出 PPI 值。SVG 屬於向量圖像，因此不受 PPI 值的影響。

## 電路圖表

對於支援的電路相關需求，請選擇 **Circuit diagram** 並搭配 **CircuitikZ source file**。Notemd 會先產生結構化的電路描述，再寫入 CircuitikZ 原始碼。這種方式比要求模型直接撰寫不受限制的 LaTeX 更為可靠。

目前的 CircuitikZ 支援功能是刻意限制的。以通用來源的 NMOS 放大器以及支援的 CMOS 組成元件表現最佳。在將生成的電路用於技術文件或出版品之前，務必先檢查預覽結果與原始碼。

SVG 預覽是由 Notemd 產生的輔助檢視工具，並不能證明本地的 LaTeX 或 TikZJax 安裝已成功編譯 `.tex` 檔案。

## 疑難排解

### 結果出現錯誤的圖表類型

請不要使用 Auto-detect，而是直接選擇圖表類型。若是電路圖，請選擇 **Circuit diagram**，並使用 **Generate diagram** 指令，而非舊版的 **Summarise as Mermaid diagram** 指令。

### 某種原始格式不適合

將原始格式改回 **Auto**，或選擇適用於該圖表類型的格式。CircuitikZ 只適用於電路圖；Draw.io 與 Drawnix 則適用於支援的結構化圖表。

### 缺少匯出按鈕

SVG、PNG 與 PDF 需要視覺預覽功能。如果只有原始文字，請先儲存原始檔案，再在對應的外部工具中打開它。

### CircuitikZ 檔案無法在外部編譯

請確認您的 LaTeX 環境已包含 `circuitikz` 套件。此插件並未內建完整的 LaTeX 發行版。
