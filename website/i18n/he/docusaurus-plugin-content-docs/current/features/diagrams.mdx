# תרשימים

<TLDR>
בחרו **איזה סוג של תרשים אתם צריכים**, ולאחר מכן בחרו את **פורמט המקור הניתן לעריכה** שלו. עיינו בתוצאה ב‑Obsidian וייצאו אותה כ‑SVG, PNG או PDF.
</TLDR>

## הבנת שלושת האפשרויות

| אפשרות | מה היא שולטת בו | דוגמאות |
|---|---|---|
| סוג התרשים | מה משמעותו של התרשים | תרשים זרימה, תרשים סדרה, תרשים מעגלים, גרף נתונים |
| פורמט המקור | הקובץ הניתן לעריכה או לשימוש חוזר ש‑Notemd יוצר | Mermaid, JSON Canvas, Draw.io, Drawnix, CircuitikZ |
| פורמט הייצוא | התמונה או המסמך שיישלחו לשיתוף | SVG, PNG, PDF |

Draw.io ו‑Drawnix הם **פורמטי מקור**, ולא סוגי תרשימים. CircuitikZ הוא פורמט מקור שנועד לתרשימי מעגלים. אותה תוצאת תצוגה ניתן לייצא למספר פורמטים ויזואליים, כך שבחירה ב‑CircuitikZ אינה מגבילה אתכם ל‑SVG בלבד.

## בחירת פורמט מקור

| פורמט | הכי מתאים ל‑ | קובץ המקור |
|---|---|---|
| Mermaid | תרשימים שנשמרים ישירות ב‑Markdown | בלוק קוד Mermaid |
| JSON Canvas | עבודה עם לוח העבודה המובנה של Obsidian | `.canvas` |
| Draw.io | עריכה מאוחרת ב‑diagrams.net או בכלים תואמים | `.drawio` |
| Drawnix | עריכה בסגנון שרטוט ידני או לוח לבן | `.drawnix` |
| CircuitikZ | מעגלים חשמליים ומעגלי טרנזיסטורים ממוקדים | `.tex` |

השאירו את **Preferred source format** על **Auto** אם אינכם זקוקים למערכת עריכה מסוימת. Notemd יבחר פורמט תואם לסוג התרשים שבחרתם.

## יצירה וייצוא של תרשים

1. פתחו את ההערה או בחרו את הטקסט שאתם רוצים להמיר לתרשים.
2. בחרו **Generate diagram** מהסרגל הצדדי של Notemd או מלוח הפקודות.
3. בחרו **סוג תרשים**, או השאירו אותו על **Auto-detect**.
4. בחרו **פורמט מקור**, או השאירו אותו על **Auto**.
5. בדקו את תוצאת התצוגה.
6. בחרו **Save source file**, **Export SVG**, **Export PNG** או **Export PDF**.

PNG ו‑PDF משתמשים ברמת ה‑PPI של ייצוא התמונה שנקבעה בהגדרות Notemd. SVG נשאר תמונה וקטורית ואינו מושפע מערך PPI.

## תרשימי מעגלים

בחרו **Circuit diagram** עם **CircuitikZ source file** עבור בקשות לתרשימי מעגלים שנתמכות. Notemd יוצר תחילה תיאור ממוקד של המעגל ולאחר מכן כותב את קוד CircuitikZ. גישה זו אמינה יותר מאשר בקשה למודל לכתוב LaTeX ללא הגבלות.

התמיכה הנוכחית ב‑CircuitikZ מוגבלת בכוונה. מגברי NMOS עם מקורות נפוצים ובלוקים CMOS שנתמכים יעבדו הכי טוב. תמיד בדקו את תוצאת התצוגה ואת קוד המקור לפני שאתם משתמשים במעגל שנוצר בעבודות טכניות או בפרסומים.

תוצאת ה‑SVG היא רק כלי עזר לבדיקה שנוצר על ידי Notemd. היא אינה הוכחה לכך שהתקנת LaTeX מקומית או TikZJax הצליחה לסקור את קובץ ה‑`.tex` בהצלחה.

## פתרון בעיות

### התוצאה היא סוג תרשים שגוי

בחרו במפורש את סוג התרשים במקום להשתמש ב‑Auto-detect. עבור מעגלים, בחרו **Circuit diagram** והשתמשו בפקודה **Generate diagram**, ולא בפקודה הישנה **Summarise as Mermaid diagram**.

### פורמט המקור אינו מתאים

החזירו את פורמט המקור ל‑**Auto**, או בחרו פורמט שנועד לסוג התרשים הספציפי. CircuitikZ מיועד לתרשימי מעגלים; Draw.io ו‑Drawnix מיועדים לתרשימים ממוקדים שנתמכים.

### לחצני הייצוא חסרים

SVG, PNG ו‑PDF דורשים תצוגה ויזואלית. אם יש רק טקסט מקור, שמרו את קובץ המקור ופתחו אותו בכלי חיצוני התואם.

### קובץ CircuitikZ אינו מתקבל בעריכה חיצונית

בדקו שסביבת ה‑LaTeX שלכם כוללת את החבילה `circuitikz`. התוסף אינו מכיל חבילת LaTeX מלאה.
