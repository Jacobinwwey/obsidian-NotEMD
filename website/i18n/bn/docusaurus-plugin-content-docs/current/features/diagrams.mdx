# ডায়াগ্রামসমূহ

<TLDR>
আপনার কী ধরনের ডায়াগ্রাম দরকার তা নির্বাচন করুন, এরপর সেটির **সম্পাদনযোগ্য সোর্স ফরম্যাট** বেছে নিন। Obsidian-এ ফলাফলটি প্রিভিউ করুন এবং এটিকে SVG, PNG অথবা PDF হিসেবে এক্সপোর্ট করুন।
</TLDR>

## তিনটি বিকল্প সম্পর্কে ধারণা নেওয়া

| বিকল্প | এটি কী নিয়ন্ত্রণ করে | উদাহরণসমূহ |
|---|---|---|
| ডায়াগ্রামের ধরন | ডায়াগ্রামটির অর্থ কী | ফ্লোচার্ট, সিকোয়েন্স ডায়াগ্রাম, সার্কিট ডায়াগ্রাম, ডেটা চার্ট |
| সোর্স ফরম্যাট | Notemd দ্বারা তৈরি হওয়া সম্পাদনযোগ্য বা পুনরায় ব্যবহারযোগ্য ফাইল | Mermaid, JSON Canvas, Draw.io, Drawnix, CircuitikZ |
| এক্সপোর্ট ফরম্যাট | শেয়ারিংয়ের জন্য ব্যবহৃত ছবি বা ডকুমেন্ট | SVG, PNG, PDF |

Draw.io এবং Drawnix হলো **সোর্স ফরম্যাট**, ডায়াগ্রামের ধরন নয়। CircuitikZ হলো সার্কিট ডায়াগ্রামের জন্য ডিজাইন করা একটি সোর্স ফরম্যাট। একই প্রিভিউটিকে বিভিন্ন ভিজ্যুয়াল ফরম্যাটে এক্সপোর্ট করা যায়; তাই CircuitikZ নির্বাচন করলেও আপনি শুধুমাত্র SVG-এর সাথেই সীমাবদ্ধ থাকবেন না।

## একটি সোর্স ফরম্যাট নির্বাচন করুন

| ফরম্যাট | কোন ক্ষেত্রে সবচেয়ে ভালো | সোর্স ফাইল |
|---|---|---|
| Mermaid | Markdown-এর মধ্যে সরাসরি ডায়াগ্রাম রাখার জন্য | Mermaid কোড ব্লক |
| JSON Canvas | Obsidian-এর নেটিভ ক্যানভাস ওয়ার্কফ্লোর জন্য | `.canvas` |
| Draw.io | diagrams.net বা সংগত টুলগুলিতে পরবর্তীতে সম্পাদনা করার জন্য | `.drawio` |
| Drawnix | হাতে আঁকা বা হোয়াইটবোর্ড-স্টাইলের সম্পাদনার জন্য | `.drawnix` |
| CircuitikZ | কাঠামোগত বৈদ্যুতিক ও ট্রানজিস্টর সার্কিটের জন্য | `.tex` |

যদি আপনার নির্দিষ্ট কোনো এডিটরের প্রয়োজন না হয়, তাহলে **Preferred source format**-কে **Auto**-এ রেখে দিন। Notemd নির্বাচিত ডায়াগ্রামের ধরনের জন্য উপযুক্ত ফরম্যাটটি নির্বাচন করবে।

## ডায়াগ্রাম তৈরি ও এক্সপোর্ট করা

1. নোটটি খুলুন অথবা যে টেক্সটটি ভিজ্যুয়ালাইজ করতে চান সেটি নির্বাচন করুন।
2. Notemd-এর সাইডবার বা কমান্ড প্যালেট থেকে **Generate diagram** নির্বাচন করুন।
3. একটি **ডায়াগ্রামের ধরন** নির্বাচন করুন, অথবা এটিকে **Auto-detect**-এ রেখে দিন।
4. একটি **সোর্স ফরম্যাট** নির্বাচন করুন, অথবা এটিকে **Auto**-এ রেখে দিন।
5. প্রিভিউটি পরীক্ষা করুন।
6. **Save source file**, **Export SVG**, **Export PNG** অথবা **Export PDF** নির্বাচন করুন।

PNG এবং PDF-এ Notemd-এর সেটিংসে কনফিগার করা ইমেজ এক্সপোর্ট PPI ব্যবহৃত হয়। SVG হলো একটি ভেক্টর ইমেজ, তাই এটি PPI মানের দ্বারা প্রভাবিত হয় না।

## সার্কিট ডায়াগ্রামসমূহ

সমর্থিত সার্কিট অনুরোধের জন্য **Circuit diagram** নির্বাচন করুন এবং **CircuitikZ source file** ব্যবহার করুন। Notemd প্রথমে একটি কাঠামোগত সার্কিট বিবরণ তৈরি করে এবং তারপর CircuitikZ সোর্সটি লেখে। মডেলকে অনির্বাধ LaTeX লেখার জন্য অনুরোধ করার চেয়ে এটি অনেক বেশি নির্ভরযোগ্য।

বর্তমানে CircuitikZ-এর সমর্থন ইচ্ছাকৃতভাবে সীমিত রাখা হয়েছে। সাধারণ-সোর্সযুক্ত NMOS অ্যাম্প্লিফায়ার এবং সমর্থিত CMOS বিল্ডিং ব্লকগুলি সবচেয়ে ভালোভাবে কাজ করে। প্রযুক্তিগত বা প্রকাশনা কাজে তৈরি করা সার্কিটগুলি ব্যবহার করার আগে সর্বদা প্রিভিউ ও সোর্সটি পরীক্ষা করুন।

SVG প্রিভিউটি হলো Notemd দ্বারা তৈরি একটি পর্যালোচনা-সহায়ক উপাদান। এটি কোনো স্থানীয় LaTeX বা TikZJax ইনস্টলেশন `.tex` ফাইলটিকে সফলভাবে কম্পাইল করেছে কিনা তার প্রমাণ নয়।

## সমস্যা সমাধান

### ফলাফলটি ভুল ডায়াগ্রামের ধরনের হয়েছে

Auto-detect ব্যবহার করার পরিবর্তে স্পষ্টভাবে ডায়াগ্রামের ধরন নির্বাচন করুন। সার্কিটের ক্ষেত্রে **Circuit diagram** নির্বাচন করুন এবং **Generate diagram** কমান্ড ব্যবহার করুন; পুরানো **Summarise as Mermaid diagram** কমান্ডটি ব্যবহার করবেন না।

### কোনো সোর্স ফরম্যাট উপযুক্ত নয়

সোর্স ফরম্যাটটিকে **Auto**-এ ফিরিয়ে দিন, অথবা ঐ ডায়াগ্রামের ধরনের জন্য উপযুক্ত ফরম্যাট নির্বাচন করুন। CircuitikZ হলো সার্কিট ডায়াগ্রামের জন্য; Draw.io এবং Drawnix হলো সমর্থিত কাঠামোগত ডায়াগ্রামসমূহের জন্য।

### এক্সপোর্ট বাটনগুলি দেখা যাচ্ছে না

SVG, PNG এবং PDF-এর জন্য একটি ভিজ্যুয়াল প্রিভিউ প্রয়োজন। যদি শুধুমাত্র সোর্স টেক্সটই থাকে, তাহলে সোর্স ফাইলটি সংরক্ষণ করুন এবং এটিকে সংগত বাহ্যিক টুলে খুলুন।

### CircuitikZ ফাইলটি বাহ্যিকভাবে কম্পাইল হচ্ছে না

নিশ্চিত হওয়া উচিত যে আপনার LaTeX এনভায়রনমেন্টে `circuitikz` প্যাকেজটি রয়েছে। এই প্লাগইনটি কোনো সম্পূর্ণ LaTeX ডিস্ট্রিবিউশন সহ আসে না।
